❓ 금융정보 도움말

금융정보(MyCash)는 은행거래(bank_after)와 신용카드(card_after)를 하나로 합쳐 병합 관리·위험도 분류·종합 분석을 제공합니다.

목차

  • 1. 처음 사용하는 경우 (빠른 시작)
  • 2. 사용 흐름
  • 3. 메뉴·버튼별 안내
  • 4. cash_after 생성(병합)
  • 5. 위험도분류(1~10호)
  • 6. 파일 위치 정리
  • 7. 문제 해결
  • 8. 자주 묻는 질문

1. 처음 사용하는 경우 (빠른 시작)

  1. 은행·카드 데이터 준비: 먼저 은행거래와 신용카드 메뉴에서 각각 전처리·카테고리 적용을 끝내 bank_after, card_after를 만들어 두세요. 둘 중 하나만 있어도 병합 가능합니다.
  2. 금융정보 병합작업: 상단 메뉴에서 병합작업을 클릭한 뒤, 화면의 「cash_after 생성(병합)」 또는 「병합작업 다시 실행」 버튼을 누릅니다. bank_after + card_after가 합쳐져 cash_after.json이 생성되고 1~10호 위험도가 자동 적용됩니다.
  3. 업종분류·종합분석: 업종분류에서 위험도별 필터·출력, 종합분석에서 통계·차트와 검토 종합의견을 확인합니다.

중요: 금융정보 메뉴에 들어갈 때마다 기존 cash_after.json이 삭제됩니다. 데이터를 보려면 매번 「cash_after 생성(병합)」을 실행해야 합니다.

2. 사용 흐름

  1. 은행·카드 전처리: .source/Bank, .source/Card에 원본 엑셀을 넣고 각각 전처리하여 bank_after, card_after를 만듭니다.
  2. 금융정보 병합작업(/cash/): 화면에서 은행·카드 데이터를 확인한 뒤 「cash_after 생성(병합)」을 클릭합니다. bank_after + card_after가 합쳐져 MyCash/cash_after.json이 생성되고, 1~10호 위험도가 자동 적용됩니다.
  3. 금융정보 업종분류(/cash/category): 「금융정보(위험도분류) 적용후」 테이블에서 위험도키워드·위험도분류·위험도·필터·출력을 확인합니다.
  4. 금융정보 종합분석(/cash/analysis/basic): cash_after 기준 통계·차트를 확인합니다.
  5. 금융정보 검토 종합의견: 종합분석 페이지 우측 컨테이너에서 확인합니다.

참고: 금융정보 메뉴 진입 시 기존 cash_after.json이 삭제되므로, 데이터를 보려면 전처리 후 다시 「cash_after 생성(병합)」을 실행하세요.

3. 메뉴·버튼별 안내

금융정보 병합작업 (/cash/)

은행거래(bank_after)와 신용카드(card_after) 테이블을 한 화면에서 볼 수 있습니다.

  • 「cash_after 생성(병합)」 / 「병합작업 다시 실행」: 두 데이터를 합쳐 MyCash/cash_after.json을 만들고, category_table 기반 업종분류·위험도 매칭과 1~10호 위험도 지표를 적용한 뒤 저장합니다. 병합 후 화면에 결과가 반영됩니다.
  • bank_after·card_after가 없으면 "데이터가 없습니다" 등 메시지가 나올 수 있습니다. 은행·신용카드 메뉴에서 먼저 전처리·카테고리 적용을 완료하세요.

금융정보 업종분류 (/cash/category)

카테고리 정의 테이블과 금융정보(위험도분류) 적용후 테이블을 제공합니다.

  • 적용후 테이블: cash_after.json을 위험도·위험도키워드·거래일·거래시간 내림차순으로 정렬. 헤더는 스크롤 시 고정됩니다.
  • 각 컬럼의 드롭다운으로 필터링하고 「📄 출력」으로 인쇄용 창을 띄울 수 있습니다. 1호(분류제외지표) 행은 출력 시 제외되며, 위험도 0.1 이상만 인쇄됩니다.

금융정보 종합분석 (/cash/analysis/basic)

cash_after 기준 전체 통계, 카테고리별·월별·금융사별 집계와 차트를 제공합니다. 우측 컨테이너에 검토 종합의견이 표시됩니다.

4. cash_after 생성(병합)

bank_after와 card_after를 병합할 때 다음이 수행됩니다.

  • 금융사·계좌번호·거래일·거래시간·입금액·출금액·기타거래·키워드·카테고리·사업자번호·폐업 등 컬럼 병합
  • category_table 기반으로 위험도키워드→위험도분류·위험도 매칭 (data/category_table.json의 위험도 분류 행)
  • risk_indicators 적용: 1~10호 순서로 위험도키워드·위험도분류·위험도 저장(1호 기본 위험도 0.1)
  • 고위험(가상자산·증권투자·금전대부) 카테고리 보정 시 위험도 5.0 설정

결과 파일: MyCash/cash_after.json

5. 위험도분류(1~10호) 상세

금융정보(cash_after) 생성(병합) 시 1호(분류제외지표, 0.1)를 기본값으로 두고, 2호 → 3호 → … → 10호 순차 적용하며 조건 만족 시 덮어씁니다. 프린터 출력 시에는 1호(분류제외지표) 행은 제외됩니다.

4.1 검색 텍스트 (5~10호 공통)

5호부터 10호까지 매칭 시 사용하는 검색 텍스트는 (카테고리 + 키워드) + 기타거래를 붙인 문자열입니다. 공백 기준 동일 단어는 한 번만 남기고 키워드 포함 여부를 판단합니다.

4.2 호별 상세 조건 (호수, 위험도분류, 위험도, 금액/횟수 조건, 키워드·기타 조건)

호수위험도분류위험도금액/횟수 조건키워드·기타 조건
1호분류제외지표0.1금액제한 없음2호~10호에 해당하지 않은 거래
2호심야폐업지표0.5폐업: 금액 무관 / 심야: 출금액 10만원 이상폐업 컬럼이 폐업인 거래(금액 무관), 또는 거래시간이 심야구분에 해당하고 출금액 10만원(포함) 이상인 거래
3호자료소명지표1.0출금액 500만원 이상해당 행의 키워드(또는 기타거래) 값을 위험도키워드에 저장
4호비정형지표1.5출금만 100만원 이상, 같은 키워드 3회 이상해당 키워드(특정인)를 위험도키워드에 저장
5호투기성지표2.0출금만 50만원 이상 (입금 0)검색 텍스트에 증권, 선물, 자산운용, 위탁, 증권입금 중 하나 포함
6호사기파산지표2.5출금만 50만원 이상 (입금 0)검색 텍스트에 대부, P2P, 카드깡, 원리금 중 하나 포함
7호가상자산지표3.0출금만 50만원 이상 (입금 0)카테고리 "가상자산" 또는 업비트, 빗썸, 코인원, 코빗, 가상자산, VASP, 거래소, 코인, 비트코인, 암호화폐 중 하나 포함
8호자산은닉지표3.5출금만 50만원 이상 (입금 0)검색 텍스트에 해외송금, 고액현금인출, 외화송금, 영문성명, Wise, TransferWise, SWIFT 중 하나 포함
9호과소비지표4.0출금만 30만원 이상 (입금 0)검색 텍스트에 백화점, 명품, 귀금속, 유흥, 고가가전, 고가가구, 유흥주점, 무도장, 콜라텍, 댄스홀 중 하나 포함
10호사행성지표5.0출금만 10만원 이상 (입금 0)검색 텍스트에 경마, 복권, 사설도박, 도박기계, 사행성게임기, 오락기구, 휴게텔, 키스방, 대화방, 안마, 마사지, 사행성, 도박 중 하나 포함

4.3 위험도키워드 저장 방식

  • 1호·2호: 1호는 키워드 저장 없음. 2호는 심야 또는 폐업 조건 해당 시 위험도 0.5로 설정.
  • 3호·4호: 해당 행의 키워드(또는 기타거래) 값을 위험도키워드에 저장.
  • 5~10호: 검색 텍스트에서 처음 매칭된 키워드 하나를 위험도키워드에 저장.

4.4 고위험 카테고리 보정

cash_after 생성 시 가상자산·증권투자·금전대부 등 고위험 카테고리에 해당하는 행은 별도 보정으로 위험도 5.0이 설정될 수 있습니다. (category_table 기반 또는 내부 규칙 적용)

4.5 프린터 출력

금융정보(위험도분류) 적용후(cash_after.json)를 프린터로 출력할 때 1호(분류제외지표) 행은 제외하고, 위험도 0.1 이상인 행만 인쇄됩니다.

7. 문제 해결

「cash_after 생성 중...」만 보이고 생성이 안 돼요

금융정보 병합후에서 데이터가 없을 때 자동으로 cash_after 생성을 시도하며 이 메시지를 띄웁니다. 생성에 실패하면 그 아래에 서버가 반환한 오류 사유가 표시됩니다. 흔한 원인은 다음과 같습니다.

  • bank_after와 card_after가 모두 없거나 비어 있음 — cash_after는 bank_after + card_after 병합으로 생성됩니다. 둘 중 하나라도 있으면 생성 가능합니다. 은행 또는 신용카드 전처리에서 병합작업 다시 실행으로 한 개 이상 만든 뒤, 금융정보에서 병합작업 다시 실행을 시도하세요.
  • 병합 결과 데이터가 비어 있음 — bank_after·card_after는 있으나 읽기 오류나 컬럼 문제로 병합 결과가 0건인 경우. 은행·카드 after 파일이 정상 생성되었는지 해당 메뉴에서 확인하세요.
  • 임시 파일 쓰기/교체 실패 — MyCash 폴더에 쓰기 권한이 없거나, 디스크 부족·한글 경로 등으로 cash_after.json 저장이 실패한 경우. 프로젝트 경로를 한글이 없는 경로(C:\CursorWorkspace 등)로 옮기고 다시 시도하세요.
  • 요청 시간 초과 — 데이터가 많을 때 120초 안에 완료되지 않으면 타임아웃됩니다. 새로고침(F5) 후 병합작업 다시 실행 버튼으로 재시도하세요.

수동으로 생성하려면: 금융정보 병합작업 페이지에서 병합작업 다시 실행 버튼을 누르면 동일한 API(POST /api/generate-category)가 호출됩니다. 실패 시 버튼 위/아래에 표시되는 오류 문구를 확인하세요.

데이터가 없습니다

금융정보 업종분류에서 이 메시지가 나오면 cash_after.json이 없거나 비어 있는 상태입니다. 금융정보 병합작업에서 cash_after 생성(병합)을 실행하세요. bank_after와 card_after 둘 중 하나라도 있으면 cash_after 생성이 가능합니다.

위험도가 모두 0.1로 나와요

1~10호 조건에 맞는 거래가 없거나, 예전에 생성된 cash_after를 쓰는 경우입니다. cash_after 생성(병합)을 다시 실행하면 최신 위험도 규칙이 적용됩니다.

한글 깨짐 / Add-Content 오류

경로에 한글이 있거나 TEMP 폴더가 한글 경로일 때 발생할 수 있습니다. readme의 한글_깨짐_Add-Content_오류.md, start-cursor-no-hangul.ps1을 참고해 한글이 없는 경로(C:\CursorWorkspace 등)에서 작업하세요.

8. 자주 묻는 질문

Q. 금융정보에 들어가면 cash_after가 사라져요.
의도된 동작입니다. 금융정보 메뉴 진입 시 cash_after.json을 삭제하므로, 매번 「cash_after 생성(병합)」으로 최신 병합·위험도 결과를 만듭니다.

Q. 위험도키워드에 지표명(비정형지표 등)이 들어가요.
1호·2호는 별도 저장 없음/심야 조건. 3·4호는 해당 행 키워드, 5~10호는 매칭된 키워드(증권, 대부 등)를 저장합니다. cash_after를 다시 생성하면 반영됩니다.

Q. 통합 도움말은 어디서 보나요?
홈에서 금융정보 보고서 또는 /help로 접속하면 은행·카드·금융정보 전체 도움말을 볼 수 있습니다.
