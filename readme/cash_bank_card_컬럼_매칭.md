# cash_after.json ↔ bank_after / card_after 컬럼 매칭

금융정보(cash_after)는 **은행거래(bank_after)** 와 **신용카드(card_after)** 를 하나의 통일된 스키마로 합친 결과입니다.  
병합 시 각 소스 컬럼이 cash_after의 어떤 컬럼으로 들어가는지 정리합니다.

---

## 1. cash_after 최종 컬럼 (저장 순서)

| cash_after 컬럼 | 설명 |
|----------------|------|
| 금융사 | 은행명 또는 카드사명 |
| 계좌번호 | 은행 계좌번호 또는 카드번호(마스킹) |
| 거래일 | 거래일 또는 이용일 |
| 거래시간 | 거래시간 또는 이용시간 |
| 입금액 | 입금액 |
| 출금액 | 출금액 |
| 취소 | 취소 여부 |
| 기타거래 | 적요·내용 등 요약(은행) / 가맹점명(카드) |
| 키워드 | 분류용 키워드 |
| 카테고리 | 계정과목/카테고리 |
| 사업자번호 | 사업자번호(은행은 공란) |
| 폐업 | 폐업 여부(카드만 값 있음, 은행은 공란) |
| 출처 | `은행거래` 또는 `신용카드` |
| 위험도키워드 | 업종분류·위험도 매칭용 |
| 위험도분류 | 1~10호 분류명 |
| 위험도 | 위험도 점수(0.1~5.0) |

## 2. bank_after → cash_after 매칭

| bank_after 컬럼 | → cash_after 컬럼 | 비고 |
|-----------------|-------------------|------|
| 은행명 | **금융사** | |
| 계좌번호 | **계좌번호** | |
| 거래일 | **거래일** | |
| 거래시간 | **거래시간** | |
| 입금액 | **입금액** | |
| 출금액 | **출금액** | |
| 취소 | **취소** | (구분→취소 후 사용) |
| 기타거래 | **기타거래** | bank_after 기타거래 그대로 |
| 키워드 | **키워드** | |
| 카테고리 | **카테고리** | |
| 사업자번호 | **사업자번호** | 은행은 **항상 공란** `""` |
| (없음) | **폐업** | 은행은 **항상 공란** `""` |
| (없음) | **출처** | **`은행거래`** 고정 |

- bank_after에는 **잔액** 컬럼이 없고(삭제됨), **사업자번호·폐업**은 출금액 뒤에 있으나 cash 병합 시에는 **사업자번호/폐업 값을 쓰지 않고** 공란으로 채웁니다.

## 5. 참고

- **병합 순서**: bank_after 행 먼저, 이어서 card_after 행.  
- **위험도**: 병합 후 `risk_indicators.apply_risk_indicators()`로 1~10호 적용.  
- **출처**: 요약/필터용. 은행건·카드건 구분에 사용.  
- **폐업**: 카드의 과세유형 '폐업'만 cash에 반영하며, 은행은 해당 없어 항상 공란.
