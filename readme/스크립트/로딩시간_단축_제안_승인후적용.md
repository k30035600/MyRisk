# 로딩 시간 단축 — 제안 (승인 후 적용)

bank/card/cash 페이지 체감 로딩을 줄이기 위한 **구체적 제안**입니다. **승인해 주시면 해당 항목을 적용**하겠습니다.

---

## 이미 적용된 것 (참고)

- **캐시**: cash_after, bank_after, card_after 파일 로드 후 메모리 캐시로 재사용. 같은 세션에서 재조회 시 파일 재읽기 없음.
- **gzip**: app.py에서 JSON/텍스트 응답 1KB 이상 시 gzip 압축 전송.
- **limit + 나머지 1회**: 첫 요청 limit=1000, total>1000일 때만 offset=1000으로 한 번 더 요청해 요청 횟수 최소화.

---

## 제안 항목 (승인 후 적용)

### 1. 프론트: 독립 API 병렬 요청 (권장)

**내용**: 페이지 로드 시 서로 의존하지 않는 API를 동시에 호출합니다.  
예: 테이블 데이터(`loadRightData`)와 카테고리 테이블(`loadCategoryQueryTable`)을 `Promise.all([...])`로 한 번에 요청.

**효과**: 순차 호출 대비 전체 대기 시간 단축.  
**대상**: MyBank, MyCard, MyCash의 index/category/analysis 페이지에서 두 개 이상의 독립 fetch가 있는 구간.

**승인 시**: 해당 페이지의 스크립트에서 병렬 호출로 수정.

---

### 2. 프론트: 첫 화면 우선 렌더링 (선택)

**내용**: 테이블 데이터 수신 즉시 테이블부터 그리고, 차트(월별 추이 등)는 다음 프레임(`requestAnimationFrame` 또는 `setTimeout(..., 0)`)에서 요청·렌더링.

**효과**: "먼저 보이는 것"을 빨리 표시해 체감 대기 시간 감소.  
**대상**: 기본분석/종합분석 페이지.

**승인 시**: 해당 페이지에서 테이블 렌더 후 차트 API 호출 순서로 변경.

---

### 3. API: fetch 타임아웃 통일 (권장)

**내용**: 프론트에서 데이터 요청 시 타임아웃을 60초(또는 120초)로 통일. 무한 로딩 방지.

**효과**: 응답이 늦을 때 무한 대기 대신 타임아웃 후 "시간 초과" 안내 가능.  
**대상**: bank/card/cash 공통으로 fetch를 쓰는 모든 페이지.

**승인 시**: 공통 유틸 또는 각 fetch 래퍼에 timeout 적용.

---

### 4. 로딩 표시 문구 명확화 (권장)

**내용**: "데이터 로딩 중…", "카테고리 불러오는 중…" 등 문구를 한 곳에서 통일하고, 가능하면 스피너/진행 표시를 유지.

**효과**: 사용자가 대기 상태를 명확히 인지해 체감 대기 시간 완화.  
**대상**: bank/card/cash 템플릿의 로딩 메시지.

**승인 시**: 기존 로딩 문구 검색 후 통일·보강.

## 요약 표

| 번호 | 항목 | 구분 | 효과 | 적용 상태 |
|------|------|------|------|-----------|
| 1 | 독립 API 병렬 요청 | 권장 | 전체 대기 시간 단축 | ✅ **적용**: MyBank/index는 이미 병렬 호출 (async/await, not awaited in DOMContentLoaded). MyCash도 Promise.all 사용. |
| 2 | 첫 화면 우선(테이블→차트) | 선택 | 체감 로딩 개선 | (분석 페이지 차트는 테이블 데이터 기반 렌더링이므로 별도 fetch 없음 — 적용 불필요) |
| 3 | fetch 타임아웃 통일 | 권장 | 무한 로딩 방지 | ✅ **적용**: MyBank/index(60초), MyBank/analysis(60초), MyCash/index(120초), MyCash/analysis(60초), MyCard/analysis(60초)에 `fetchWithTimeout` 추가. MyCard/index는 이미 60초 적용됨. |
| 4 | 로딩 문구 통일·명확화 | 권장 | 체감 개선 | (현재 "데이터 로딩중 ... (0건)" 등 이미 일관적으로 사용됨 — 별도 수정 불필요) |

## 적용 상세

- **fetchWithTimeout(url, options, timeoutMs)**: AbortController 기반 타임아웃 fetch 유틸. 지정 시간(기본 60초/120초) 내 응답 없으면 자동 abort.  
  - 파일: `MyBank/templates/index.html`, `MyBank/templates/analysis_basic.html`, `MyCard/templates/analysis_basic.html`, `MyCash/templates/index.html`, `MyCash/templates/analysis_basic.html`
- **코드 개선 5번**: bank_app.py·card_app.py 핵심 `except: pass` → `[WARN]` 로그로 보강.
