# MyRisk 코드 개선안 일괄 제안

전체 코드 검토 후 정리한 개선 제안입니다. **아래 권장·선택 항목이 적용됨** (일괄 적용 완료).

---

## 1. start_web.py — gunicorn 실패 시 fallback (권장)

**현상**: Railway/Heroku에서는 `gunicorn`으로 실행하지만, **Windows 로컬**에서 `python start_web.py`를 실행하면 gunicorn이 없어 `FileNotFoundError`로 종료됩니다.

**제안**: `execvp` 실패 시(예: gunicorn 미설치) **waitress**로 `app:app` 실행. 로컬에서 start_web.py만으로도 서버 기동 가능.

```python
# gunicorn 실행 실패 시 (Windows 등) waitress로 대체
try:
    os.execvp("gunicorn", [...])
except (OSError, FileNotFoundError):
    from waitress import serve
    from app import app
    serve(app, host="0.0.0.0", port=int(port))
```

## 2. app.py — 버전 기본값 상수화 (선택)

**현상**: `_get_version()` 내 기본값 `'26/02/05'`가 매직 스트링으로 있음.

**제안**: 모듈 상단에 `_VERSION_DEFAULT = '26/02/05'` 정의 후, 실패 시 `return _VERSION_DEFAULT` 사용. 변경 시 한 곳만 수정 가능.

---

## 3. lib/data_json_io.py — JSON 직렬화 fallback 중복 제거 (선택)

**현상**: `ImportError` 시 인라인으로 정의하는 `_json_serializable` 함수가 `lib.shared_app_utils.json_safe_val`과 거의 동일함.

**제안**: fallback용 함수를 `shared_app_utils`에 `_json_safe_val_fallback`으로 두고, `data_json_io`에서는 `from lib.shared_app_utils import json_safe_val as _json_serializable` 실패 시 `from lib.shared_app_utils import _json_safe_val_fallback as _json_serializable` 시도. (또는 그대로 두어도 동작에는 문제 없음.)

---

## 4. run_full_flow.py — 서버 기동 후 포트 확인 (선택)

**현상**: `subprocess.Popen`으로 app.py를 띄운 뒤 곧바로 "5. http://127.0.0.1:8080"만 출력. 실제로 포트가 열렸는지 알 수 없음.

**제안**: Popen 후 1~2초 대기한 뒤, `socket`으로 127.0.0.1:8080 연결 시도. 성공하면 "Server ready", 실패하면 "WARN: Server may not be ready yet" 로그. (선택 사항.)

---

## 5. 예외 처리 — 중요 API 실패 시 로그 (선택)

**현상**: `except Exception: pass`로 조용히 넘기는 구간이 많음. 디버깅 시 원인 파악이 어려울 수 있음.

**제안**: 카테고리 테이블 로드/저장, after JSON 저장 등 **데이터 일관성에 영향 있는** 예외만, `traceback.print_exc()` 대신 한 줄 로그(예: `print(f"[WARN] category_table load failed: {e}", flush=True)`)로 남기기. 기존 `pass` 유지해도 됨.

---

## 6. lib 공개 함수 — 타입 힌트·docstring 보강 (선택)

**현상**: `lib/path_config.py`, `lib/data_json_io.py` 등 공개 함수에 반환 타입·Args/Returns docstring이 없음.

**제안**:  
- `get_data_dir() -> str`, `get_temp_dir() -> str`  
- `safe_read_data_json(path, default_empty=True) -> pd.DataFrame | None`  
등 **점진적으로** 추가. 새로 손대는 함수부터 적용해도 됨.

---

## 7. scripts 실행 — 오류 시 exit code (권장)

**현상**: `scripts/json_정리.py`, `move_*.py` 등에서 오류 발생 시 `sys.exit(1)` 없이 종료하면, 배치/CI에서 실패로 간주되지 않을 수 있음.

**제안**: 치명적 오류(파일 없음, 권한 오류로 인한 실패 등) 시 `sys.exit(1)` 호출. 정상 완료 시는 그대로 종료.
