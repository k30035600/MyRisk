# 전체화면 로직 전수 검사

## 1. 구성 요소

| 파일 | 역할 |
|------|------|
| **templates/_fullscreen_btn.html** | 홈에서만 사용. 버튼 클릭 시 `localStorage.myrisk_fullscreen='1'` 저장 후 `requestFullscreen()` 호출. FHD(1920×1080) 이하에서만 버튼 표시. |
| **templates/_fullscreen_persist.html** | 여러 페이지에 include. 페이지 로드 시 `myrisk_fullscreen==='1'`이면 150ms 후 자동으로 `requestFullscreen()` 시도. |

## 2. include 위치

- **전체화면 버튼**: `templates/index.html`(홈) 에만.
- **persist 스크립트**: 루트 templates, MyBank, MyCard, MyCash 의 index/category/analysis_basic/help, reset, 404, help, print_analysis, risk_report_page, opinion_fragment 등.

## 3. 발생했던 문제

- **증상**: 메인에서 은행거래 전처리로 넘어가면, 사용자가 전체화면을 선택하지 않았는데 전체화면이 됨.
- **원인**: 은행거래는 **iframe** 안에 로드됨. iframe 내부 문서(/bank/)에서도 `_fullscreen_persist.html`가 실행되고, 같은 localStorage를 보기 때문에 예전에 저장된 `myrisk_fullscreen='1'`이 있으면 150ms 후 자동으로 `requestFullscreen()`를 호출함.

## 4. 수정 사항

- **`_fullscreen_persist.html`** (루트·MyBank·MyCard·MyCash 공통): 스크립트 맨 앞에 **`if (window.self !== window.top) return;`** 추가.
- **효과**: iframe 안에서 열린 페이지에서는 자동 전체화면 복원을 시도하지 않음. **최상위 문서(홈 또는 탭에서 직접 연 페이지)**에서만 선호가 있으면 자동 복원 시도.

## 5. 동작 정리

- **홈(/)**: FHD 이하에서만 「⛶ 전체 화면」 버튼 표시. 클릭 시 전체화면 + localStorage에 선호 저장.
- **홈에서 은행/카드/금융 링크 클릭**: 같은 문서에서 iframe으로 로드. iframe 내부에서는 persist 미실행 → **자동 전체화면 없음**.
- **주소창에서 직접 /bank/ 등 접속**: 최상위 문서이므로 persist 실행. 이전에 선호가 있으면 자동 전체화면 시도(브라우저 정책에 따라 차단될 수 있음).
