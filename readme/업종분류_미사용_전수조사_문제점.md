# 업종분류·위험도 매칭 — category_table만 사용

## 1. 조사 개요

금융정보(MyCash)에서 **category_table.json만** 사용합니다. 업종분류·위험도는 **get_risk_class_table_data**, **get_risk_class_map_for_apply**, **export_risk_class_table_to_xlsx** 등으로 제공됩니다.

---

## 2. 핵심 의존성: `lib.category_table_io` 없음 시 영향

- **`get_risk_class_map_for_apply`**, **`get_risk_class_table_data`**, **`export_risk_class_table_to_xlsx`** 는 **모두 `lib.category_table_io`** 에서만 제공됩니다.
- **readme/backup_업종분류_to_xlsx.py**: `from lib.category_table_io import export_risk_class_table_to_xlsx` 사용. **lib 없으면 스크립트 실행 시점에 ImportError로 즉시 실패.** 폴백 없음.

---

## 3. API·템플릿

- **`/api/risk-class-table`**: 업종분류 조회 데이터는 **`get_risk_class_table_data()`**(category_table 기반)에서만 생성.
- 클라이언트는 `/api/risk-class-table` 호출로 업종분류 테이블 데이터를 사용합니다.

---

## 4. 백업 스크립트

| 스크립트 | 역할 | 출력 파일 |
|----------|------|-----------|
| **readme/backup_업종분류_to_xlsx.py** | category_table 기반 업종분류 데이터 → xlsx | data/업종분류_table.xlsx |

- xlsx는 **category_table 기반으로만** 생성.
