【명세서】

【발명의 명칭】

국문: 인공지능 기반의 금융거래 위험도 분석 및 법적 소명 자료 자동 생성 시스템 및 방법

영문: AI-based Financial Transaction Risk Analysis and Automated Legal Explanation Document Generation System and Method

【기술분야】

본 발명은 금융 데이터 분석 및 법률 서비스(Legal-Tech) 기술에 관한 것으로, 프로세서 및 메모리를 포함하는 컴퓨팅 장치에 의해 수행되며, 이기종 금융기관의 거래 내역 데이터를 통합 전처리하고, 법적 기준에 따른 위험도를 정량적으로 산정하여, 개인회생 및 파산 절차에 필요한 소명 자료를 자동 생성하는 시스템 및 방법에 관한 것이다.

【배경기술】

개인회생 및 파산 신청 건수가 증가함에 따라, 채무자는 방대한 금융 거래 내역에 대해 면책 불허가 사유(사치, 도박, 자산 은닉 등)가 없음을 법원 및 파산관재인에게 소명해야 한다. 그러나 기존 방식에는 다음과 같은 한계가 존재한다.

첫째, 은행 거래 내역과 신용카드 내역은 금융기관마다 데이터 포맷이 상이하여 통합 분석에 막대한 수기 작업이 소요된다.

둘째, 거래 적요나 가맹점명이 비정형 텍스트로 기록되어 단순 키워드 검색만으로는 법적 위험 거래를 정확히 분류하기 어렵다.

셋째, 분석 결과에 대해 법원 제출용 소명 자료 작성 시, 법률 지식 부족으로 방어권 행사가 제한된다.

넷째, 파산 신청 직전 집중 거래, 과다 빈도 거래, 투자금 유출 후 부분 회수 등의 패턴을 종합 분석하는 정량적 모델이 부재하다.

한편, 선행기술로서 인공지능 기반의 법률 문서 분석 시스템(KR102289935B1, 문헌1)은 NLP를 이용하여 법률 문서의 누락·위험 요소를 탐지하고, 법률문서 자동작성 시스템(KR102022412B1, 문헌2)은 Q&A 방식으로 법률 문서를 생성하나, 이들은 법률 문서 텍스트의 분석 및 작성에 한정되며 금융 거래 데이터를 직접 분석하지 않는다. 스크래핑을 이용한 회계 처리 시스템(KR102824993B1, 문헌3)은 금융기관 입출금 데이터를 수집하여 회계 분개를 자동화하나 법적 리스크 분석 기능이 없고, 푸시메시지 기반 가계부 관리 서비스(KR101580947B1, 문헌4)는 소비 패턴 관리에 집중하며 법적 소명 기능이 없다.

또한, AI를 이용한 재무 관리 클라우드 서비스(KR102486186B1, 문헌5)는 시계열 분석과 ARIMA/LSTM 등을 활용하여 기업의 자금 흐름을 예측하고, 빅데이터 및 기계학습을 이용한 자동 분개 서버(KR101914620B1, 문헌6)는 신뢰도 임계치를 통한 복수 계정과목 추천 기능을 제공하며, 손익계산서 계정과목 자동 분류 시스템(KR101892671B1, 문헌7)은 거래처-계정과목 1:1 매칭을 수행하나, 이들은 모두 회계·세무 또는 기업 재무 예측을 목적으로 하며, 채무자회생법에 기반한 법적 위험도의 정량적 산정과 소명서 자동 생성 기능은 결여되어 있다.

따라서 이기종 금융 데이터를 통합하고, 법적 기준에 따른 위험도를 정량적으로 산출하며, 시간적·금액적·빈도적 패턴을 복합적으로 반영하여, 법적 소명 문안을 자동 생성하는 지능형 시스템의 개발이 요구된다.

【발명의 내용】

【해결하려는 과제】

상기 선행기술들은 각각 법률 문서 텍스트 분석(문헌1), 양식 기반 법률 문서 작성(문헌2), 회계 분개 자동화(문헌3), 소비 패턴 분석(문헌4), AI 기반 재무 관리·자금 예측(문헌5), 빅데이터·ML 기반 자동 분개(문헌6), 거래처-계정과목 매칭(문헌7)에 집중하고 있으나, 채무자 회생 및 파산에 관한 법률 제564조에서 규정하는 면책 불허가 사유를 정량적으로 산정하는 위험도 분석 엔진을 갖추고 있지 않으며, 금융 거래 내역에서 직접 법적 소명 자료를 자동 생성하는 기능도 결여되어 있다.

특히, 문헌5의 시계열 분석(ARIMA/LSTM)은 기업 자금 흐름의 미래 예측을 목적으로 하는 반면, 본 발명의 시계열 근접성 계수(T)는 파산 신청 직전의 의심 거래 패턴을 탐지하는 것을 목적으로 하여 기술적 사상이 상이하다. 또한, 문헌6의 신뢰도 임계치는 분류 결과의 확실성을 평가하는 반면, 본 발명의 동적 임계 계수(Θ)는 금액 규모와 거래 빈도의 복합 가중을 통해 법적 위험도의 크기를 보정하는 것으로 목적과 산출 방식이 근본적으로 다르다.

본 발명은 이기종 금융 데이터의 통합 전처리, 3단계 계층적 분류, 법적 근거 기반의 단계적 위험도 산정, 시계열·금액·빈도 패턴의 복합 위험도 보정, 및 소명 자료 자동 생성을 일관된 파이프라인으로 수행하는 시스템 및 방법을 제공하는 것을 목적으로 한다.

【과제의 해결수단】

본 발명은 프로세서 및 메모리를 포함하는 컴퓨팅 장치에 의해 수행되는 금융거래 위험도 분석 방법을 제공한다.

첫째, 데이터 전처리부는 복수의 금융기관으로부터 서로 다른 포맷으로 발행된 은행 거래 내역 데이터 및 신용카드 승인 내역 데이터를 수신하여, 금융기관별 전용 파서를 통해 단일한 통합 데이터 구조로 정규화한다.

둘째, 3단계 계층적 분류 엔진을 통해 분류 정확도를 극대화한다. 전처리→계정과목 분류(최장 일치 방식)→후처리. 복수 키워드 동시 매칭 시 5단계 충돌 해결 캐스케이드 적용.

셋째, 단계적 위험도 상향(Step-up Scoring) 알고리즘. 제1호(0.1)~제10호(5.0) 순차 판정, 상위 호수 조건 충족 시 갱신.

R = w × log10(1 + f × avg_a) × T × Θ × S

넷째, 위 공식으로 정량적 위험도 종합 점수를 산출한다. T는 시계열 근접성(집중도·최근성), Θ는 동적 임계(금액·건수 복합), S는 입출금 대칭성(회수 비율).

다섯째, 소명서 생성부는 지표별 법적 소명 전략 템플릿과 채무자회생법 조항 참조를 조합하여 소명서 초안을 자동 생성한다.

【발명의 효과】

본 발명에 따른 금융거래 위험도 분석 및 법적 소명 자료 자동 생성 시스템 및 방법에 의하면 다음과 같은 효과를 기대할 수 있다.

1. 정밀도·효율성 극대화: 이기종 금융기관의 파편화된 데이터를 단일 구조로 통합하고, 3단계 계층적 분류 엔진과 최장 일치 방식의 알고리즘, 5단계 충돌 해결 캐스케이드를 통해 비정형 거래 텍스트를 높은 정확도로 분류함으로써, 수기 분석 시 발생하는 데이터 누락 및 오분류를 원천적으로 차단한다.

2. 정량적 위험도 산출: 채무자회생법 제564조의 면책 불허가 사유를 수치화한 1호 내지 10호의 지표를 단계적 위험도 상향(Step-up Scoring) 방식으로 적용하고, 시계열 근접성 계수(T), 동적 임계 계수(Θ), 입출금 대칭성 계수(S)의 3차원 복합 보정과 로그 스케일 가중치 모델을 적용하여, 주관적 판단을 배제한 정량적 리스크 평가와 법적 소명 우선순위의 명확한 제시가 가능하다.

3. 방어권 강화 및 소명 절차 간소화: 위험 거래에 대한 법적 소명 전략 템플릿과 채무자회생법 조항 참조를 자동 매칭하여 소명서 초안을 생성함으로써, 법률 지식이 부족한 채무자의 방어권을 실질적으로 보장한다. 위험 거래와 소명 문안, 법적 근거를 유기적으로 결합한 보고서를 자동 생성함으로써 파산관재인의 심사 시간을 단축하고 사법 행정 비용을 절감한다.

4. 산업상 이용 가능성: 개인 파산뿐만 아니라 회생, 이혼 재산 분할, 기업 감사 등 금융 내역 소명이 필요한 모든 법률 분야로 즉시 확장 가능하며, 법률 대리인(변호사, 법무사) 사무소의 필수 업무 자동화 솔루션이 될 수 있다.

【도면의 간단한 설명】

도 1: 시스템 블록도 (100~400 계층). 데이터 입력 계층(100), 처리 엔진 계층(200), 데이터 저장 계층(300), 출력 보고서 계층(400)의 유기적 결합 관계를 도시한 시스템 블록도.

도 2: 전체 파이프라인 흐름도. 이기종 금융 데이터(은행·신용카드)의 전처리, 분류, 통합, 위험도 산정, 소명서 생성까지의 전체 파이프라인을 도시한 데이터 흐름도.

도 3: 3단계 카테고리 분류 상세도. 전처리→계정과목 분류→후처리의 3단계 분류 엔진 상세 동작. 최장 일치 방식과 5단계 충돌 해결 캐스케이드 적용 과정 포함.

도 4: 1~10호 위험도 산정 순서도 (VASP 포함). 채무자회생법 제564조 기반 Step-up Scoring 실행 순서도. 제1호(0.1)~제10호(5.0) 순차 판정 경로와 제7호 VASP 3단계 식별 특수 경로 포함.

도 5: R 점수·등급·소명서 생성 프로세스 맵. R 점수 산출(T·Θ·S 복합 보정), 등급 판정, 소명 전략 매칭, 법적 근거 매칭, 보고서 합성을 포함하는 소명서 자동 생성 프로세스.

도 6: 모듈 구성도. 메인 웹 서버, 은행 모듈, 카드 모듈, 금융정보 모듈, 공통 라이브러리의 모듈 구성 관계도.

도 7: 웹 UI 예시도. 분석 결과가 출력되는 웹 UI 예시. 위험도별 색상 구분, 지표별 통계, 소명서 초안 표시 포함.

4.5 R 점수 산출 (도 5)

R = w × log10(1 + f × avg_a) × T × Θ × S

T: T = 1 + 0.3×C + 0.2×P (C=집중도, P=최근성, 범위 1.00~1.50)
Θ: Θ = amount_factor × count_factor (범위 0.00~2.60)
S: S = 1−(입금/출금), 하한 0.3 (범위 0.30~1.00)
등급: 합산 R<50 안전, <100 보통, <150 주의, ≥150 심각.

4.6 선행기술 대비 차별적 기술 구성

T 계수 vs 문헌5(KR102486186B1): 문헌5는 ARIMA/LSTM으로 미래 예측. 본 발명의 T 계수는 과거 거래의 시간적 집중도(C)와 최근성(P)을 산출하여 파산 직전 의심 패턴을 탐지. 예측이 아닌 탐지·진단 목적이며 수식(T=1+αC+βP)도 상이.

Θ 계수 vs 문헌6(KR101914620B1): 문헌6의 신뢰도 임계치는 ML 분류 확실성 평가. 본 발명의 Θ는 금액 누진 가중(amount_factor)과 건수 비례 가중(count_factor)의 곱으로 법적 위험도 크기 보정. 목적과 산출 방식이 근본적으로 상이.

5단계 캐스케이드 vs 문헌7(KR101892671B1): 문헌7은 1:1 매칭 + 주계정과목등록부(정적). 본 발명은 (i) 길이, (ii) 위험도 코드, (iii) 위치, (iv) 소분류, (v) 기존 유지의 5단계 동적 해결.

4.7 소명서 자동 생성 (도 5)

단계 1. 위험거래 집계(≥0.5) → 단계 2. 상위 5개 선정 → 단계 3. SUGGESTION_TEMPLATES 10종 → 단계 4. LEGAL_REFERENCES 매칭 → 단계 5. 보고서 합성.

【특허청구범위】

【청구항 1】 (독립항 — 방법)

프로세서 및 메모리를 포함하는 컴퓨팅 장치에 의해 수행되는 금융거래 위험도 분석 및 법적 소명 자료 자동 생성 방법으로서,

(a) 복수의 금융기관으로부터 서로 다른 포맷으로 발행된 은행 거래 내역 데이터 및 신용카드 승인 내역 데이터를 수신하여, 금융기관별 전용 파서를 통해 단일한 통합 데이터 구조로 정규화하는 데이터 전처리 단계;

(b) 사전 정의된 카테고리 테이블을 참조하여, 상기 통합 데이터의 적요 또는 가맹점 명칭에 포함된 비정형 텍스트를 (i) 전처리, (ii) 계정과목 분류, (iii) 후처리의 3단계를 거쳐 자동 분류하되, 상기 계정과목 분류 시 검색 키워드의 길이 역순으로 매칭을 시도하는 최장 일치(Longest Match) 방식을 적용하는 3단계 계층적 카테고리 분류 단계;

(c) 분류 완료된 은행 거래 데이터와 신용카드 거래 데이터를 단일 통합 데이터로 병합하는 이기종 데이터 통합 단계;

(d) 상기 통합 데이터의 각 거래에 대하여, 채무자 회생 및 파산에 관한 법률 제564조에 대응하는 제1호(분류제외) 내지 제10호(사행성) 위험도 지표를 순차 적용하되, 상위 호수 조건 충족 시 기 산정된 위험도를 갱신하는 단계적 위험도 상향(Step-up Scoring) 방식의 위험도 산정 단계; 및

(e) 위험도 가중치(w), 거래 건수(f), 평균 거래 금액(avg_a), 시계열 근접성 계수(T), 동적 임계 계수(Θ), 입출금 대칭성 계수(S)를 사용하여 R = w × log₁₀(1 + f × avg_a) × T × Θ × S 공식으로 위험도 종합 점수(R)를 산출하고, 지표별 법적 소명 전략 템플릿과 법조항 참조를 매칭하여 소명서 초안을 자동 생성하는 소명 자료 생성 단계;

를 포함하는 것을 특징으로 하는 금융거래 위험도 분석 및 법적 소명 자료 자동 생성 방법.

【청구항 2】 (5단계 충돌 해결)

제1항에 있어서, 계정과목 분류 시 복수 키워드 동시 매칭 시 (i) 키워드 길이 우선, (ii) 위험도 코드 우선, (iii) 텍스트 내 출현 위치 우선, (iv) 소분류 코드 순, (v) 기존 분류 유지의 5단계 충돌 해결 캐스케이드를 적용하는 것을 특징으로 하는 방법.

【청구항 3】 (위험도 산정 상세)

제1항에 있어서, 제1호(0.1) 기본값 부여 후 제2호(0.5)~제10호(5.0) 순차 판정, 제2호(심야·폐업) 해당 시 3~10호 생략, 제5호~제10호는 카테고리 테이블 위험도분류 키워드+기준 출금액 결합 조건으로 판정하는 것을 특징으로 하는 방법.

【청구항 4】 (R 점수 및 T 계수)

제1항에 있어서, T = 1 + α × C + β × P (C: 활동기간 대비 집중도, P: 최근 25% 구간 건수 비율)로 산출하여 단기간 집중·파산 직전 의심 패턴을 반영하는 것을 특징으로 하는 방법.

【청구항 5】 (소명서: 정형 + AI)

제1항에 있어서, (a) 정형 생성(템플릿 치환) 또는 (b) 생성형 AI 합성 중 선택 수행, AI 불가 시 정형으로 자동 전환하는 것을 특징으로 하는 방법.

【청구항 6】 (VASP 3단계 식별)

제1항에 있어서, 가상자산 거래 식별은 (a) 카테고리 매칭 → (b) 접두번호 정규표현식 → (c) 사업자명 매칭 순서로 수행, 접두번호 테이블은 카테고리 테이블의 VASP계좌 분류 행으로 동적 확장 가능한 것을 특징으로 하는 방법.

【청구항 7】 (동적 임계 계수 Θ)

제4항에 있어서, Θ = amount_factor × count_factor. 금액 임계값 미만 시 선형 감점, 3배 초과 시 누진 가중(γ, 상한 2.0), 건수 기준 이상 시 비례 가중(δ, 상한 1.3)을 적용하는 것을 특징으로 하는 방법.

【청구항 8】 (인터랙티브 Q&A)

제1항에 있어서, 위험 항목 기반 맞춤 질문 생성 → 사용자 답변 수집 → 소명서 재생성. 미입력 항목은 정형 생성 문구 유지하는 것을 특징으로 하는 방법.

【청구항 9】 (입출금 대칭성 S)

제4항에 있어서, S = 1 − (입금합계/출금합계), 하한 0.3. 투기성(5호)·가상자산(7호)·자산은닉(8호)에만 적용, 비적용 지표는 S=1.0으로 하는 것을 특징으로 하는 방법.

【청구항 10】 (외부 API 검증)

제4항에 있어서, 사업자등록번호를 외부 API에 조회하여 업종·폐업 여부 확인. 개인정보보호법 등 법령 허용 범위 내 수행하는 것을 특징으로 하는 방법.

【청구항 11】 (독립항 — 시스템)

프로세서; 메모리; 및 상기 메모리에 저장되어 상기 프로세서에 의해 실행되는 프로그램을 포함하는 금융거래 위험도 분석 및 법적 소명 자료 자동 생성 시스템으로서,

상기 프로그램은,

복수의 금융기관으로부터 수신한 이기종 거래 내역 데이터를, 금융기관별 전용 파서를 통해 날짜 포맷, 금액 표기, 컬럼 구조를 단일한 통합 데이터 구조로 정규화하는 데이터 전처리부;

사전 정의된 카테고리 테이블을 참조하여, 상기 통합 데이터의 적요 또는 가맹점 명칭에 포함된 비정형 텍스트를 전처리, 최장 일치(Longest Match) 방식의 계정과목 분류, 및 후처리의 3단계를 거쳐 자동 분류하되, 복수 키워드 동시 매칭 시 키워드 길이, 위험도 코드, 텍스트 내 출현 위치, 소분류 코드, 기존 분류 유지의 5단계 충돌 해결 캐스케이드를 적용하는 분류 엔진부;

분류 완료된 은행 거래 데이터와 신용카드 거래 데이터를 단일 통합 데이터로 병합하는 데이터 통합부;

상기 통합 데이터의 각 거래에 대하여 채무자 회생 및 파산에 관한 법률 제564조에 대응하는 제1호(분류제외, 위험도 0.1) 내지 제10호(사행성, 위험도 5.0)의 위험도 지표를 순차적으로 적용하되, 상위 호수의 조건이 충족될 경우 기 산정된 위험도를 해당 상위 호수에 대응하는 가중 점수로 갱신하는 단계적 위험도 상향(Step-up Scoring) 방식에 따라 각 거래의 최종 위험도를 결정하는 위험도 산정부; 및

상기 결정된 최종 위험도에 기초하여, 위험도 가중치(w), 거래 건수(f), 평균 거래 금액(avg_a), 시계열 근접성 계수(T), 동적 임계 계수(Θ), 입출금 대칭성 계수(S)를 사용한 R = w × log₁₀(1 + f × avg_a) × T × Θ × S 공식으로 위험도 종합 점수를 산출하고, 지표별 법적 소명 전략 템플릿과 채무자회생법 조항 참조 정보를 매칭하여 소명서 초안을 자동 생성하는 소명서 생성부;

를 포함하는 것을 특징으로 하는 금융거래 위험도 분석 및 법적 소명 자료 자동 생성 시스템.
