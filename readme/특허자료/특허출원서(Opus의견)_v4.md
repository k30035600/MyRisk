### 1.3 실제 코드 분석 결과 요약

#### 구현 확인됨 (명세서 청구항에 기재)

| 기술 요소 | 구현 위치 | 상세 |
|-----------|-----------|------|
| 1~10호 Step-up Scoring | MyCash/risk_indicators.py | 1호 기본(0.1) → 10호(5.0) 순차 덮어쓰기 |
| 최장 일치(Longest Match) | process_bank_data.py, process_card_data.py | 5단계 충돌 해결 캐스케이드 포함 |
| 3단계 분류 엔진 | category_table_io.py + 각 process 모듈 | 전처리 → 계정과목 → 후처리 |
| R 점수 산출 | cash_app.py _calculate_risk_report | R = w × log₁₀(1+f×avg_a) × T × Θ × S |
| 시계열 근접성 계수(T) | cash_app.py _compute_temporal_proximity | T = 1 + α×C + β×P |
| 동적 임계 계수(Θ) | cash_app.py _compute_threshold_factor | Θ = amount_factor × count_factor |
| 입출금 대칭성 계수(S) | cash_app.py _compute_symmetry_factor | S = 1−(입금/출금), 하한 0.3 |
| 채무자회생법 매핑 | cash_app.py LEGAL_REFERENCES | 3호~10호 각 해당 항 |
| 소명 템플릿 초안 | cash_app.py SUGGESTION_TEMPLATES | 지표별 10종 |
| VASP 3단계 식별 | risk_indicators.py detect_vasp | (a) 카테고리 → (b) 접두번호 → (c) 사업자명 |
| 5단계 매칭 충돌 해결 | process_bank_data.py _best_match | 길이→위험도→위치→소분류→기존 유지 |
| 이기종 데이터 통합 | cash_app.py merge_bank_card_to_cash_after | 은행+카드 → cash_after |
| 원자적 파일 쓰기 | category_table_io.py, data_json_io.py | tempfile.mkstemp + os.replace |
| 스레드 안전 캐시 | category_table_io.py _ct_lock | threading.Lock |
| 경로 탐색 방어 | app.py serve_img, api_restore | os.path.realpath 검증 |
