## 제2장 특허 명세서 초안

### 【발명의 명칭】

국문: 인공지능 기반의 금융거래 위험도 분석 및 법적 소명 자료 자동 생성 시스템 및 방법

영문: AI-based Financial Transaction Risk Analysis and Automated Legal Explanation Document Generation System and Method

### 【기술분야】

본 발명은 금융 데이터 분석 및 법률 서비스(Legal-Tech) 기술에 관한 것으로, 상세하게는 프로세서 및 메모리를 포함하는 컴퓨팅 장치에 의해 수행되며, 이기종 금융기관의 거래 내역 데이터를 통합 전처리하고, 법적 기준에 따른 위험도를 정량적으로 산정하여, 개인회생 및 파산 절차에 필요한 소명 자료를 자동 생성하는 시스템 및 방법에 관한 것이다.

### 【배경기술】

개인회생 및 파산 신청 건수가 증가함에 따라, 채무자는 방대한 금융 거래 내역에 대해 면책 불허가 사유(사치, 도박, 자산 은닉 등)가 없음을 법원 및 파산관재인에게 소명해야 한다. 그러나 기존 방식에는 다음과 같은 한계가 존재한다.

첫째, 은행 거래 내역과 신용카드 내역은 금융기관마다 데이터 포맷이 상이하여 이를 통합적으로 분석하기 위해서는 막대한 수기 작업과 시간이 소요된다. 국민은행, 신한은행, 하나은행 등 각 금융기관은 엑셀 출력 형식이 서로 달라, 날짜 포맷, 컬럼 구조, 금액 표기 방식 등을 개별적으로 파악하여 수동 정리해야 한다.

둘째, 거래 적요나 가맹점명이 비정형 텍스트로 기록되어 있어, 단순 키워드 검색만으로는 실질적인 법적 위험 거래를 정확히 분류해내기 어렵다. 예를 들어 '스타벅스'와 '스타벅스커피코리아'가 동시에 존재하거나, 간편결제 서비스(예: '네이버페이(배달의민족)')로 실제 가맹점이 은폐된 경우 분류의 정확도가 크게 떨어진다.

셋째, 분석된 결과에 대해 채무자가 법원에 제출할 소명 자료를 작성할 때, 어떤 거래가 면책 불허가 사유에 해당하는지 판단하기 어렵고, 적절한 법률 용어를 선택하는 데 어려움을 겪어 방어권 행사가 제한되는 문제가 있다.

넷째, 파산 신청 직전에 집중된 거래, 금액 규모에 비해 과다한 빈도의 거래, 또는 투자금 유출 후 부분 회수된 거래 등의 패턴을 종합적으로 분석하여 위험도에 반영하는 정량적 모델이 부재하다.

따라서 이기종 금융 데이터를 통합하고, 법적 기준에 따른 위험도를 정량적으로 산출하며, 시간적·금액적·빈도적 패턴을 복합적으로 반영하여, 이를 바탕으로 법적 소명 문안을 자동 생성하는 지능형 시스템의 개발이 요구된다.

[문헌6] 빅데이터 및 기계학습을 이용한 자동 분개 수행 방법 및 이를 실현하는 서버 (등록번호: KR101914620B1, 출원인: (주)아이퀘스트)

[문헌7] 손익계산서에 작성되는 비용의 계정과목을 자동으로 분류하는 계정과목 분류시스템 (등록번호: KR101892671B1)

### 【해결하려는 과제】

상기 선행기술들은 각각 법률 문서 텍스트 분석(문헌1), 양식 기반 법률 문서 작성(문헌2), 회계 분개 자동화(문헌3), 소비 패턴 분석(문헌4), AI 시계열 기반 기업 재무 관리(문헌5), ML 빅데이터 기반 회계 분개(문헌6), 거래처명 기반 계정과목 분류(문헌7)에 집중하고 있으나, 채무자 회생 및 파산에 관한 법률 제564조에서 규정하는 면책 불허가 사유를 **정량적으로 산정**하는 위험도 분석 엔진을 갖추고 있지 않으며, 금융 거래 내역에서 직접 법적 소명 자료를 자동 생성하는 기능도 결여되어 있다.

특히, 문헌5의 시계열 분석은 거래 원인과 결과의 시간적 연계(최소 비용값 매칭)에 초점을 맞추고 있어 위험 거래의 시간적 집중도·최근성을 반영하는 보정 모델과는 기술적 목적이 상이하고, 문헌6의 ML 기반 신뢰도 임계치는 분류 결과의 확신도 척도이지 거래 금액·빈도의 복합 가중 보정 계수와는 성격이 다르며, 문헌7의 거래처-계정과목 1:1 수동 매핑은 3단계 계층적 자동 분류와 구조적으로 상이하다.

본 발명은 상기 종래 기술의 문제점을 해결하기 위해, 이기종 금융 데이터의 통합 전처리, 3단계 계층적 분류, 법적 근거 기반의 단계적 위험도 산정, 시계열·금액·빈도 패턴의 복합 위험도 보정, 및 소명 자료 자동 생성을 일관된 파이프라인으로 수행하는 시스템 및 방법을 제공하는 것을 목적으로 한다.

### 【과제의 해결 수단】

본 발명은 상기한 문제점을 해결하기 위해, 프로세서 및 메모리를 포함하는 컴퓨팅 장치에 의해 수행되는 금융거래 위험도 분석 방법을 제공하며, 그 해결 수단은 다음과 같다.

**첫째**, 데이터 전처리부는 복수의 금융기관으로부터 서로 다른 포맷으로 발행된 은행 거래 내역 데이터 및 신용카드 승인 내역 데이터를 수신하여, 금융기관별 전용 파서(Parser)를 통해 날짜 포맷, 금액 표기, 컬럼 구조를 단일한 통합 데이터 구조로 정규화한다.

**둘째**, 3단계 계층적 분류 엔진을 통해 데이터의 분류 정확도를 극대화한다. 전처리 단계에서 비정형 데이터를 정규화하고, 계정과목 분류 단계에서 사전 정의된 카테고리 테이블의 키워드를 문자열 길이 역순으로 정렬하여 '최장 일치(Longest Match) 방식'의 알고리즘을 적용함으로써 오분류를 최소화하며, 후처리를 통해 법적 용어로 재정규화한다. 특히 복수의 키워드가 동시에 매칭될 경우 (i) 키워드 길이 우선, (ii) 위험도 코드 우선, (iii) 텍스트 내 출현 위치 우선, (iv) 소분류 코드 순, (v) 기존 분류 유지의 **5단계 충돌 해결 캐스케이드**를 적용하여 분류의 일관성을 확보한다. 이는 문헌7의 거래처-계정과목 1:1 수동 매핑과 달리 자동화된 다단계 분류 체계를 구성한다.

**셋째**, 단계적 위험도 상향(Step-up Scoring) 알고리즘을 적용한다. 채무자 회생 및 파산에 관한 법률 제564조의 각 면책 불허가 사유에 대응하는 제1호(분류제외, 위험도 0.1) 내지 제10호(사행성, 위험도 5.0)의 위험도 지표를 순차적으로 판정하되, 상위 호수의 조건이 충족될 경우 해당 거래의 위험도를 상위 호수에 대응하는 가중 점수로 갱신(Overwriting)하여 최종 위험 등급을 결정한다.

**넷째**, 은행 거래와 신용카드 거래를 단일 통합 데이터로 병합한 후, 위험도가 소정 임계값 이상인 거래를 필터링하고, 지표별로 그룹핑하여 건수 및 금액을 합산한다. 이때 하기 수식에 기초하여 정량적 위험도 종합 점수(R)를 산출한다:

$$R = w \times \log_{10}(1 + f \times \overline{a}) \times T \times \Theta \times S$$

여기서 w는 해당 지표의 법적 민감도 가중치(0.1~5.0), f는 거래 건수, ā는 평균 거래 금액이며, T는 시계열 근접성 계수(거래의 시간적 집중도와 최근성을 반영, 1.00~1.50), Θ는 동적 임계 계수(금액 규모와 거래 빈도의 복합 가중, 0.00~2.60), S는 입출금 대칭성 계수(입금 회수 비율에 따른 하향 보정, 0.30~1.00)이다. 상기 T는 문헌5의 원인-결과 시간적 연계와 달리 위험 거래의 시간적 집중도(C)와 최근성(P)을 반영하는 보정 승수이며, 상기 Θ는 문헌6의 분류 신뢰도와 달리 거래 금액 규모와 빈도의 복합 가중 보정 계수이다.

**다섯째**, 소명서 생성부는 산출된 위험도 점수를 바탕으로, 지표별 사전 정의된 법적 소명 전략 템플릿과 법적 근거(채무자회생법 제564조 해당 항) 참조 정보를 조합하여, 각 위험 거래군에 대한 소명서 초안을 자동 생성한다.

### 【발명의 효과】

본 발명에 따른 금융거래 위험도 분석 및 법적 소명 자료 자동 생성 시스템 및 방법에 의하면 다음과 같은 효과를 기대할 수 있다.

**1. 금융 데이터 분석의 정밀도 및 효율성 극대화**

이기종 금융기관의 파편화된 데이터를 단일 데이터 구조로 통합함으로써, 수기 분석 시 발생하는 데이터 누락 및 오류를 원천적으로 차단한다. 3단계 계층적 분류 엔진과 최장 일치 방식의 알고리즘, 그리고 5단계 충돌 해결 캐스케이드를 통해 비정형 거래 텍스트를 높은 정확도로 분류한다.

**2. 법적 기준에 부합하는 정량적 위험도 산출**

채무자회생법 제564조의 면책 불허가 사유를 수치화한 1호 내지 10호의 지표를 단계적 위험도 상향(Step-up Scoring) 방식으로 적용함으로써, 주관적 판단을 배제한 정량적 리스크 평가가 가능하다. 시계열 근접성 계수(T), 동적 임계 계수(Θ), 입출금 대칭성 계수(S)의 복합 보정 모델을 통해 시간·금액·빈도·회수 패턴을 입체적으로 반영한다.

**3. 채무자의 사법 방어권 강화 및 소명 절차의 간소화**

위험 거래에 대한 법적 소명 전략 템플릿과 채무자회생법 조항 참조를 자동 매칭하여 소명서 초안을 생성함으로써, 법률 지식이 부족한 채무자의 방어권을 실질적으로 보장한다.

**4. 산업상 이용 가능성**

개인 파산뿐만 아니라 회생, 이혼 재산 분할, 기업 감사 등 금융 내역 소명이 필요한 모든 법률 분야로 즉시 확장 가능한 기술적 토대를 가지고 있다.

## 제3장 특허청구범위

### 【청구항 1】 (독립항 — 방법 발명)

프로세서 및 메모리를 포함하는 컴퓨팅 장치에 의해 수행되는 금융거래 위험도 분석 및 법적 소명 자료 자동 생성 방법으로서,

(a) 복수의 금융기관으로부터 서로 다른 포맷으로 발행된 은행 거래 내역 데이터 및 신용카드 승인 내역 데이터를 수신하여, 금융기관별 전용 파서를 통해 단일한 통합 데이터 구조로 정규화하는 데이터 전처리 단계;

(b) 사전 정의된 카테고리 테이블을 참조하여, 상기 통합 데이터의 적요 또는 가맹점 명칭에 포함된 비정형 텍스트를 (i) 전처리, (ii) 계정과목 분류, (iii) 후처리의 3단계를 거쳐 자동 분류하되, 상기 계정과목 분류 시 검색 키워드의 길이 역순으로 매칭을 시도하는 최장 일치(Longest Match) 방식을 적용하여 분류 정확도를 확보하는 3단계 계층적 카테고리 분류 단계;

(c) 상기 분류가 완료된 은행 거래 데이터와 신용카드 거래 데이터를 단일 통합 데이터로 병합하는 이기종 데이터 통합 단계;

(d) 상기 통합 데이터의 각 거래에 대하여, 채무자 회생 및 파산에 관한 법률 제564조의 면책 불허가 사유에 대응하도록 설계된 제1호(분류제외) 내지 제10호(사행성)의 위험도 지표를 순차적으로 적용하되, 하위 호수에서 상위 호수로 갈수록 위험도가 가중되며 상위 호수의 조건이 충족될 경우 기 산정된 위험도를 해당 상위 호수에 대응하는 가중 점수로 갱신하는 단계적 위험도 상향(Step-up Scoring) 방식에 따라 각 거래의 최종 위험도를 결정하는 법적 근거 기반의 위험도 산정 단계; 및

(e) 상기 결정된 최종 위험도가 소정 임계값 이상인 위험 거래를 지표별로 그룹핑하고, 위험도 가중치(w), 거래 건수(f), 평균 거래 금액(avg_a), 시계열 근접성 계수(T), 동적 임계 계수(Θ), 입출금 대칭성 계수(S)를 사용하여 R = w × log₁₀(1 + f × avg_a) × T × Θ × S 공식으로 정량적 위험도 종합 점수(R)를 산출하고, 각 지표에 대응하는 법적 소명 전략 템플릿 및 채무자회생법 조항 참조 정보를 매칭하여 소명서 초안을 자동 생성하는 소명 자료 생성 단계;

를 포함하는 것을 특징으로 하는 금융거래 위험도 분석 및 법적 소명 자료 자동 생성 방법.

### 【청구항 2】 (종속항 — 5단계 충돌 해결 캐스케이드)

제1항에 있어서,

상기 3단계 계층적 카테고리 분류 단계의 계정과목 분류 시, 복수의 키워드가 동일 거래 텍스트에 동시 매칭되는 경우, (i) 키워드 문자열 길이가 긴 것을 우선하고, (ii) 동일 길이이면 위험도 코드가 높은 것을 우선하며, (iii) 위험도 코드도 동일하면 거래 텍스트 내 출현 위치가 앞선 것을 우선하고, (iv) 출현 위치도 동일하면 소분류 코드 순으로 우선하며, (v) 모든 조건이 동일하면 기존 분류를 유지하는 5단계 충돌 해결 캐스케이드를 적용하는 것을 특징으로 하는 방법.

### 【청구항 3】 (종속항 — 위험도 산정 로직 상세)

제1항에 있어서,

상기 법적 근거 기반의 위험도 산정 단계는, 각 거래에 대하여 제1호(분류제외, 위험도 0.1)를 기본값으로 부여한 후, 제2호(심야폐업, 위험도 0.5) 내지 제10호(사행성, 위험도 5.0)의 조건을 순차적으로 판정하되, 상기 제2호는 해당 거래처의 폐업 여부 또는 심야 시간대 거래와 소정 기준 출금액의 결합 조건에 의해 판정되고, 제2호 해당 거래는 제3호~제10호 판정을 생략하며, 상기 제5호 내지 제10호는 상기 카테고리 테이블에 정의된 위험도분류 키워드와의 매칭 및 소정 기준 출금액의 결합 조건에 의해 판정되며, 상위 호수의 조건이 충족될 경우 기 설정된 하위 호수의 점수를 상기 상위 호수에 대응하는 가중 점수로 갱신(Overwriting)하여 최종 위험 등급을 결정하는 것을 특징으로 하는 방법.

### 【청구항 4】 (종속항 — R 점수 산출 및 시계열 근접성 계수 T)

제1항에 있어서,

상기 소명 자료 생성 단계는 위험도 지표별 가중치(w), 거래 건수(f), 평균 거래 금액(avg_a), 시계열 근접성 계수(T), 동적 임계 계수(Θ), 입출금 대칭성 계수(S)를 사용하여 R = w × log₁₀(1 + f × avg_a) × T × Θ × S 공식으로 R 점수를 산출하되,

T = 1 + α × C + β × P
(C: 해당 지표 거래의 활동 기간 대비 전체 분석 기간의 집중도, P: 전체 분석 기간의 최근 25% 구간에 해당하는 거래 건수 비율, α·β: 가중 상수)

로 산출하여 단기간 집중 거래 및 파산 직전 의심 거래 패턴을 정량적으로 반영하고, 지표별 사전 정의된 법적 소명 템플릿을 적용하여 법조항 참조를 포함한 소명서 초안을 자동 생성하는 것을 특징으로 하는 방법.

### 【청구항 5】 (종속항 — 소명서 생성 방식: 정형 + AI 선택)

제1항에 있어서,

상기 소명서 생성 단계는
(a) 사전 정의된 지표별 소명 템플릿에 거래 건수·금액을 치환하여 정형 소명 문안을 생성하는 정형 생성 방식, 또는
(b) 상기 구조화된 위험도 분석 결과(지표명, R 점수, T 계수, 건수, 금액)와 법적 참조 정보(LEGAL_REFERENCES)를 자연어 생성 모델에 입력하여 사건별 맞춤 소명 문안을 합성하는 생성형 합성 방식
중 하나 이상을 선택적으로 수행하되, 생성형 합성 방식이 불가한 경우 정형 생성 방식으로 자동 전환하는 것을 특징으로 하는 방법.

### 【청구항 6】 (종속항 — VASP 3단계 식별)

제1항에 있어서,

상기 위험도 산정 단계에서 가상자산 관련 거래의 식별은
(a) 분류 단계에서 부여된 카테고리(가상자산)에 의한 매칭,
(b) 거래 내역의 비정형 텍스트(적요, 내용, 송금메모)에서 정규표현식으로 VASP(가상자산사업자) 가상계좌번호의 접두번호 대역을 추출하여 사전 정의된 접두번호 테이블과 대조하는 계좌번호 식별 방식, 또는
(c) 상기 비정형 텍스트에서 등록 VASP 사업자명·서비스명을 탐색하는 사업자명 매칭 방식
중 하나 이상을 단계적으로 수행하되, (a) → (b) → (c) 순서로 우선 적용하고, 상기 접두번호 테이블은 카테고리 테이블의 VASP계좌 분류 행으로 동적 확장이 가능한 것을 특징으로 하는 방법.

### 【청구항 7】 (종속항 — 동적 임계 계수 Θ)

제4항에 있어서,

상기 R 점수 산출 단계는 동적 임계 계수(Θ)를 추가 승수로 적용하되, Θ = amount_factor × count_factor 로 산출하며,

(a) amount_factor는 해당 지표의 거래 총액이 사전 정의된 임계값 미만인 경우 선형 감점(총액/임계값), 임계값의 3배 이내인 경우 1.0, 3배를 초과하는 경우 누진 가중(1 + γ × log₂(총액/임계값), γ: 가중 상수, 상한 2.0)을 적용하고,

(b) count_factor는 해당 지표의 거래 건수가 기준 건수 이상인 경우 건수 비례 가중(1 + δ × (건수/기준건수 − 1), δ: 가중 상수, 상한 1.3)을 적용하여,

금액 규모와 거래 빈도를 복합적으로 반영한 위험도 보정을 수행하는 것을 특징으로 하는 방법.

### 【청구항 8】 (종속항 — 인터랙티브 질의응답)

제1항에 있어서,

상기 소명서 생성 단계는
(a) 위험도 분석 결과에서 추출한 상위 위험 항목의 지표명, 거래 건수, 금액, 법조항을 기반으로 지표별 사전 정의된 질의 템플릿에서 맞춤형 질문을 생성하는 단계,
(b) 사용자 인터페이스를 통해 상기 질문에 대한 사용자의 답변을 수집하는 단계, 및
(c) 수집된 답변을 소명서 초안 템플릿의 가변 구간에 치환하여 사용자 실제 답변이 반영된 소명서 초안을 재생성하는 단계
를 선택적으로 추가 수행하되, 답변이 미입력된 항목에 대해서는 정형 생성 방식(제5항 (a))의 자동 생성 문구를 기본값으로 유지하는 것을 특징으로 하는 방법.

### 【청구항 9】 (종속항 — 입출금 대칭성 계수 S)

제4항에 있어서,

상기 R 점수 산출 단계는 입출금 대칭성 계수(S)를 추가 승수로 적용하되, S = 1 − (해당 지표 입금 합계 / 출금 합계) 로 산출하며, 입금(회수) 비율이 높을수록 S가 감소하여 R 점수를 하향 보정하고, 완전 회수 시에도 최소 위험도를 유지하기 위해 하한(0.3)을 적용하며, 투기성지표(5호), 가상자산지표(7호), 자산은닉지표(8호) 등 입금 회수가 의미 있는 지표에만 선택적으로 적용하고, 비적용 지표에 대해서는 S = 1.0 으로 하여 보정을 생략하는 것을 특징으로 하는 방법.

### 【청구항 10】 (종속항 — 외부 사업자 검증)

제4항에 있어서,

상기 위험도 산정 시스템은, 거래내역에 포함된 사업자등록번호를 외부 사업자 정보 API에 조회하여 해당 사업자의 업종코드·업태·종목을 수신하고, 수신된 업종 정보를 카테고리 테이블의 업종분류와 대조하여 분류 정확도를 보강할 수 있으며, 동일한 외부 API를 통해 사업자의 폐업 여부를 실시간 확인하여 폐업 가맹점 거래에 대한 위험도를 가중할 수 있고, 상기 외부 API 연동은 개인정보보호법, 실명확인법, 전자금융거래법 등 관련 법령이 허용하는 범위 내에서 정보주체의 동의 또는 법률상 근거에 따라 수행되는 것을 특징으로 하는 방법.

### 【청구항 11】 (독립항 — 시스템 발명)

프로세서; 메모리; 및 상기 메모리에 저장되어 상기 프로세서에 의해 실행되는 프로그램을 포함하는 금융거래 위험도 분석 시스템으로서,

를 포함하는 것을 특징으로 하는 금융거래 위험도 분석 시스템.

## 제4장 발명을 실시하기 위한 구체적인 내용

### 4.1 시스템 전체 구성 (제1도 관련)

본 발명에 따른 시스템은 데이터 입력 계층(100), 처리 엔진 계층(200), 데이터 저장 계층(300), 및 출력 보고서 계층(400)으로 구성된다.

**데이터 입력 계층(100)**은 복수의 금융기관(국민은행, 신한은행, 하나은행 등)에서 발행한 은행 거래 내역 파일(110)과, 복수의 카드사에서 발행한 신용카드 승인 내역 파일(120)을 수신한다. 각 파일은 엑셀(xls, xlsx) 형식이며, 금융기관마다 컬럼 구조, 날짜 포맷, 금액 표기 방식이 상이하다.

**처리 엔진 계층(200)**은 은행 전처리 엔진(210), 카드 전처리 엔진(220), 카테고리 분류 엔진(230), 금융정보 통합 엔진(240), 위험도 산정 엔진(250), 및 소명서 생성 엔진(260)을 포함한다.

**데이터 저장 계층(300)**은 은행 전처리 데이터(310), 카드 전처리 데이터(320), 카테고리 규칙 데이터베이스(330), 은행 분류 완료 데이터(340), 카드 분류 완료 데이터(350), 및 통합 분석 데이터(360)를 저장한다. 데이터 무결성을 보장하기 위해 원자적 파일 쓰기(Atomic Write)를 적용하며, 다중 스레드 환경에서의 안전성을 위해 캐시 접근에 상호 배제 잠금(Mutex Lock)을 사용한다.

**출력 보고서 계층(400)**은 은행거래 분석 보고서(410), 신용카드 분석 보고서(420), 금융정보 종합분석 보고서(430), 위험거래 분석 보고서(440), 및 종합의견서(소명서 초안)(450)를 생성한다.

### 4.2 데이터 전처리 및 정규화 과정 (제2도 관련)

은행 전처리 엔진(210)은 금융기관별 전용 파서를 구동하여 각 은행의 고유한 엑셀 포맷을 인식한다. 국민은행은 상단 메타 행이 있고, 신한은행은 날짜 형식이 상이하며, 하나은행은 컬럼 명칭이 다르다. 각 파서는 이러한 차이를 자동 감지하여 [거래일, 적요, 입금액, 출금액, 잔액, 내용, 송금메모, 거래점] 등의 표준 컬럼으로 매핑한다.

카드 전처리 엔진(220)은 카드사별 엑셀 파일의 헤더를 자동 감지하고, 이용금액을 입금액/출금액으로 분리한다. 취소 거래(취소여부 Y 또는 음수 이용금액)는 자동 식별하여 구분한다.

### 4.3 3단계 계층적 카테고리 분류 엔진 (제3도 관련)

**S1. 전처리 단계**: 카테고리 규칙 데이터베이스에서 '분류=전처리' 행을 로드하고, 키워드 길이 역순으로 정렬한 후, 원본 데이터의 적요란에 포함된 비정형 문자열을 순차적으로 치환한다.

**S2. 계정과목 분류 단계**: 정제된 텍스트에 대해 '분류=계정과목' 행에서 키워드를 검색한다. **최장 일치(Longest Match) 방식**을 적용하며, 복수 키워드 동시 매칭 시 **5단계 충돌 해결 캐스케이드**를 적용한다:
1. 매칭된 키워드 문자열의 길이가 긴 것을 우선
2. 길이 동일 시 위험도 코드 우선순위(M<L<D<F<P<H)
3. 위험도 코드 동일 시 텍스트 내 매칭 위치(인덱스) 우선
4. 매칭 위치 동일 시 소분류 코드 사전적 순서
5. 모든 조건 동일 시 기존 분류 유지

**S3. 후처리 단계**: '분류=후처리' 행을 적용하여 법적 용어로 최종 보정한다.

### 4.5 R 점수 산출 및 복합 보정 모델 (제5도 관련)

$$R = w \times \log_{10}(1 + f \times \overline{a}) \times T \times \Theta \times S$$

**시계열 근접성 계수(T)**: T = 1 + α × C + β × P
- C (집중도) = 1 − (해당 지표 활동기간 / 전체 분석기간)
- P (최근성) = 전체 분석기간 최근 25% 구간의 거래 건수 / 전체 건수
- α = 0.3, β = 0.2
- 범위: 1.00 ~ 1.50

**동적 임계 계수(Θ)**: Θ = amount_factor × count_factor
- amount_factor: 금액 < threshold → 금액/threshold (선형 감점); threshold ≤ 금액 ≤ 3×threshold → 1.0; 금액 > 3×threshold → 1 + γ × log₂(금액/threshold) (상한 2.0)
- count_factor: 건수 ≥ count_threshold → 1 + δ × (건수/count_threshold − 1) (상한 1.3); 건수 < count_threshold → 1.0
- γ = 0.1, δ = 0.05, count_threshold = 10
- 범위: 0.00 ~ 2.60

**입출금 대칭성 계수(S)**: S = 1 − (입금합계 / 출금합계)
- 적용 대상: 투기성(5호), 가상자산(7호), 자산은닉(8호)
- 하한 0.3 (완전 회수 시에도 최소 위험도 유지)
- 비적용 지표: S = 1.0
- 범위: 0.30 ~ 1.00

**등급 판정 (전체 합산)**:
- R합계 < 50 → 안전(Safe)
- R합계 < 100 → 보통(Normal)
- R합계 < 150 → 주의(Caution)
- R합계 ≥ 150 → 심각(Critical)

### 4.6 소명서 자동 생성 (제6도 관련)

**단계 1. 위험거래 집계**: 위험도 ≥ 0.5인 거래 추출, 지표별 그룹핑.

**단계 2. 상위 항목 선정**: 금액 기준 상위 5개 항목 우선 소명 대상.

**단계 3. 소명 전략 매칭**: 지표별 소명 전략 템플릿(SUGGESTION_TEMPLATES) 10종 호출.

**단계 4. 법적 근거 매칭**: 채무자회생법 제564조 해당 항(LEGAL_REFERENCES) 첨부.

**단계 5. 보고서 합성**: 위험거래 분석 보고서, 종합의견서, 인쇄용 보고서 자동 생성.

### 4.8 다른 실시예 (선택적 확장)

**생성형 AI 소명 합성**: 본 발명의 다른 실시예에서는 소명서 생성 단계에서 생성형 AI(Large Language Model)를 연동하여 사건별 맞춤 소명 문안을 합성할 수 있다. 정형 생성 방식의 템플릿 출력을 기본값으로 하되, AI 합성 결과가 가용한 경우 이를 우선 사용하고, AI 합성 불가 시 정형 생성으로 자동 폴백한다.

**인터랙티브 질의응답**: 위험도 분석 결과를 기반으로 지표별 맞춤 질문을 생성하고, 사용자 인터페이스를 통해 답변을 수집하여 소명서 초안에 반영하는 양방향 소명서 작성 기능을 더 포함할 수 있다. 답변 미입력 항목은 정형 생성 문구를 유지한다.

**외부 사업자 검증**: 거래내역의 사업자등록번호를 외부 API(국세청, VAN, PG 등)에 조회하여 업종 확인 및 폐업 여부를 실시간 검증할 수 있다. 다만 현재 이 확장이 적용되지 않는 이유는 기술적 한계가 아닌 법적·제도적 제약에 기인한다:
- 은행 이체 내역에는 사업자등록번호가 미포함
- 개인정보보호법에 따른 정보주체 동의 필요
- 국세청 업종분류 접근 제한(주업종/부업종만 공개 조회)
- VAN/PG 가맹점 정보 조회 시 별도 동의 절차 필요

상기 법적·제도적 장치가 보완되는 경우, 시스템은 카드 매출전표 사업자등록번호 → 국세청 API 업종 자동 분류, VAN/PG API 폐업 실시간 확인, FIU 등록 정보 대조 등으로 확장될 수 있다.

## 제6장 요약서

### 【요약】

본 발명은 개인회생 및 파산 절차에서 채무자의 금융 거래 내역을 자동 분석하여 법적 소명 자료를 생성하는 시스템에 관한 것이다. 복수의 금융기관에서 발행된 이기종 거래 데이터를 단일 구조로 통합한 후, 전처리-계정과목분류-후처리의 3단계 계층적 분류 엔진과 최장 일치(Longest Match) 알고리즘, 5단계 충돌 해결 캐스케이드를 통해 비정형 거래 텍스트를 정밀 분류한다. 분류 완료된 각 거래에 대해 채무자회생법 제564조에 대응하는 1호~10호 위험도 지표를 단계적 위험도 상향(Step-up Scoring) 방식으로 적용하여 법적 리스크를 정량화한다. 시계열 근접성 계수(T), 동적 임계 계수(Θ), 입출금 대칭성 계수(S)의 복합 보정 모델과 로그 스케일 가중치를 통해 위험도 종합 점수(R)를 산출하고, 최종적으로 위험 거래에 대한 소명 전략 템플릿과 법적 근거를 자동 매칭하여 법원 제출용 소명서 초안을 생성한다.
