【발명의 명칭】

국문: 인공지능 기반의 금융거래 위험도 분석 및 법적 소명 자료 자동 생성 시스템 및 방법

영문: AI-based Financial Transaction Risk Analysis and Automated Legal Explanation Document Generation System and Method

【기술분야】

본 발명은 금융 데이터 분석 및 법률 서비스(Legal-Tech) 기술에 관한 것으로, 프로세서 및 메모리를 포함하는 컴퓨팅 장치에 의해 수행되며, 이기종 금융기관의 거래 내역 데이터를 통합 전처리하고, 법적 기준에 따른 위험도를 정량적으로 산정하여, 개인회생 및 파산 절차에 필요한 소명 자료를 자동 생성하는 시스템 및 방법에 관한 것이다.

【배경기술】

개인회생 및 파산 신청 건수가 증가함에 따라, 채무자는 방대한 금융 거래 내역에 대해 면책 불허가 사유(사치, 도박, 자산 은닉 등)가 없음을 법원 및 파산관재인에게 소명해야 한다. 그러나 기존 방식에는 다음과 같은 한계가 존재한다.

첫째, 은행 거래 내역과 신용카드 내역은 금융기관마다 데이터 포맷이 상이하여 통합 분석에 막대한 수기 작업이 소요된다.

둘째, 거래 적요나 가맹점명이 비정형 텍스트로 기록되어 단순 키워드 검색만으로는 법적 위험 거래를 정확히 분류하기 어렵다.

셋째, 분석 결과에 대해 법원 제출용 소명 자료 작성 시, 법률 지식 부족으로 방어권 행사가 제한된다.

넷째, 파산 신청 직전 집중 거래, 과다 빈도 거래, 투자금 유출 후 부분 회수 등의 패턴을 종합 분석하는 정량적 모델이 부재하다.

한편, 선행기술로서 AI를 이용한 재무 관리 클라우드 서비스(KR102486186B1)는 시계열 분석과 ARIMA/LSTM 등을 활용하여 기업의 자금 흐름을 예측하고, 빅데이터 및 기계학습을 이용한 자동 분개 서버(KR101914620B1)는 신뢰도 임계치를 통한 복수 계정과목 추천 기능을 제공하며, 손익계산서 계정과목 자동 분류 시스템(KR101892671B1)은 거래처-계정과목 1:1 매칭을 수행하나, 이들은 모두 회계·세무 또는 기업 재무 예측을 목적으로 하며, 채무자회생법에 기반한 법적 위험도의 정량적 산정과 소명서 자동 생성 기능은 결여되어 있다.

따라서 이기종 금융 데이터를 통합하고, 법적 기준에 따른 위험도를 정량적으로 산출하며, 시간적·금액적·빈도적 패턴을 복합적으로 반영하여, 법적 소명 문안을 자동 생성하는 지능형 시스템의 개발이 요구된다.

【해결하려는 과제】

상기 선행기술들은 각각 법률 문서 텍스트 분석(문헌1), 양식 기반 법률 문서 작성(문헌2), 회계 분개 자동화(문헌3), 소비 패턴 분석(문헌4), AI 기반 재무 관리·자금 예측(문헌5), 빅데이터·ML 기반 자동 분개(문헌6), 거래처-계정과목 매칭(문헌7)에 집중하고 있으나, 채무자 회생 및 파산에 관한 법률 제564조에서 규정하는 면책 불허가 사유를 정량적으로 산정하는 위험도 분석 엔진을 갖추고 있지 않으며, 금융 거래 내역에서 직접 법적 소명 자료를 자동 생성하는 기능도 결여되어 있다.

특히, 문헌5의 시계열 분석(ARIMA/LSTM)은 기업 자금 흐름의 미래 예측을 목적으로 하는 반면, 본 발명의 시계열 근접성 계수(T)는 파산 신청 직전의 의심 거래 패턴을 탐지하는 것을 목적으로 하여 기술적 사상이 상이하다. 또한, 문헌6의 신뢰도 임계치는 분류 결과의 확실성을 평가하는 반면, 본 발명의 동적 임계 계수(Θ)는 금액 규모와 거래 빈도의 복합 가중을 통해 법적 위험도의 크기를 보정하는 것으로 목적과 산출 방식이 근본적으로 다르다.

본 발명은 이기종 금융 데이터의 통합 전처리, 3단계 계층적 분류, 법적 근거 기반의 단계적 위험도 산정, 시계열·금액·빈도 패턴의 복합 위험도 보정, 및 소명 자료 자동 생성을 일관된 파이프라인으로 수행하는 시스템 및 방법을 제공하는 것을 목적으로 한다.

【과제의 해결 수단】

본 발명은 프로세서 및 메모리를 포함하는 컴퓨팅 장치에 의해 수행되는 금융거래 위험도 분석 방법을 제공한다.

첫째, 데이터 전처리부는 복수의 금융기관으로부터 서로 다른 포맷으로 발행된 은행 거래 내역 데이터 및 신용카드 승인 내역 데이터를 수신하여, 금융기관별 전용 파서를 통해 단일한 통합 데이터 구조로 정규화한다.

둘째, 3단계 계층적 분류 엔진을 통해 분류 정확도를 극대화한다. 전처리→계정과목 분류(최장 일치 방식)→후처리. 복수 키워드 동시 매칭 시 5단계 충돌 해결 캐스케이드 적용.

셋째, 단계적 위험도 상향(Step-up Scoring) 알고리즘. 제1호(0.1)~제10호(5.0) 순차 판정, 상위 호수 조건 충족 시 갱신.

넷째, R = w × log₁₀(1 + f × avg_a) × T × Θ × S 공식으로 정량적 위험도 종합 점수를 산출한다. T는 시계열 근접성(집중도·최근성), Θ는 동적 임계(금액·건수 복합), S는 입출금 대칭성(회수 비율).

다섯째, 소명서 생성부는 지표별 법적 소명 전략 템플릿과 채무자회생법 조항 참조를 조합하여 소명서 초안을 자동 생성한다.

【발명의 효과】

1. 정밀도·효율성 극대화: 이기종 데이터 통합, 3단계 분류, 5단계 캐스케이드.

2. 정량적 위험도 산출: 1~10호 Step-up Scoring + T·Θ·S 복합 보정.

3. 방어권 강화: 소명 전략 템플릿+법조항 자동 매칭.

4. 산업상 이용: 회생, 이혼 재산 분할, 기업 감사 등 확장 가능.

제3장 특허청구범위

【청구항 1】 (독립항 — 방법)

프로세서 및 메모리를 포함하는 컴퓨팅 장치에 의해 수행되는 금융거래 위험도 분석 및 법적 소명 자료 자동 생성 방법으로서,

(a) 복수의 금융기관으로부터 서로 다른 포맷으로 발행된 은행 거래 내역 데이터 및 신용카드 승인 내역 데이터를 수신하여, 금융기관별 전용 파서를 통해 단일한 통합 데이터 구조로 정규화하는 데이터 전처리 단계;

(b) 사전 정의된 카테고리 테이블을 참조하여, 상기 통합 데이터의 적요 또는 가맹점 명칭에 포함된 비정형 텍스트를 (i) 전처리, (ii) 계정과목 분류, (iii) 후처리의 3단계를 거쳐 자동 분류하되, 상기 계정과목 분류 시 검색 키워드의 길이 역순으로 매칭을 시도하는 최장 일치(Longest Match) 방식을 적용하는 3단계 계층적 카테고리 분류 단계;

(c) 분류 완료된 은행 거래 데이터와 신용카드 거래 데이터를 단일 통합 데이터로 병합하는 이기종 데이터 통합 단계;

(d) 상기 통합 데이터의 각 거래에 대하여, 채무자 회생 및 파산에 관한 법률 제564조에 대응하는 제1호(분류제외) 내지 제10호(사행성) 위험도 지표를 순차 적용하되, 상위 호수 조건 충족 시 기 산정된 위험도를 갱신하는 단계적 위험도 상향(Step-up Scoring) 방식의 위험도 산정 단계; 및

(e) 위험도 가중치(w), 거래 건수(f), 평균 거래 금액(avg_a), 시계열 근접성 계수(T), 동적 임계 계수(Θ), 입출금 대칭성 계수(S)를 사용하여 R = w × log₁₀(1 + f × avg_a) × T × Θ × S 공식으로 위험도 종합 점수(R)를 산출하고, 지표별 법적 소명 전략 템플릿과 법조항 참조를 매칭하여 소명서 초안을 자동 생성하는 소명 자료 생성 단계;

를 포함하는 것을 특징으로 하는 금융거래 위험도 분석 및 법적 소명 자료 자동 생성 방법.

【청구항 2】 (5단계 충돌 해결)

제1항에 있어서, 계정과목 분류 시 복수 키워드 동시 매칭 시 (i) 키워드 길이 우선, (ii) 위험도 코드 우선, (iii) 텍스트 내 출현 위치 우선, (iv) 소분류 코드 순, (v) 기존 분류 유지의 5단계 충돌 해결 캐스케이드를 적용하는 것을 특징.

【청구항 3】 (위험도 산정 상세)

제1항에 있어서, 제1호(0.1) 기본값 부여 후 제2호(0.5)~제10호(5.0) 순차 판정, 제2호(심야·폐업) 해당 시 3~10호 생략, 제5호~제10호는 카테고리 테이블 위험도분류 키워드+기준 출금액 결합 조건으로 판정.

【청구항 4】 (R 점수 및 T 계수)

제1항에 있어서, T = 1 + α × C + β × P (C: 활동기간 대비 집중도, P: 최근 25% 구간 건수 비율)로 산출하여 단기간 집중·파산 직전 의심 패턴을 반영.

【청구항 5】 (소명서: 정형 + AI)

제1항에 있어서, (a) 정형 생성(템플릿 치환) 또는 (b) 생성형 AI 합성 중 선택 수행, AI 불가 시 정형으로 자동 전환.

【청구항 6】 (VASP 3단계 식별)

제1항에 있어서, 가상자산 거래 식별은 (a) 카테고리 매칭 → (b) 접두번호 정규표현식 → (c) 사업자명 매칭 순서로 수행, 접두번호 테이블은 카테고리 테이블의 VASP계좌 분류 행으로 동적 확장 가능.

【청구항 7】 (동적 임계 계수 Θ)

제4항에 있어서, Θ = amount_factor × count_factor. 금액 임계값 미만 시 선형 감점, 3배 초과 시 누진 가중(γ, 상한 2.0), 건수 기준 이상 시 비례 가중(δ, 상한 1.3).

【청구항 8】 (인터랙티브 Q&A)

제1항에 있어서, 위험 항목 기반 맞춤 질문 생성 → 사용자 답변 수집 → 소명서 재생성. 미입력 항목은 정형 생성 문구 유지.

【청구항 9】 (입출금 대칭성 S)

제4항에 있어서, S = 1 − (입금합계/출금합계), 하한 0.3. 투기성(5호)·가상자산(7호)·자산은닉(8호)에만 적용, 비적용 지표는 S=1.0.

【청구항 10】 (외부 API 검증)

제4항에 있어서, 사업자등록번호를 외부 API에 조회하여 업종·폐업 여부 확인. 개인정보보호법 등 법령 허용 범위 내 수행.

【청구항 11】 (독립항 — 시스템)

프로세서, 메모리, 프로그램을 포함하는 시스템으로서: 데이터 전처리부 + 분류 엔진부(3단계, 최장 일치) + 데이터 통합부 + 위험도 산정부(1~10호 Step-up) + 소명서 생성부(R 점수 + 템플릿).

4.5 R 점수 산출 (제5도)

4.6 선행기술 대비 차별적 기술 구성

#### T 계수 vs 문헌5(KR102486186B1) 시계열 분석

문헌5는 ARIMA/LSTM으로 기업 자금 흐름의 미래 예측을 수행하며, 원인-결과 거래의 최소 비용값 매칭에 시계열을 활용한다. 반면, 본 발명의 T 계수는 과거 거래의 시간적 집중도(C)와 최근성(P)을 산출하여 파산 신청 직전의 의심 거래 패턴을 탐지하는 것으로, 예측이 아닌 탐지·진단 목적이며, 산출 수식(T=1+αC+βP)도 근본적으로 상이하다.

#### Θ 계수 vs 문헌6(KR101914620B1) 신뢰도 임계치

문헌6의 신뢰도 임계치는 ML 분류 결과의 확실성을 평가하여 임계치 이하 시 복수 후보를 추천하는 메커니즘이다. 반면, 본 발명의 Θ 계수는 금액 규모에 대한 누진 가중(amount_factor)과 거래 빈도에 대한 비례 가중(count_factor)의 곱으로, 법적 위험도의 크기를 보정하는 승수이다. 목적(분류 확실성 vs 위험도 크기 보정)과 산출 방식이 근본적으로 상이하다.

#### 5단계 캐스케이드 vs 문헌7(KR101892671B1) 주계정과목

문헌7은 거래처 상호와 계정과목의 1:1 매칭에서 복수 계정과목이 존재할 때 주계정과목등록부(R)를 통해 사전 지정된 하나를 선택한다. 반면, 본 발명은 키워드 매칭 시 동적으로 (i) 키워드 길이, (ii) 위험도 코드 우선순위, (iii) 텍스트 내 출현 위치, (iv) 소분류 코드, (v) 기존 분류 유지의 5단계 기준을 자동 적용한다. 사전 지정(정적) vs 동적 다기준 해결이라는 근본적 차이가 있다.
