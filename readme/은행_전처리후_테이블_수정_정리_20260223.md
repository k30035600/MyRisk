# 오늘 수정 사항 정리 (2026-02-23)

## 1. 테이블 헤더 뒤로 행이 비치는 문제 해결

**현상:** 은행거래 전처리후 테이블 스크롤 시, 본문 행이 스티키 헤더 뒤로 비치거나 헤더 위로 겹쳐 보임.

### 1-1. CSS만으로 시도 (초기)

- **MyBank/templates/category.html**
  - `#category-applied-table` thead: `z-index: 100`, `isolation: isolate`, thead th에 `box-shadow` 추가.
- **MyBank/templates/index.html**
  - 전처리전/전처리후 `thead`/`th`: `z-index`, `isolation`, `transform: translateZ(0)`, `box-shadow` 강화.
  - `#processed-data-table` td: `opacity` 제거 → `color: rgba(0,0,0,0.82)` (stacking context 감소).
  - `#source-data-table` / `#processed-data-table` thead 전체에 sticky·레이어 스타일 적용.
  - tbody에 `position: relative; z-index: 0` 지정.
  - 테이블에 `isolation: isolate`, thead th에 `box-shadow: 0 6px 0 0 #FFCC80` (헤더색 막으로 가림).

→ 브라우저/환경에 따라 완전히 해결되지 않음.

### 1-2. 구조 분리 (최종 방식)

- **헤더와 본문을 DOM에서 분리**
  - 스크롤 영역: 본문 테이블만 (`tbody`만 포함).
  - 헤더: 별도 div + 테이블(`.processed-data-header-table`)로 상단 고정, 스크롤 밖에 위치.
- **HTML 구조**
  - `.processed-data-table-wrap`  
    → `.processed-data-table-header` (헤더용 테이블)  
    → `.processed-data-table-body-wrap` (overflow-y: auto)  
      → `#processed-data-table` (thead 없음, tbody만).
- **CSS 추가**
  - `.processed-data-table-wrap`, `.processed-data-header-table`, `.processed-data-table-body-wrap` 스타일.
  - 컬럼 너비 비율은 기존과 동일(7.45%, 12.77% 등).
  - `.box-content.has-split-table`: `height: 75vh`, `overflow: hidden`, flex 레이아웃.
- **JS**
  - 테이블 렌더 시 위 구조로 HTML 생성.
  - `contentDiv.classList.add('has-split-table')` (데이터 있을 때).
  - `applyProcessedTableLayout(contentDiv)`: body-wrap 높이를 `contentDiv 높이 − 헤더 높이`로 계산해 지정, 최대 10프레임 재시도.
  - 데이터 없음/오류 시 `classList.remove('has-split-table')`.

→ 스크롤은 본문 영역만 하므로 행이 헤더 위로 그려지지 않음.

## 2. 은행거래 전처리후 테이블 출력 코드 정리

**파일:** `MyBank/templates/index.html`

### 2-1. 상수

- `PROCESSED_TABLE_COLUMN_ORDER` — 표시 컬럼 순서 (11개).
- `PROCESSED_TABLE_EXCLUDE_COLS` — 제외 컬럼 (잔액, 입출금, 거래유형, 키워드, 카테고리, 기타거래).

### 2-2. 헬퍼 함수

| 함수 | 역할 |
|------|------|
| `getProcessedColumns(dataColumns)` | 표시 컬럼 목록 계산 (순서·제외 반영) |
| `normalizeProcessedRows(rows)` | 메모 삭제, 입출금 보정, 적요 `-`→공백, 취소 표기 통일 |
| `getProcessedCellStyle(col)` | 컬럼별 셀 스타일 문자열 (입금/출금 색, 정렬) |
| `getProcessedDisplayValue(col, value)` | 컬럼별 표시값 포맷 (날짜/시간/금액) |
| `buildProcessedTableHtml(columns, rows, sumDeposit, sumWithdraw)` | 헤더 분리 구조의 테이블 HTML 생성 |
| `applyProcessedTableLayout(contentDiv)` | body-wrap 높이 계산 및 스크롤 영역 적용 |

### 2-3. loadRightData() 변경

- 데이터 있음: `normalizeProcessedRows` → `getProcessedColumns` → 입출금 합계 계산 → `buildProcessedTableHtml` → `innerHTML` + `has-split-table` → `applyProcessedTableLayout`.
- 기존 인라인 HTML 조립·레이아웃 IIFE 제거, 동작은 동일하게 유지.

## 4. 앱 전체 테이블에 동일 방식 반영 (오늘 확장)

**적용한 테이블 (헤더/본문 구조 분리)**

| 앱 | 화면 | 테이블 | 비고 |
|----|------|--------|------|
| MyBank | index (전처리전) | #source-data-table | wrap + source-data-header-table + body-wrap + tbody |
| MyBank | index (전처리후) | #processed-data-table | 기존 적용 유지 |
| MyCard | index (전처리전) | #source-data-table | 동일 패턴 |
| MyCard | index (전처리후) | #processed-data-table | 동일 패턴 |
| MyCash | index (전처리전) | #source-data-table | 동일 패턴 |
| MyCash | index (전처리후) | #processed-data-table | 동일 패턴 |

**공통 사항**
- 각 앱 `index.html`에 `.box-content.has-split-table`, `.source-data-table-wrap`, `.processed-data-table-wrap`, `.source-data-table-header`, `.processed-data-table-header`, `.source-data-header-table`, `.processed-data-header-table`, `.source-data-table-body-wrap`, `.processed-data-table-body-wrap` CSS 추가.
- 테이블 렌더 시 wrap → 헤더용 테이블 → body-wrap → 본문 테이블(tbody만) 순으로 HTML 생성, `has-split-table` 클래스 추가, `applySplitTableLayout(contentDiv)` 호출.
- MyBank는 `applySplitTableLayout`를 전처리전/전처리후 공용으로 사용. MyCard·MyCash는 각각 동일 함수 추가.

**카테고리 적용/조회 테이블**
- `#category-applied-table`, `#category-query-table` 등은 헤더가 2행(컬럼명 + 필터 드롭다운)이라, 구조 분리 시 필터 DOM·ID 참조와 컬럼 너비 동기화 등 추가 수정이 필요합니다. 현재는 기존 스티키 헤더 + z-index·box-shadow 방식 유지.
