# 금융정보 위험도 지표 매칭 가이드 (Risk Index Matching)

금융정보 데이터(cash_after)의 위험도를 산출하고, **키워드**, **기타거래**, **입금액**, **출금액** 등을 기반으로 1호부터 10호까지의 지표를 순차 매칭하여 시스템에 반영하는 절차를 정의합니다.

---

## 1. 위험도 지표 기준표 (1~10호)

`lib/category_table_io.py`의 `RISK_CLASS_TO_VALUE` 기준.

| 호수 | 위험도분류(지표명) | 위험도 | 매칭 조건 요약 |
|------|---------------------|--------|----------------|
| 1호 | 분류제외지표 | 0.1 | 2~10호 조건에 해당하지 않는 행(기본값) |
| 2호 | 심야폐업지표 | 0.5 | 폐업="폐업" 또는 심야시간대 + 출금 ≥ 10만원 |
| 3호 | 자료소명지표 | 1.0 | 출금 ≥ 500만원(하한금액 설정 가능) |
| 4호 | 비정형지표 | 1.5 | 특정인=키워드 기준, 출금만 ≥ 100만원, 3회 이상 |
| 5호 | 투기성지표 | 2.0 | 출금만 ≥ 50만원 + 증권/선물/자산운용/위탁 키워드 |
| 6호 | 사기파산지표 | 2.5 | 출금만 ≥ 50만원 + 대부/P2P/카드깡/원리금 키워드 |
| 7호 | 가상자산지표 | 3.0 | 출금만 ≥ 50만원 + 가상자산 분류 또는 코인 거래소 키워드 |
| 8호 | 자산은닉지표 | 3.5 | 출금만 ≥ 50만원 + 해외송금/SWIFT/Wise 키워드 |
| 9호 | 과소비지표 | 4.0 | 출금만 ≥ 30만원 + 백화점/명품/유흥 키워드 |
| 10호 | 사행성지표 | 5.0 | 출금만 ≥ 10만원 + 경마/복권/도박/안마 키워드 |

- **위험도분류**: cash_after의 **위험도분류** 컬럼에 저장되는 값.
- **위험도**: 동일 행의 **위험도** 컬럼에 저장되는 수치(0.1 ~ 5.0).
- category_table.json에서 **분류** 또는 **카테고리**가 위 표의 위험도분류명인 행은 업종분류·위험도 매칭에 사용됨.

## 2. 기본 처리 원칙

- **초기화**: 모든 행의 기본 위험도는 **0.1**(1호 분류제외지표)로 설정됩니다.
- **1차분류·2차분류**: 위험도 적용을 **1차분류**(3호·4호)와 **2차분류**(5호~10호)로 나눕니다. 1차분류에 해당하는 행은 2차분류에서 제외됩니다. 2호(심야폐업)는 별도 선행 적용.
- **데이터 업데이트**: 매칭된 결과는 cash_after 테이블의 **위험도키워드**, **위험도분류**, **위험도** 컬럼에 저장됩니다.

### 2호 (심야폐업지표) — 선행 적용

- **폐업** = "폐업"이면 금액 무관 2호 적용.
- **거래시간**이 심야 시간대이고 **출금액** ≥ 10만원이면 2호 적용.
- 위험도키워드: "폐업" / "심야" / "폐업 심야".

### 1차분류 (3호·4호)

- **정렬**: cash_after를 **키워드 → 거래일** 올림차순으로 정렬한 뒤 적용합니다.
- **적용 기준**: **출금액** 조건으로만 위험도 적용(3호: 출금 500만원 이상, 4호: 출금만 100만원 이상·동일 키워드 3회 이상).
- 3호 또는 4호에 해당하는 행은 **2차분류(5호~10호)를 적용하지 않습니다.**

### 2차분류 (5호~10호)

- **정렬**: cash_after를 **카테고리 → 키워드 → 거래일** 올림차순으로 정렬한 뒤 적용합니다.
- **적용 기준**: **(카테고리+키워드+기타거래)** 결합 텍스트를 대상으로 키워드 매칭 후 위험도 적용.
- 2호·3호·4호에 해당한 행은 이 단계에서 skip됩니다.

## 3. 호별 컬럼 매칭 방식

**category_table.json만 사용합니다.** `lib/category_table_io.py`의 `get_risk_class_table_data()`·`get_risk_class_map_for_apply()`가 category_table의 위험도 분류 행(1~10호)으로 매칭합니다.

### 3.1 거래 데이터(cash_after 행)에서 쓰는 주요 컬럼

| 컬럼명 | 용도 |
|--------|------|
| **입금액** | 금액 조건(출금만 해당 시 0인지 확인) |
| **출금액** | 금액 조건(최소 출금액 판단) |
| **거래시간** | 2호 심야구분 여부 |
| **폐업** | 2호 폐업 여부 |
| **키워드** | 3·4호 위험도키워드 저장, 5~10호 검색 텍스트 |
| **기타거래** | 3·4호 위험도키워드 대체, 5~10호 검색 텍스트 |
| **카테고리** | 5~10호 검색 텍스트, 7호는 "가상자산" 직접 비교 |

### 3.2 업종분류 컬럼 (category_table 기반)

| 컬럼명 | 용도 |
|--------|------|
| **업종분류** | 위험도 유형 이름 → 거래의 **위험도분류**와 매칭 |
| **위험도** | 위험도 수치 → 거래의 **위험도**와 매칭 |
| **업종코드** | 하한금액(원, 숫자) 또는 조건 설명. 숫자만 있으면 해당 호의 최소 출금액(하한)으로 사용 |
| **키워드** | 키워드·조건 설명(참고용) |

### 3.3 호별 컬럼 매칭 상세

#### 1호 — 분류제외지표 (위험도 0.1)

| 구분 | 컬럼·매칭 |
|------|-----------|
| **거래에서 보는 컬럼** | 없음(2~10호 조건에 해당하지 않는 경우 기본값) |
| **설정값** | 위험도분류: 분류제외지표, 위험도: 0.1, 위험도키워드: 비움 |
| **category_table 매칭** | **업종분류** = "분류제외지표", **위험도** = "0.1" |

#### 2호 — 심야폐업지표 (위험도 0.5)

| 구분 | 컬럼·매칭 |
|------|-----------|
| **거래에서 보는 컬럼** | **폐업**, **거래시간**, **출금액** |
| **조건** | (1) 폐업="폐업"이면 금액 무관. (2) 거래시간이 심야 시간대이고 출금액 ≥ 10만원 |
| **설정값** | 위험도키워드: "폐업"/"심야"/"폐업 심야", 위험도분류: 심야폐업지표, 위험도: 0.5 |
| **category_table 매칭** | **업종분류** = "심야폐업지표", **위험도** = "0.5", **키워드** = "심야 10만원 이상 및 폐업에 해당하는 거래" |

#### 3호 — 자료소명지표 (위험도 1.0)

| 구분 | 컬럼·매칭 |
|------|-----------|
| **거래에서 보는 컬럼** | **출금액**, **키워드**, **기타거래** |
| **조건** | 2호 해당 행 제외, 출금액 ≥ 500만원 |
| **설정값** | 위험도키워드: 해당 행의 키워드(없으면 기타거래), 위험도분류: 자료소명지표, 위험도: 1.0 |
| **category_table 매칭** | **업종분류** = "자료소명지표", **업종코드** = 하한금액(숫자) |

#### 4호 — 비정형지표 (위험도 1.5)

| 구분 | 컬럼·매칭 |
|------|-----------|
| **거래에서 보는 컬럼** | **입금액**, **출금액**, **키워드**, **기타거래** |
| **조건** | 2호 제외, 출금액 ≥ 100만원, 입금액 ≤ 0, 같은 키워드(또는 기타거래)로 3회 이상 |
| **설정값** | 위험도키워드: 해당 행의 키워드(또는 기타거래), 위험도분류: 비정형지표, 위험도: 1.5 |
| **category_table 매칭** | **업종분류** = "비정형지표", **업종코드** = 하한금액(숫자) |

#### 5호~10호

| 구분 | 컬럼·매칭 |
|------|-----------|
| **거래에서 보는 컬럼** | **입금액**, **출금액**, **카테고리**, **키워드**, **기타거래** |
| **조건** | 2·3·4호 제외, (카테고리+키워드+기타거래) 결합 텍스트에서 키워드 매칭 + 출금액 ≥ min_out |
| **설정값** | 위험도키워드: 매칭된 키워드, 위험도분류: 해당 호 지표명, 위험도: 해당 수치 |
| **category_table 매칭** | **업종분류** = 각 호 지표명, **위험도**, **업종코드**(하한금액), **키워드** |

### 3.4 호별 컬럼 매칭 요약표

| 호수 | 위험도분류 | 거래에서 사용하는 컬럼 | category_table 매칭 컬럼 |
|------|-----------|----------------------|--------------------------|
| 1호 | 분류제외지표 | (기본값) | 업종분류, 위험도 |
| 2호 | 심야폐업지표 | 폐업, 거래시간, 출금액 | 업종분류, 위험도, 키워드 |
| 3호~10호 | (각 호별 위험도분류) | 입금액, 출금액, 카테고리, 키워드, 기타거래 등 | 업종분류, 위험도, 업종코드(하한금액), 키워드 |

- **거래 행의 위험도분류** = category_table 위험도 분류 행의 **업종분류** 값과 대응.
- **거래 행의 위험도** = **위험도** 값과 대응.
- **위험도키워드**는 category_table 매칭 시 키로 사용. **업종코드**에 숫자(원)만 넣으면 해당 호의 **하한금액**(최소 출금액)으로 사용.

## 4. 하한금액·상한금액 설정

### 4.1 하한금액 (현재 지원)

- **파일**: `data/category_table.json`
- **대상 행**: `분류` = `"위험도분류"` 인 행만 사용.
- **업종코드** 컬럼에 **숫자만** 넣으면(예: `1000000`, `5000000`), 그 값(원)이 해당 **카테고리(위험도분류명)** 의 **최소 출금액(min_out)** 으로 사용됩니다.
- `risk_indicators.py`의 `_parse_min_out(업종코드)`가 숫자로 파싱하고, `_load_위험도분류_keywords()`에서 호별 `min_out`을 로드합니다.
- 매칭 시 **출금액 >= min_out** 일 때만 해당 위험도분류가 적용됩니다. 업종코드가 비어 있거나 숫자가 아니면 **0**(금액 조건 없음)으로 처리.

같은 카테고리(호)가 여러 행이면, 같은 호 내에서 **가장 작은 min_out**이 사용됩니다(`min(cur, parsed_min)` 처리).

**설정 예시** (category_table.json):

```json
{
  "분류": "위험도분류",
  "키워드": "증권, 선물, 자산운용",
  "카테고리": "투기성지표",
  "위험도": "2",
  "업종코드": "500000"
}
```

위 예: **투기성지표** 호에 대해 **최소 출금액 50만 원** 설정. 출금액이 50만 원 이상인 거래만 이 호 조건(키워드 매칭 시) 적용 대상.

### 4.2 상한금액 (미지원)

현재 상한금액을 읽거나 적용하는 로직은 없습니다. 구현하려면:

1. **업종코드 규칙 확장** — `"1000000-50000000"` 형태로 저장, `_parse_max_out`을 만들어 상한 파싱 후 `출금액 <= max_out` 조건 추가.
2. **별도 컬럼 사용** — category_table에 `하한금액`, `상한금액` 컬럼을 추가하고 `_load_위험도분류_keywords()`에서 읽어 각 호별 min/max로 사용.
3. **기존 업종코드 유지** — 하한만 두고, 상한은 별도 JSON 설정이나 다른 필드로 관리.

구현 시 `risk_indicators.py`의 `_load_위험도분류_keywords` 반환 구조에 `max_out`을 넣고, `apply_risk_indicators()`에서 `출금액 <= max_out` 검사만 추가.

## 5. 지표별 키워드 매칭 상세

### [3호·4호] 자료소명·비정형 (키워드를 위험도키워드로 저장)

- **3호 (자료소명지표)**: 출금 **500만원 이상**인 행 1건만 있어도 해당 행의 **키워드**(또는 기타거래) 값을 **위험도키워드**에 저장.
- **4호 (비정형지표)**: **특정인 = 키워드**. 같은 키워드(동일인)로 묶어, **출금만 100만원 이상**인 거래가 **3회 이상**이면 그 키워드의 해당 행들에 4호 부여. **위험도키워드**에는 그 **키워드** 값을 저장.

### [5호~7호] 투자 및 투기 관련 (출금만 조건)

- **5호 (투기성)**: **출금만 50만원 이상**. 증권/선물/자산운용/위탁 관련 거래 내역.
- **6호 (사기파산)**: **출금만 50만원 이상**. 동일 가맹점 카드깡, 대부업체 원리금 반복 상환. (키워드: 대부, P2P, 카드깡, 원리금)
- **7호 (가상자산)**: **출금만 50만원 이상**. "가상자산" 또는 VASP 거래소 명칭 매칭. category_table에서 분류 "가상자산", 키워드(업비트/빗썸/코인원/코빗 등) 매칭.

### [8호~10호] 자산 은닉 및 불건전 소비

- **8호 (자산은닉)**: SWIFT 코드(8~11자리), 해외 송금 플랫폼(Wise/TransferWise) 매칭.
- **9호 (과소비)**: 백화점/명품 외 업종코드 기반(유흥주점 56211, 무도장 91291 등) 또는 키워드 매칭.
- **10호 (사행성)**: 사설도박, 복권, 불건전 게임 S/W 개발(5821), 향락업소(휴게텔, 키스방, 안마 등).

## 6. 분류·저장 흐름

### 저장 구조 (cash_after.json)

| 컬럼 | 설명 |
|------|------|
| **위험도키워드** | 매칭된 키워드 (3·4호: 행의 키워드/기타거래, 5~10호: 매칭 키워드). 지표명은 넣지 않음 |
| **위험도분류** | 1~10호 지표명 (분류제외지표, 심야폐업지표, 자료소명지표, …, 사행성지표) |
| **위험도** | 숫자 0.1 ~ 5.0 (높을수록 위험) |

구 컬럼명(업종코드, 업종분류)은 로드 시 위험도키워드·위험도분류로 자동 변환됩니다.

### 흐름

1. **cash_after 생성**
   은행거래(bank_after) + 신용카드(card_after) 병합 시, 각 행에 `위험도키워드`, `위험도분류`, `위험도` 컬럼이 비어 있는 상태로 생성됩니다.

2. **category_table 기반 위험도 매칭**
   `_apply_업종분류_from_category_table()`:
   - category_table 기반(`get_risk_class_map_for_apply` 등)으로 **위험도키워드**를 매칭
   - 매칭된 행에 **위험도분류**, **위험도** 저장

3. **1~10호 위험도 지표 적용**
   `risk_indicators.apply_risk_indicators()`:
   - 모든 행 **위험도**를 0.1로 초기화
   - **2호**(심야폐업) 선행 적용
   - **1차분류**: **키워드 → 거래일** 올림차순 정렬 후 **3호·4호** 적용(출금액 기준). 해당 행은 2차분류에서 skip
   - **2차분류**: **카테고리 → 키워드 → 거래일** 올림차순 정렬 후 **5호~10호** 적용((카테고리+키워드+기타거래) 텍스트 매칭)

4. **고위험 카테고리 보정**
   `_apply_risk_category_by_keywords()`:
   - 카테고리/키워드/기타거래에서 가상자산·증권투자·금전대부 매칭 시 **위험도분류**, **위험도 5.0** 설정

5. **저장**
   최종 DataFrame을 `MyCash/cash_after.json`에 저장합니다.
   키 이름: `위험도키워드`, `위험도분류`, `위험도` (과거 업종코드/업종분류는 사용하지 않음).
