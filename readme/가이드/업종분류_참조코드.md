# 업종분류 참조 코드 (category_table만 사용)

## 1. 파일·모듈

- 업종분류·위험도 매칭은 **lib/category_table_io.py**와 **data/category_table.json**만 사용합니다.
- **data/category_table.json**: 위험도 분류 행(분류/카테고리=1~10호 이름)으로 업종분류 조회·매칭.
- **백업(xlsx)**: data/업종분류_table.xlsx는 `get_risk_class_table_data()` 결과를 `export_risk_class_table_to_xlsx()`로 내보낸 것 (readme/backup_업종분류_to_xlsx.py).

---

## 2. JSON 구조

- **컬럼**: `업종분류`, `위험도`, `업종코드`, `키워드`
- 업종분류 예: 자료소명지표, 비정형지표, 투기성지표, 심야폐업지표 등
- 위험도: 위험도 수치 (예: "1.0", "0.5")

---

## 3. 참조 위치 (Python)

### 3.1 MyCash/cash_app.py

| 위치 | 용도 |
|------|------|
| **_apply_업종분류_from_category_table(df)** (352~386행) | cash_after 행별로 **위험도키워드** → 위험도분류·위험도 매칭 (in-place). `lib.category_table_io.get_risk_class_map_for_apply()` 사용 |
| **index()** | 업종분류 조회 데이터는 `/api/risk-class-table`에서 로드 |
| **get_risk_class_table()** (1090~1109행) | API `/api/risk-class-table`: `get_risk_class_table_data()` 호출 후 JSON 응답 |
| **merge_bank_card_to_cash_after()** (524~526행) | cash_after 생성 시 “(4/6) category_table 기반 업종분류·위험도 매칭” 단계에서 `_apply_업종분류_from_category_table(df)` 호출 |

### 3.2 lib.category_table_io (업종분류·위험도 매칭)

- **get_risk_class_table_data()**  
  - category_table.json에서 위험도 분류 행만 추려 업종분류 형식 리스트 반환.  
  - 페이지/API에서 “업종분류 조회” 테이블·드롭다운용으로 사용.

- **get_risk_class_map_for_apply()**  
  - 반환: `(code_to_업종분류, code_to_위험도)`  
  - `_apply_업종분류_from_category_table()`에서 행의 **위험도키워드** 값을 키로 사용해 위험도분류·위험도 매칭.

- **export_risk_class_table_to_xlsx(xlsx_path)**  
  - get_risk_class_table_data() 결과를 xlsx로 내보내기 (category_table 기반).

### 3.3 readme/backup_업종분류_to_xlsx.py

- **backup_업종분류_table_to_xlsx()**  
  - `lib.category_table_io.export_risk_class_table_to_xlsx(xlsx_path=...)` 호출로 data/업종분류_table.xlsx 생성 (category_table 기반).

## 5. 사용 흐름 요약

1. **병합 시**:  
   `merge_bank_card_to_cash_after()` → `_apply_업종분류_from_category_table(df)` →  
   `get_risk_class_map_for_apply()`로 맵 조회 → 각 행의 **위험도키워드**로 위험도분류·위험도 채움.

2. **화면·API**:  
   `get_risk_class_table_data()`로 업종분류 데이터 생성 →  
   병합작업/업종분류 페이지 테이블 및 `/api/risk-class-table` 응답에 사용.

3. **백업**:  
   `python readme/backup_업종분류_to_xlsx.py` → category_table 기반으로 data/업종분류_table.xlsx 생성.

## 7. category_table 컬럼 매칭 구조

### 업종분류 조회 컬럼 ↔ category_table 컬럼

| 업종분류 조회용 컬럼 | category_table 컬럼 |
|-------------------------|----------------------|
| **업종분류** | **분류** |
| **(표시용 키워드)** | **키워드** |
| **업종분류**(이름) | **카테고리** |

- **위험도**(수치)는 코드에서 분류명별 고정값으로 부여합니다. (예: 자료소명지표→1.0, 사행성지표→5.0)
- 구현: **lib/category_table_io.py**가 `data/category_table.json`만 읽고, **분류** 또는 **카테고리**가 위험도 분류명(1~10호)인 행만 사용해 `get_risk_class_table_data()`, `get_risk_class_map_for_apply()`를 제공합니다.

### 거래 데이터(cash_after)에서 사용하는 주요 컬럼

| 컬럼명 | 용도 |
|--------|------|
| **입금액** | 금액 조건(출금만 해당 시 0인지 확인) |
| **출금액** | 금액 조건(최소 출금액 판단) |
| **거래시간** | 2호 심야구분 여부 |
| **폐업** | 2호 폐업 여부 |
| **키워드** | 3·4호 위험도키워드 저장, 5~10호 검색 텍스트 |
| **기타거래** | 3·4호 위험도키워드 대체, 5~10호 검색 텍스트 |
| **카테고리** | 5~10호 검색 텍스트, 7호는 "가상자산" 직접 비교 |

### category_table.json 매칭용 컬럼

| 컬럼명 | 용도 |
|--------|------|
| **카테고리** | 위험도 유형 이름 → 거래의 **위험도분류**와 매칭 |
| **위험도** | 위험도 수치 → 거래의 **위험도**와 매칭 |
| **업종코드** | 하한금액(원, 숫자) 또는 조건 설명 |
| **키워드** | 키워드·조건 설명(참고용) |

### 매칭 흐름

1. **risk_indicators**에서 1~10호 조건을 순서대로 적용하여, 각 행에 **위험도키워드**, **위험도분류**, **위험도**를 채움.
2. **_apply_분류_from_category**에서 행의 **위험도키워드**를 키로 category_table과 조회하여, **위험도분류**(카테고리), **위험도**를 최종 반영.
3. 호별 매칭 상세는 [위험도_호별_분류방법.md](위험도_호별_분류방법.md) 참고.
