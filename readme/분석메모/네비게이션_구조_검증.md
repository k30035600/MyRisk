# 네비게이션 구조 검증

## 1. 계층 구조

| 계층 | 경로 | 역할 | 네비게이션 |
|------|------|------|------------|
| **루트(홈)** | `/` | 서비스 선택 (은행/카드/금융정보 카드 3개) | 카드 클릭 → iframe으로 해당 서비스 열기. 「← 서비스 선택」은 iframe 열림 시에만 표시(플로팅). 「⛶ 전체 화면」은 FHD 이하에서만 표시. |
| **은행(MyBank)** | `/bank/`, `/bank/category`, `/bank/analysis/basic`, `/bank/help` | 은행거래 전처리·카테고리·기본분석·도움말 | main-nav: 🏠 홈 → **이 서비스 홈(/bank/)**, 전처리, 카테고리, 기본분석, 도움말 |
| **카드(MyCard)** | `/card/`, `/card/category`, `/card/analysis/basic`, `/card/help` | 신용카드 전처리·카테고리·기본분석·도움말 | main-nav: 🏠 홈 → **이 서비스 홈(/card/)**, 전처리, 카테고리, 기본분석, 도움말 |
| **금융(MyCash)** | `/cash/`, `/cash/category`, `/cash/analysis/basic`, `/cash/help` | 금융정보 병합·업종분류·종합분석·도움말 | main-nav: 🏠 홈 → **이 서비스 홈(/cash/)**, 병합작업, 업종분류, 종합분석, 도움말 |
| **루트 도움말** | `/help` | 통합 도움말 | main-nav: 🈁 홈(/) + 은행/카드/금융 각 메뉴 링크 |

## 2. 「홈」 하나로 통일 (← 서비스 선택 버튼 제거)

- **「🏠 홈」** = 항상 **루트(서비스 선택, 세 카드 화면)** 로 이동.
- **iframe으로 연 경우**: 앱 내 「홈」 클릭 시 iframe이 부모에게 `postMessage('myrisk-close-embed')`를 보내고, 부모가 iframe을 닫아 세 카드 화면을 다시 보여 줌. 별도의 「← 서비스 선택」 버튼 없이 **네비의 「홈」만** 사용.
- **직접 주소로 연 경우** (새 탭 등): 「홈」은 `href="/"` 로 루트로 이동.

## 3. 적용 사항

- **「🏠 홈」** = `href="/"` (루트). iframe 내부일 때는 클릭 시 `postMessage`로 부모가 iframe 닫기.
- **전처리/병합작업** = 각 서비스 인덱스(/bank/, /card/, /cash/). 인덱스 페이지에서 active는 이 항목에만 적용.
- 플로팅 「← 서비스 선택」 버튼 제거. 네비의 「홈」만으로 서비스 선택 화면 복귀.
- 루트 도움말(`/help`)의 「🏠 홈」은 `/` 유지 (루트가 맞음).
- **print_analysis.html**, **risk_report_page.html**, **opinion_fragment.html**: 인쇄/보고용 뷰로 main-nav 없음. 필요 시 상위 서비스(종합분석/도움말)로 돌아가는 링크만 있으면 됨.
- export 폴더 내 정적 HTML은 참고용이므로 동일 규칙으로 수정 가능하나, 앱 라우트와 동기화 여부는 선택.
