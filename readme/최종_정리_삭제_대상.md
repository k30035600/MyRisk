# MyRisk 최종 정리 — 수동 삭제 대상

파일이 사용 중이면 스크립트 삭제 시 "Access denied"가 납니다. **앱·에디터·터미널을 모두 닫은 뒤** 아래를 수동으로 삭제하세요. **readme 폴더는 삭제하지 않습니다.**

## 삭제해도 되는 항목

| 대상 | 설명 | 방법 |
|------|------|------|
| **share/** | 가상환경(venv) 잔여(man 페이지 등). `.gitignore`에 있음. | 폴더 통째로 삭제 |
| **Lib/** | 프로젝트 안에 만든 venv의 `Lib`(site-packages). 우리 코드는 **lib/**(소문자). `.gitignore`에 있음. | 폴더 통째로 삭제 (venv 불필요 시) |
| **Scripts/** (venv 쪽) | venv의 `Scripts`(pip, python.exe 등)인 경우만. **우리 스크립트가 있는 scripts/** 는 삭제하지 않음. | venv용 Scripts만 삭제. 프로젝트용 scripts/ 는 유지 |
| **__pycache__/** | Python 캐시. 어디든 삭제 가능. (app.py 기동 시에도 정리함) | 각 `__pycache__` 폴더 삭제. 루트·MyBank·MyCard·MyCash·lib 등 |
| **temp/** 안의 파일 | 임시 JSON 등. (app.py 기동 시에도 비움) | `temp/` 폴더는 두고, **안의 파일만** 삭제 |

## 삭제하지 말 것

- **readme/** — 문서·가이드 전부 유지
- **lib/** (소문자) — 우리 공통 모듈 패키지
- **scripts/** (소문자) — git-commit-push.ps1, json_정리.py 등 프로젝트 스크립트
- **data/, .source/** — 앱 데이터

## 권장 순서

1. 서버·Python·Cursor 등 **전부 종료**
2. **share/** 삭제
3. venv를 프로젝트 안에 둔 경우: **Lib/**, **Scripts/** (venv용) 삭제. (가상환경을 다시 쓰려면 나중에 `python -m venv .venv` 로 `.venv` 에 새로 만들기)
4. **__pycache__** — 루트, lib, MyBank, MyCard, MyCash 아래 각각 삭제
5. **temp/** — 폴더 안의 파일만 삭제

이후에는 **가상환경을 프로젝트 밖** 또는 **.venv** 에 두고, `.gitignore`에 이미 있는 `Scripts/`, `share/`, `Lib/` 이 저장소에 올라가지 않도록 두면 됩니다.
