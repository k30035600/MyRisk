# 위험도분류에서 하한금액·상한금액 정하는 방법

## 요약

- **하한금액(최소 출금액)**: `category_table.json`에서 **분류='위험도분류'** 인 행의 **업종코드**에 **숫자(원)**만 넣으면, 해당 호의 최소 출금액으로 사용됩니다. 거래의 출금액이 이 값 이상일 때만 해당 위험도분류가 적용됩니다.
- **상한금액**: 현재 코드에는 **구현되어 있지 않습니다**. 상한을 쓰려면 `MyCash/risk_indicators.py`의 `_parse_min_out` 확장 또는 별도 컬럼/규칙이 필요합니다.

---

## 1. 하한금액 설정 (현재 지원)

### 1.1 동작 방식

- **파일**: `data/category_table.json`
- **대상 행**: `분류` = `"위험도분류"` 인 행만 사용.
- **업종코드** 컬럼에 **숫자만** 넣으면(예: `1000000`, `5000000`), 그 값(원)이 해당 **카테고리(위험도분류명)** 의 **최소 출금액(min_out)** 으로 사용됩니다.
- `MyCash/risk_indicators.py`의 `_parse_min_out(업종코드)`가 업종코드를 숫자로 파싱하고, `_load_위험도분류_keywords()`에서 호별 `min_out`을 로드합니다.
- 매칭 시 **출금액 >= min_out** 일 때만 해당 위험도분류가 적용됩니다. 업종코드가 비어 있거나 숫자가 아니면 **0**(금액 조건 없음)으로 처리됩니다.

### 1.2 설정 예시 (category_table.json)

위험도분류 행에서 **업종코드**에만 숫자(원)를 넣으면 하한으로 사용됩니다.

```json
{
  "분류": "위험도분류",
  "키워드": "증권, 선물, 자산운용",
  "카테고리": "투기성지표",
  "위험도": "2",
  "업종코드": "500000"
}
```

- 위 예: **투기성지표** 호에 대해 **최소 출금액 50만 원**으로 설정. 출금액이 50만 원 이상인 거래만 이 호 조건(키워드 매칭 시) 적용 대상이 됩니다.

같은 카테고리(호)가 여러 행이면, 같은 호 내에서 **가장 작은 min_out**이 사용됩니다(코드에서 `min(cur, parsed_min)` 처리).

### 1.3 참고 코드 위치

- `MyCash/risk_indicators.py`
  - `_parse_min_out(업종코드)` : 업종코드 → 최소 출금액(원) 파싱
  - `_load_위험도분류_keywords()` : category_table에서 분류='위험도분류' 행만 읽어, 카테고리별 `keywords`, `위험도`, `min_out` 구성
- 매칭 시 `출금액 >= min_out` 조건: `apply_risk_indicators()` 내부에서 해당 호 적용 시 사용

---

## 2. 상한금액을 쓰고 싶을 때

현재는 **상한금액**을 읽거나 적용하는 로직이 없습니다. 사용하려면 예를 들어 다음 중 하나가 필요합니다.

1. **업종코드 규칙 확장**  
   예: `"1000000-50000000"` 형태로 저장하고, `_parse_min_out`와 별도로 `_parse_max_out`를 만들어 상한을 파싱한 뒤, 적용 시 `출금액 <= max_out` 조건을 추가.

2. **별도 컬럼 사용**  
   category_table에 `하한금액`, `상한금액` 컬럼(또는 동일 의미의 키)을 추가하고, `_load_위험도분류_keywords()`에서 읽어서 각 호별 min/max로 사용.

3. **기존 업종코드 의미 유지**  
   지금처럼 업종코드 = 하한(숫자)만 두고, 상한은 다른 필드(예: 키워드 열에 특수 형식)나 별도 JSON 설정으로 두고, risk_indicators에서만 새로 파싱·적용.

구체적으로 구현할 때는 `risk_indicators.py`의 `_load_위험도분류_keywords` 반환 구조에 `max_out`을 넣고, `apply_risk_indicators()`에서 `출금액 <= max_out` 검사만 추가하면 됩니다.

---

## 3. 정리

| 항목       | 설정 방법 |
|------------|-----------|
| **하한금액** | `category_table.json`에서 분류='위험도분류' 행의 **업종코드**에 **숫자(원)** 입력. 출금액 >= 해당 값일 때만 해당 호 적용. |
| **상한금액** | **미지원**. 위와 같이 규칙/컬럼 확장 후 `risk_indicators.py`에 상한 조건 추가 필요. |
