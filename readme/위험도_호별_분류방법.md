# 위험도 분류 — 호별 컬럼 매칭 방식 (한글)

**category_table.json만 사용합니다.** `lib/category_table_io.py`의 `get_risk_class_table_data()`·`get_risk_class_map_for_apply()`가 category_table의 위험도 분류 행(1~10호)으로 매칭합니다. ([업종분류_대체.md](업종분류_대체.md) 참고.)

금융정보(cash_after) 생성 시 **거래 데이터의 컬럼**과 **업종분류(위험도 분류) 매칭**을 이용해 각 행에 위험도(1~10호)를 부여합니다. 아래는 **호별로 어떤 컬럼을 보고, category_table 기반으로 어떤 컬럼과 매칭하는지** 한글로 정리한 내용입니다.

---

## 1. 공통 구조

### 1.1 거래 데이터(cash_after 행)에서 쓰는 주요 컬럼

| 컬럼명     | 용도 |
|-----------|------|
| **입금액** | 금액 조건(출금만 해당 시 0인지 확인) |
| **출금액** | 금액 조건(최소 출금액 판단) |
| **거래시간** | 2호 심야구분 여부 |
| **폐업**   | 2호 폐업 여부 |
| **키워드** | 3·4호 위험도키워드 저장, 5~10호 검색 텍스트 |
| **기타거래** | 3·4호 위험도키워드 대체, 5~10호 검색 텍스트 |
| **카테고리** | 5~10호 검색 텍스트, 7호는 "가상자산" 직접 비교 |

### 1.2 업종분류 컬럼 (category_table 기반)

아래 컬럼명은 category_table의 위험도 분류 행(1~10호)에서 사용됩니다.

| 컬럼명 | 용도 |
|--------|------|
| **업종분류** | 위험도 유형 이름 → 거래의 **위험도분류**와 매칭 |
| **위험도** | 위험도 수치 → 거래의 **위험도**와 매칭 |
| **업종코드** | 하한금액(원, 숫자) 또는 조건 설명(참고용). 숫자만 있으면 해당 호의 최소 출금액(하한)으로 사용. |
| **키워드** | 키워드·조건 설명(참고용) |

### 1.3 매칭 흐름 요약

1. **risk_indicators**에서 1~10호 조건을 순서대로 적용하여, 각 행에 **위험도키워드**, **위험도분류**, **위험도**를 채움.
2. **category_table 기반 매칭**에서 행의 **위험도키워드**를 키로 조회하여, **위험도분류**(업종분류), **위험도**를 최종 반영.

## 2. 1호 — 분류제외지표 (위험도 0.1)

| 구분 | 컬럼·매칭 |
|------|------------|
| **거래에서 보는 컬럼** | 없음(2~10호 조건에 해당하지 않는 경우 기본값) |
| **설정값** | 위험도분류: (비움 또는 분류제외지표), 위험도: 0.1, 위험도키워드: 비움 |
| **category_table 매칭** | **업종분류** = "분류제외지표", **위험도** = "0.1" 인 행과 대응 |

---

## 3. 2호 — 심야폐업지표 (위험도 0.5)

| 구분 | 컬럼·매칭 |
|------|------------|
| **거래에서 보는 컬럼** | **폐업**, **거래시간**, **출금액** |
| **조건** | (1) **폐업** = "폐업" 이면 금액 무관 2호 적용. (2) **거래시간**이 category_table '심야구분' 시간대이고 **출금액** ≥ 10만원이면 2호 적용. |
| **설정값** | 위험도키워드: "폐업" 또는 "심야" 또는 "폐업 심야", 위험도분류: 심야폐업지표, 위험도: 0.5 |
| **category_table 매칭** | **업종분류** = "심야폐업지표", **위험도** = "0.5", **키워드** = "심야 10만원 이상 및 폐업에 해당하는 거래" |

---

## 4. 3호 — 자료소명지표 (위험도 1.0)

| 구분 | 컬럼·매칭 |
|------|------------|
| **거래에서 보는 컬럼** | **출금액**, **키워드**, **기타거래** |
| **조건** | 2호 해당 행 제외, **출금액** ≥ 500만원 |
| **설정값** | 위험도키워드: 해당 행의 **키워드**(없으면 **기타거래**), 위험도분류: 자료소명지표, 위험도: 1.0 |
| **category_table 매칭** | **업종분류** = "자료소명지표", **위험도** = "1.0", **업종코드** = "출금 1000만원 이상(숫자만 넣으면 하한금액)", **키워드** = "키워드(또는 기타거래)" |

---

## 5. 4호 — 비정형지표 (위험도 1.5)

| 구분 | 컬럼·매칭 |
|------|------------|
| **거래에서 보는 컬럼** | **입금액**, **출금액**, **키워드**, **기타거래** |
| **조건** | 2호 제외, **출금액** ≥ 100만원, **입금액** ≤ 0, 같은 **키워드**(또는 **기타거래**)로 3회 이상 |
| **설정값** | 위험도키워드: 해당 행의 **키워드**(또는 **기타거래**), 위험도분류: 비정형지표, 위험도: 1.5 |
| **category_table 매칭** | **업종분류** = "비정형지표", **위험도** = "1.5", **업종코드** = "출금만 100만원 이상", **키워드** = "거래가 같은 키워드(특정인) 5회 이상" 등 |

## 6. 5호 ~ 10호

(5호 투기성지표, 6호 사기파산지표, 7호 가상자산지표, 8호 자산은닉지표, 9호 과소비지표, 10호 사행성지표 — 각 호별 **거래 컬럼**·**조건**·**설정값**·**category_table 매칭**은 위와 동일한 형식으로, [업종분류_대체.md](업종분류_대체.md) 및 `MyCash/risk_indicators.py` 참고.)

## 12. 호별 컬럼 매칭 요약표

| 호수 | 위험도분류 | 거래에서 사용하는 컬럼 | category_table 매칭 컬럼 |
|------|------------|------------------------|--------------------------|
| 1호 | 분류제외지표 | (기본값) | 업종분류, 위험도 |
| 2호 | 심야폐업지표 | 폐업, 거래시간, 출금액 | 업종분류, 위험도, 키워드 |
| 3호~10호 | (각 호별 위험도분류) | 입금액, 출금액, 카테고리, 키워드, 기타거래 등 | 업종분류, 위험도, 업종코드(하한금액), 키워드 |

- **거래 행의 위험도분류** = category_table 위험도 분류 행의 **업종분류** 값과 대응합니다.
- **거래 행의 위험도** = **위험도** 값과 대응합니다.
- **위험도키워드**는 category_table 매칭 시 키로 사용됩니다. **업종코드**에 숫자(원)만 넣으면 해당 호의 **하한금액**(최소 출금액)으로 사용됩니다.
