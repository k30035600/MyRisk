# 반응형 수정 시 발생할 수 있는 문제점 정리

프로젝트의 MyBank/MyCard/MyCash 템플릿과 공통 index를 기준으로, 반응형으로 수정했을 때 생길 수 있는 문제를 정리했습니다.

---

## 1. 인라인 스타일이 `@media`를 덮어씀

**원인**: 인라인 `style="..."`은 CSS의 `@media` 규칙보다 우선순위가 높습니다.

**사례** (MyBank/templates/category.html 등):

```html
<div class="bank_category-container" style="overflow-x: auto; overflow-y: auto; height: 50vh; max-height: 50vh; min-height: 50vh; flex: none;">
```

- `@media (max-width: 768px)`에서 `.bank_category-container`에 `height: 50vh` 등을 다시 지정해도, 인라인 `height: 50vh`가 그대로 적용됩니다.
- **대응**: 반응형으로 바꿀 높이/overflow는 인라인에서 제거하고, 클래스 + `@media`로만 제어하거나, JS로 뷰포트에 따라 클래스/인라인을 바꾸도록 합니다.

---

## 2. `body { overflow-x: hidden }`으로 인한 가로 스크롤 차단

**위치**: category.html, analysis_basic.html 등 `body { overflow-x: hidden; }`

**문제**:

- 작은 화면에서 테이블·차트가 뷰포트보다 넓으면, 가로 스크롤이 막혀 내용이 잘립니다.
- 테이블을 “작은 화면에서는 카드형으로 전환”하지 않고 유지하는 경우, 사용자가 끝까지 볼 수 없게 됩니다.

**대응**:

- 테이블/차트를 감싼 컨테이너에만 `overflow-x: auto`를 두고, `body`의 `overflow-x`는 제거하거나 `auto`로 두는 편이 안전합니다.
- 또는 작은 뷰포트에서는 테이블을 카드/리스트 형태로 바꾸는 방식으로 설계합니다.

---

## 3. 테이블: `table-layout: fixed` + 좁은 화면

**현상**:

- `#category-applied-table` 등에 `table-layout: fixed`와 컬럼 비율(%)이 사용됩니다.
- 화면이 좁아지면 컬럼당 픽셀이 줄어들고, `white-space: nowrap` + `text-overflow: ellipsis`만으로는 가독성이 크게 떨어집니다.

**문제**:

- 360px 이하에서도 같은 테이블 구조를 유지하면, 셀 안 글자가 과도하게 잘리거나 폰트가 너무 작아짐.
- `@media`에서 테이블용 폰트/패딩만 조정한 경우, 구조적 한계는 그대로입니다.

**대응**:

- 매우 좁은 구간(예: max-width: 480px)에서는 테이블을 숨기고 카드/리스트 레이아웃으로 전환하거나,
- 최소 컬럼 수만 남기고 나머지는 “상세 보기” 등으로 빼는 방식을 고려합니다.

---

## 4. `vh` 단위와 모바일 브라우저

**사용처**: `.chart-container`, `.bank_category-container` 등에 `height: 50vh`, `75vh`, `max-height: 50vh` 등.

**문제**:

- 모바일(특히 iOS Safari)에서 주소창 표시/숨김 시 `100vh`가 달라져 레이아웃이 튀거나, 50vh 영역이 너무 작아질 수 있습니다.
- 세로 방향으로 키보드가 올라오면 뷰포트 높이가 바뀌어 차트/테이블 영역이 다시 줄어듭니다.

**대응**:

- 중요한 높이는 `vh` 대신 `min-height` + `flex: 1` 등으로 “남는 공간 채우기” 위주로 하거나,
- `dvh`(dynamic viewport height) 등 지원 브라우저에 한해 보조적으로 사용하는 방법을 검토합니다.

---

## 5. Chart.js 컨테이너 높이와 리사이즈

**설정**: `responsive: true`, `maintainAspectRatio: false` 사용 중.

**문제**:

- Chart.js는 부모 요소의 크기에 맞춰 그리므로, 부모 높이가 `vh`나 리사이즈에 따라 바뀌면 차트가 자주 리플로우됩니다.
- 모바일에서 주소창·키보드로 뷰포트가 바뀔 때 깜빡이거나 비율이 어색해 보일 수 있습니다.

**대응**:

- 차트를 감싼 컨테이너에 명시적인 높이(또는 flex로 채운 높이)를 주고, `resize` 이벤트 시 `chart.resize()`를 호출하는 등, 리사이즈 타이밍을 한 번만 처리하도록 합니다.

---

## 6. 브레이크포인트·스타일 불일치

**현재**:

- MyBank category: 1200px, 768px, 480px, 360px
- MyCard category: 1200px, 768px (그 이하는 동일하지 않을 수 있음)
- index: 768px, 480px

**문제**:

- 같은 “작은 화면”인데 모듈마다 레이아웃이 달라지거나, 한 모듈만 깨져 보일 수 있습니다.
- 유지보수 시 어떤 breakpoint를 수정해야 할지 헷갈립니다.

**대응**:

- 공통 변수/주석으로 브레이크포인트를 정의해 두고 (예: 1200 / 768 / 480 / 360), 모든 템플릿이 같은 구간을 사용하도록 맞춥니다.

---

## 7. 스플릿 테이블(헤더/본문 분리)의 가로 스크롤 동기화

**구조**: `.split-table-header`(고정) + `.split-table-body-wrap`(스크롤).

**문제**:

- 본문만 `overflow-x: auto`로 가로 스크롤되면, 헤더는 그대로라서 **헤더와 본문 컬럼이 어긋납니다.**
- 반응형으로 테이블을 그대로 둔 채로 화면만 줄이면, 이 현상이 작은 화면에서 더 두드러집니다.

**대응**:

- 본문의 `scrollLeft`를 JS로 읽어서 헤더 테이블의 래퍼에도 같은 `scrollLeft`를 적용해, 가로 스크롤을 동기화합니다.
- 또는 작은 뷰포트에서는 스플릿 테이블을 쓰지 않고, 일반 테이블 + 가로 스크롤 한 컨테이너로만 쓰는 방법을 고려합니다.

---

## 8. 인쇄(`@media print`)와 반응형의 충돌

**현재**: 같은 파일에 `@media (max-width: ...)`와 `@media print`가 모두 있습니다. 인쇄용 스타일이 동적으로 삽입되는 부분도 있습니다.

**문제**:

- 모바일 뷰(좁은 뷰포트)에서 인쇄하면, “화면용” 좁은 레이아웃이 그대로 인쇄될 수 있습니다.
- 반대로 인쇄용으로 넓은 레이아웃을 기대했는데, 일부 반응형 규칙이 인쇄에도 적용되어 의도와 다르게 나올 수 있습니다.

**대응**:

- `@media print` 안에서는 반응형용 레이아웃(그리드 열 수, 숨김 등)을 초기화하거나, 인쇄 전용 클래스를 붙인 뷰에서만 인쇄하도록 합니다.

---

## 9. 터치 영역·가독성

**현상**:

- `@media (max-width: 768px)`에서 `.nav-item`의 `padding: 4px 6px`, `min-width: 80px` 등으로 버튼이 작아짐.
- 테이블 폰트가 11px로 유지되는 구간이 있어, 작은 화면에서 가독성·접근성 이슈가 될 수 있습니다.

**대응**:

- 터치 타겟은 최소 44×44px 정도를 유지하고, `@media (pointer: coarse)` 등으로 터치 기기만 패딩/크기를 키우는 방법을 검토합니다.
- 작은 뷰포트에서 본문/테이블 폰트는 12px 이상을 권장합니다.

---

## 10. 그리드/갭이 좁은 화면에서 과도한 비중

**사례**: `gap: 20px`, `gap: 15px`, `min-width: 120px`(nav) 등.

**문제**:

- 360px 이하에서 동일한 gap·min-width를 쓰면, 실제 콘텐츠 영역이 매우 줄어듭니다.

**대응**:

- `max-width: 360px` 등 아주 좁은 구간에서 `gap`, `padding`, `min-width`를 더 줄이거나, nav를 한 줄에 하나씩만 두는 식으로 조정합니다.

---

## 요약 체크리스트

| 구분 | 확인 사항 |
|------|------------|
| 스타일 우선순위 | 반응형으로 바꿀 값이 인라인 스타일에 있지 않은지 |
| 가로 스크롤 | body `overflow-x: hidden` 대신, 필요한 영역만 제한했는지 |
| 테이블 | 좁은 구간에서 카드/리스트 전환 또는 컬럼 축소 전략 |
| 높이 | `vh` 의존도를 줄이고, 모바일에서 키보드/주소창 영향 고려 |
| 차트 | 컨테이너 높이와 리사이즈 시점 일관성 |
| 브레이크포인트 | 모듈 간 동일한 breakpoint 사용 여부 |
| 스플릿 테이블 | 가로 스크롤 시 헤더–본문 `scrollLeft` 동기화 |
| 인쇄 | `@media print`와 반응형 규칙이 서로 덮어쓰지 않도록 |
| 터치·가독성 | 터치 영역 44px 이상, 작은 화면에서 폰트 12px 이상 |
| 좁은 화면 | 360px 이하에서 gap·min-width 추가 조정 |

이 문서는 `readme/`에 두고, 반응형 수정 시 위 항목을 점검하는 데 사용할 수 있습니다.
