# MyRisk 폴더 정리 가이드

## 현재 문제
- 루트에 파이썬 모듈·스크립트·JSON·HTML이 섞여 있음
- 분석/내보내기 결과물이 루트에 쌓임
- 실행 스크립트(move_*.py, start-server.* 등)가 루트에 있음

- **lib**: 업종분류·위험도는 category_table_io로 통합.

## JSON 정리 (한 번에)

MyRisk 아래 JSON을 **data / temp / export** 로 나누려면:

```powershell
python scripts/json_정리.py
```

| 폴더 | 넣는 JSON | 비고 |
|------|-----------|------|
| **data/** | category_table, bank_after, card_after, cash_after | 앱이 읽고 씀 |
| **temp/** | .cat_tbl_*, .data_*, _myrisk_json_temp.json | 임시 (실행 중 생성) |
| **export/** | *_summary, *_category, *_by_* JSON, *_analysis.html | 분석·내보내기 결과 (앱 미참조) |

.vscode/*.json(설정)은 건드리지 않습니다.

## 적용 순서 (추천)

### 1단계: scripts/ 사용 (적용됨)
- `scripts/` 폴더에 이동 스크립트가 있습니다.
- `python scripts/move_after_json_to_data_folder.py` — after JSON을 data/로 이동
- `python scripts/move_temp_json_to_temp_folder.py` — 임시 JSON을 temp/로 이동
- `python scripts/move_root_exports_to_export_folder.py` — 루트의 분석 결과물을 export/로 이동
- `start-server.ps1`, `git-commit-push.ps1` 등은 루트에 두어 바로 실행 가능하게 둠

### 2단계: export/ 사용 (폴더·스크립트 준비됨)
- `export/` 폴더가 있습니다. 루트의 분석/내보내기 결과물을 여기로 옮기세요.
- 한 번에 옮기기: `python scripts/move_root_exports_to_export_folder.py`
- 대상: `*_summary.json`, `*_category.json`, `*_by_*.json`, `*_analysis.html`

### 3단계: .gitignore 보강
- `export/` 를 Git 제외할지 결정. 제외 시 `.gitignore`에 `export/` 추가

### 4단계: 공통 모듈은 루트 유지
- `category_table_io.py` 등은 import 경로가 프로젝트 전반에 있으므로 **루트에 두는 것이 안전**
- 나중에 `lib/` 로 옮기려면 `sys.path` 및 모든 `import` 수정 필요
