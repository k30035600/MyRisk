# 금융정보 병합후 테이블·그래프 연동 및 적용후 그래프 1회 출력

## 1. 연동하지 못했던 이유 (원인)

- **그래프 갱신 방식**: `updateCategoryChartFromQuery()`가 `#category-query-content` 안의 `.category-query-row`를 `querySelectorAll`로 읽어, 각 행의 `data-i`로 `categoryQueryDataOriginal`에서 데이터를 꺼내 쓰도록 되어 있음.
- **실제 문제**:
  1. **같은 데이터 소스가 아님**: 초기 그리기는 `setTimeout(..., 0)` 안에서 DOM 기준으로 “보이는 행”만 읽음. 테이블은 이미 `renderCategoryQueryTable(data.data)`로 그려진 뒤인데, DOM 반영/타이밍에 따라 일부 환경에서 행을 못 찾거나, 빈 상태로 그래프가 그려질 수 있음.
  2. **호출이 여러 번 겹침**: `Promise.all([loadRightData(), loadCategoryQueryTable()])`로 병합 테이블이 1회 로드되더라도, 예전에는 `loadRightData()` 완료 시점에 `loadCategoryQueryTable()`을 한 번 더 불러 이중 호출이 났고, 각각 `renderCategoryQueryTable` → `setTimeout(updateCategoryChartFromQuery)`가 돌아 그래프가 2번 이상 그려짐.
  3. **초기 1회는 “테이블과 동일한 배열”로 그리지 않음**: 초기에는 “DOM에서 읽은 visible 행”이 아니라, “지금 막 그린 테이블과 동일한 `data.data`”로 그래프를 한 번만 그려야 연동이 명확함.

## 2. 적용후 그래프를 3번 그려야 했던 이유 (원인)

- **호출 경로**:
  1. `DOMContentLoaded` → `Promise.all([loadRightData(), loadCategoryQueryTable()])` → `loadCategoryQueryTable()` 1회 → `renderCategoryQueryTable()` → **setTimeout → updateCategoryChartFromQuery()** (1회).
  2. (과거) `loadRightData()` 완료 후 `loadCategoryQueryTable()` 재호출 → 다시 `renderCategoryQueryTable()` → **setTimeout → updateCategoryChartFromQuery()** (2회).
  3. 우측 필터(카드사/카드번호/년월) `change` 시 `loadCategoryQueryTable()` 호출(1846, 1853, 1861) → 한 번 더 `renderCategoryQueryTable()` → **setTimeout → updateCategoryChartFromQuery()** (3회).
- 즉, “적용후 그래프”가 안정적으로 보이려면 **같은 데이터가 올 때까지** 여러 번 그리기가 일어나서, 사용자 입장에서는 “3번 그려야 제대로 나온다”처럼 느껴졌음.

## 3. 수정 내용

- **연동**  
  - 병합 테이블과 동일한 배열로 그래프를 한 번만 그리도록 변경.  
  - `loadCategoryQueryTable()`에서 `renderCategoryQueryTable(data.data, ...)` 직후, **같은 `data.data`**로 `updateCategoryChartForQuery(data.data)`를 1회만 호출.  
  - 초기 그리기는 DOM을 읽지 않고 “테이블에 쓴 데이터”를 그대로 쓰므로, 테이블과 그래프가 항상 같은 데이터로 연동됨.

- **1회 출력**  
  - `renderCategoryQueryTable()` 안의 `setTimeout(updateCategoryChartFromQuery, 0)` 제거.  
  - 초기 그래프는 위에서만 `updateCategoryChartForQuery(data.data)` 1회 호출.  
  - 필터 변경 시에는 기존처럼 `filterCategoryQueryTable()` 마지막에서 `updateCategoryChartForQuery(filteredData)`만 호출되므로, 전체적으로 “적용후 그래프”는 로드 시 1번, 필터 변경 시 1번만 그려짐.

- **중복 로드 제거**  
  - `loadRightData()` 완료 시점의 `loadCategoryQueryTable()` 호출은 이미 제거된 상태이므로, 병합 테이블/그래프는 `Promise.all` 쪽 1회 로드만 유지.
