❓ 은행거래 도움말

은행거래 통합정보(MyBank)는 국민은행, 신한은행, 하나은행 등 거래 내역을 전처리·카테고리 분류·기본분석으로 관리합니다. 카테고리는 전처리 → 계정과목 분류 → 후처리 순으로 적용됩니다.

1. 처음 사용하는 경우 (빠른 시작)

  1. 원본 파일 준비: 은행에서 다운로드한 거래 내역(.xls 또는 .xlsx)을 MyRisk/.source/Bank 폴더에 넣습니다. 파일명에 국민은행, 신한은행, 하나은행 중 하나가 포함되어야 합니다. (예: 신한은행_거래내역_2024.xlsx)
  2. 전처리 실행: 상단 메뉴에서 전처리를 클릭한 뒤, 화면의 「전처리 실행」 버튼을 누릅니다. 서버가 .source/Bank의 엑셀을 읽어 bank_before를 만들고, 전처리 전·후 테이블을 화면에 띄웁니다.
  3. 카테고리 적용: 카테고리 메뉴로 이동해 「카테고리 생성」으로 규칙을 만든 다음(이미 있으면 수정 가능), 「카테고리 적용」을 누르면 bank_after가 생성됩니다.
  4. 분석 보기: 기본분석 메뉴에서 bank_after 기준 통계·차트를 확인합니다.

한 번만 위 순서대로 진행하면, 이후에는 원본만 갱신하고 전처리 → 카테고리 적용만 반복하면 됩니다.

4. 카테고리 분류 상세(은행)

은행 거래의 카테고리는 category_table.json(MyRisk/.source/category_table.json) 한 파일로 관리하며, 분류 컬럼 값에 따라 적용 시점과 대상이 다릅니다. 거래방법·거래지점 분류는 사용하지 않습니다.

1.1 적용 순서 요약

  1. 전처리: 원본 엑셀을 읽은 직후, 적요·내용·송금메모·거래점 네 컬럼에만 키워드→치환 규칙 적용. 적용 후 bank_before.xlsx 저장.
  2. 계정과목: bank_before를 읽은 뒤 before_text만 사용해 키워드 매칭 → 키워드·카테고리 컬럼 채움. (before_text는 취소 + 적요·내용·송금메모·거래점을 #로 연결한 문자열)
  3. 후처리: 계정과목 적용이 끝난 뒤, 저장 직전에 적요·내용·송금메모 세 컬럼에만 키워드→치환 규칙 적용. 적용 후 bank_after.xlsx 저장.

1.2 전처리 (분류 = 전처리)

  • 목적: 원본 텍스트를 통일·정리해 이후 계정과목 매칭 정확도를 높입니다. 예: "초록마을" → "초록증권", 오타·동의어 통합.
  • 대상 컬럼: 적요, 내용, 송금메모, 거래점 (은행 거래 표준 컬럼).
  • 규칙: category_table에서 분류 = 전처리인 행만 사용. 키워드에 있는 문자열(공백 무시 매칭)을 카테고리 값으로 치환. 긴 키워드부터 적용합니다.
  • 시점: .source/Bank 엑셀을 읽어 통합한 직후, bank_before.xlsx로 저장하기 전에 한 번만 수행됩니다. 후처리·계정과목은 이 단계에서 적용하지 않습니다.

1.3 계정과목 (분류 = 계정과목)

  • 목적: 거래 내용을 지출/수입 항목(외식, 의료비, 교통비, 이체 등)으로 분류합니다. 기본분석·금융정보 병합 시 카테고리별 집계에 사용됩니다.
  • 검색 대상: before_text 한 컬럼만 사용합니다. before_text = 취소 + 적요 + 내용 + 송금메모 + 거래점을 #로 이어 붙인 문자열(전처리 반영 후). 거래점은 포함, 기타거래 컬럼은 사용하지 않습니다.
  • 규칙: category_table에서 분류 = 계정과목인 행만 사용. 키워드에 여러 키워드를 /로 OR 조건 지정 가능(예: 파리바게뜨/베이커리). 키워드가 before_text에 포함되면 해당 행의 카테고리 값을 부여합니다.
  • 우선순위: 같은 행에 여러 규칙이 매칭되면 매칭된 키워드 길이가 긴 것이 이깁니다. 먼저 전체를 "기타거래"로 채운 뒤, 규칙을 긴 키워드 순으로 적용해 덮어씁니다.
  • 저장: 매칭된 행의 키워드 컬럼에는 실제로 매칭된 키워드(가장 긴 것), 카테고리 컬럼에는 해당 계정과목 값이 저장됩니다. 매칭되지 않은 행은 "기타거래"로 남습니다.

1.4 후처리 (분류 = 후처리)

  • 목적: 계정과목 분류가 끝난 뒤, 표시용·보고용으로 적요/내용/송금메모 문구를 한 번 더 치환합니다. 예: 상호 정리, 개인정보 마스킹 대체 표현.
  • 대상 컬럼: 적요, 내용, 송금메모 (거래점은 후처리 대상에 포함되지 않음).
  • 규칙: category_table에서 분류 = 후처리인 행만 사용. 키워드 → 카테고리 치환 방식은 전처리와 동일하며, 긴 키워드부터 적용합니다.
  • 시점: 계정과목 적용 후, bank_after.xlsx로 저장하기 직전에 수행됩니다.

1.5 분류별 요약표

분류대상 컬럼시점용도
전처리적요, 내용, 송금메모, 거래점before 저장 전원본 텍스트 통일·정리
계정과목before_text(취소+적요+내용+송금메모+거래점)after 생성 시키워드 매칭으로 카테고리·키워드 부여
후처리적요, 내용, 송금메모after 저장 직전표시용 문구 치환

참고: "카테고리 생성"은 bank_before 기준으로 category_table에 전처리·후처리·계정과목 규칙을 자동 생성·보강합니다. "카테고리 적용"을 실행하면 위 순서대로 전처리(이미 before에 반영됨) → 계정과목 → 후처리가 적용된 bank_after가 생성·갱신됩니다.

5. 파일 위치 정리

용도경로
은행 원본 엑셀MyRisk/.source/Bank — .xls, .xlsx (파일명에 국민은행/신한은행/하나은행 포함)
카테고리 규칙(은행·카드·금융정보 공통)MyRisk/.source/category_table.json
전처리 결과bank_before — .source 또는 MyBank 폴더
카테고리 적용 결과bank_after — .source 또는 MyBank 폴더 (기본분석에서 사용)

2. 사용 흐름

  1. .source/Bank에 은행별 거래 내역 .xls/.xlsx를 넣습니다. 파일명에 국민은행, 신한은행, 하나은행이 포함되어야 인식됩니다.
  2. 은행거래 전처리(/bank/)에서 「전처리 실행」을 클릭하면 전처리 규칙이 적용된 bank_before가 생성·갱신됩니다.
  3. 은행거래 카테고리(/bank/category)에서 「카테고리 생성」으로 규칙을 만든 뒤 필요 시 분류·키워드·카테고리를 수정하고, 「카테고리 적용」으로 bank_after를 생성·갱신합니다.
  4. 은행거래 기본분석(/bank/analysis/basic)에서 bank_after 기준 통계·차트를 확인합니다.

3. 메뉴·버튼별 안내

은행거래 전처리 (/bank/)

.source/Bank의 엑셀을 읽어 통합하고, 전처리 규칙을 적용한 뒤 bank_before를 저장합니다.

  • 「전처리 실행」: .source/Bank의 모든 .xls/.xlsx를 읽어 하나로 합친 뒤, category_table의 전처리 규칙을 적용하고 bank_before를 저장합니다. 화면에는 전처리 전·후 테이블이 표시됩니다.
  • 필터: 은행·년·월 드롭다운으로 보이는 범위를 줄일 수 있습니다. 테이블이 크면 필터로 구간을 좁혀 확인하세요.
  • 데이터가 없으면 .source/Bank에 파일이 있는지, 파일명에 은행명이 들어가는지 확인하세요.

은행거래 카테고리 (/bank/category)

카테고리 규칙(분류, 키워드, 카테고리)을 편집하고, 적용 결과를 확인합니다.

  • 「카테고리 생성」: bank_before를 기준으로 category_table에 전처리·후처리·계정과목 규칙을 자동으로 보강합니다. 이미 규칙이 있으면 덮어쓸 수 있으니 필요 시 백업 후 실행하세요.
  • 「카테고리 적용」: 현재 카테고리 테이블 규칙을 bank_before에 적용해 bank_after를 생성·갱신합니다. 적용 후 화면의 "카테고리 적용후" 테이블에서 결과를 확인할 수 있습니다.
  • 테이블에서 행 추가·삭제·수정으로 분류·키워드·카테고리를 직접 편집할 수 있습니다. 수정 후 반드시 「카테고리 적용」을 다시 실행해야 bank_after에 반영됩니다.

은행거래 기본분석 (/bank/analysis/basic)

bank_after 기준으로 전체 통계, 적요별·월별·은행/계좌별 집계와 차트를 제공합니다.

  • 거래건수, 입금·출금 합계, 순잔액 등 한눈에 보기.
  • 적요별·월별·은행·계좌별 필터와 차트로 패턴을 파악할 수 있습니다.

6. 문제 해결

데이터가 없음: .source/Bank에 .xls/.xlsx가 있고 파일명에 은행명이 포함되는지 확인하세요. Excel에서 파일을 닫은 뒤 전처리를 다시 실행하세요.

카테고리가 적용되지 않음: .source/category_table.json이 있는지, 「카테고리 생성」 후 「카테고리 적용」 순서로 실행했는지 확인하세요. bank_after를 다른 프로그램에서 열어두지 마세요.

통합 도움말: /help에서 은행·신용카드·금융정보 전체 안내를 볼 수 있습니다.

7. 자주 묻는 질문

Q. 지원하는 은행이 더 있나요?
현재 국민은행, 신한은행, 하나은행만 파일명에서 자동 인식합니다. 다른 은행은 동일한 컬럼 구조로 넣으면 전처리는 가능하지만, 파일명에 위 세 은행명 중 하나를 넣거나 코드에서 은행 매핑을 추가해야 할 수 있습니다.

Q. bank_before와 bank_after 차이는?
bank_before는 원본 엑셀에 전처리 규칙만 적용한 결과입니다. bank_after는 여기에 계정과목·후처리까지 적용한 최종 결과로, 기본분석과 금융정보 병합에서 사용합니다.

Q. 카테고리 규칙을 수동으로 수정하려면?
카테고리 메뉴에서 테이블의 분류·키워드·카테고리 셀을 직접 편집한 뒤, 반드시 「카테고리 적용」을 다시 눌러야 bank_after에 반영됩니다. 규칙 파일은 .source/category_table.json입니다.
