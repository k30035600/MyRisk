❓ 신용카드 도움말

신용카드 통합정보(MyCard)는 국민·신한·현대·하나 등 여러 카드사의 거래 내역을 하나로 모아 전처리 → 카테고리 분류 → 분석 순으로 관리합니다.

  • 8. 문제 해결

1. 처음 사용하는 경우 (빠른 시작)

  1. 원본 파일 준비: 카드사에서 다운로드한 거래 내역(.xls 또는 .xlsx)을 MyRisk/.source/Card 폴더에 넣습니다. 파일명에 국민카드, 신한카드, 현대카드, 하나카드 중 하나가 포함되어야 합니다. (예: 신한카드_김찬식_2024.xlsx)
  2. 전처리 실행: 상단 메뉴 전처리에서 「전처리 실행」 버튼을 누릅니다. .source/Card 엑셀이 통합되어 card_before가 생성·갱신되고, 전처리 전·후 테이블이 표시됩니다.
  3. 카테고리 적용: 카테고리 메뉴에서 「카테고리 생성」으로 규칙을 만든 뒤(이미 있으면 수정 가능), 「카테고리 적용」을 누르면 card_after가 생성됩니다.
  4. 분석 보기: 기본분석 메뉴에서 card_after 기준 통계·차트를 확인합니다.

2. 시작하기 (서버 실행·폴더 구조)

서버 실행

프로젝트 루트에서 start-server.bat 또는 python app.py를 실행한 뒤 브라우저로 접속합니다. 기본 주소는 http://localhost:8080(또는 5000)입니다. 신용카드 전처리는 /card/, 카테고리는 /card/category, 기본분석은 /card/analysis/basic입니다.

폴더 구조

  • MyRisk/.source/Card: 카드사에서 내려받은 거래 내역 원본(.xls, .xlsx). 파일명에 국민카드/신한카드/현대카드/하나카드 포함.
  • card_before: 전처리 결과(.source 또는 MyCard). 컬럼: 카드사, 카드번호, 이용일, 이용시간, 입금액, 출금액, 취소, 가맹점명, 사업자번호, 폐업 등.
  • MyRisk/.source/category_table.json: 카테고리 규칙(분류, 키워드, 카테고리). 은행·카드·금융정보 공통.
  • card_after: 카테고리 적용 최종 데이터. 기본분석에서 사용.

3. 지원 카드사 및 파일 형식

MyRisk/.source/Card에는 .xls 또는 .xlsx 파일만 사용합니다. PDF 등 다른 형식은 지원하지 않습니다.

  • 국민카드: .xls, .xlsx 지원
  • 신한카드: .xls, .xlsx 지원
  • 현대카드: .xls, .xlsx 지원
  • 하나카드: .xls, .xlsx 지원

파일명 예: 국민카드_김찬식_2021-2025.xlsx, 신한카드_김찬식_2024.xlsx, 현대카드_김찬식_2021-2025.xlsx, 하나카드_김찬식_2024-2025.xlsx

카드사명은 파일명의 첫 번째 _ 앞 부분에서 자동으로 추출됩니다. 예: 신한카드_김찬식_2024.xlsx → 카드사 = "신한카드"

4. 주요 기능

📊 전처리

MyRisk/.source/Card의 .xls/.xlsx를 읽어 표준 컬럼으로 통합합니다. 카드사는 파일명에서, 폐업 0은 공백으로 저장됩니다. MyRisk/.source/card_before.xlsx로 저장됩니다.

🏷️ 카테고리

분류(계정과목 등)와 키워드 기반 자동 분류를 지원합니다. 가맹점명에 키워드가 포함되면 해당 카테고리가 부여됩니다. category_table.json를 수정한 뒤 "카테고리 적용"으로 card_after를 갱신합니다.

📈 기본분석

전체 통계, 가맹점별·월별·카드사별·카드별 요약, 차트를 통해 거래 패턴을 빠르게 파악합니다.

5. 카테고리 분류 상세(신용카드)

신용카드 거래의 카테고리는 category_table.json(MyRisk/.source/category_table.json) 한 파일로 관리하며, 분류 컬럼 값에 따라 적용 시점과 대상이 다릅니다.

적용 순서 요약

  1. 전처리: 원본 엑셀을 읽은 직후 가맹점명·카드사 컬럼에 키워드→치환 규칙 적용. 적용 후 card_before 저장.
  2. 계정과목: card_before를 읽은 뒤 가맹점명만 사용해 키워드 매칭 → 키워드·카테고리 컬럼 채움.
  3. 후처리: 계정과목 적용이 끝난 뒤, 저장 직전에 가맹점명·카드사에 키워드→치환 규칙 적용. 적용 후 card_after 저장.

전처리 (분류 = 전처리)

  • 목적: 원본 가맹점명·카드사 문구를 통일해 계정과목 매칭 정확도를 높입니다.
  • 대상 컬럼: 가맹점명, 카드사.
  • 규칙: category_table에서 분류 = 전처리인 행만 사용. 키워드 → 카테고리 치환, 긴 키워드부터 적용.
  • 시점: .source/Card 엑셀 통합 후 card_before 저장 전에만 수행됩니다.

계정과목 (분류 = 계정과목)

  • 목적: 지출 항목별 분류(외식, 의료비, 교통비, 주거비 등). 가맹점명 기반 자동 분류.
  • 검색 대상: 가맹점명 한 컬럼만 사용합니다. (이용처·승인가맹점 등은 전처리 후 가맹점명으로 통일됨)
  • 규칙: category_table에서 분류 = 계정과목인 행만 사용. 키워드에 여러 키워드를 /로 OR 조건 지정(예: 파리바게뜨/베이커리). 가맹점명에 키워드가 포함되면 해당 카테고리를 부여합니다.
  • 우선순위: 여러 규칙이 매칭되면 매칭된 키워드 길이가 긴 것이 이깁니다. 먼저 전체를 "기타거래"로 채운 뒤, 규칙을 긴 키워드 순으로 적용해 덮어씁니다. "주식회사"→"(주)" 등 정규화 후 매칭합니다.
  • 저장: 매칭된 행의 키워드에는 실제로 매칭된 키워드(가장 긴 것), 카테고리에는 해당 계정과목 값이 저장됩니다.

후처리 (분류 = 후처리)

  • 목적: 계정과목 분류 후, 표시용으로 가맹점명·카드사 문구를 한 번 더 치환합니다.
  • 대상 컬럼: 가맹점명, 카드사.
  • 규칙: category_table에서 분류 = 후처리인 행만 사용. 키워드 → 카테고리 치환, 긴 키워드부터 적용.
  • 시점: 계정과목 적용 후, card_after 저장 직전에 수행됩니다.

분류별 요약표

분류대상 컬럼시점용도
전처리가맹점명, 카드사before 저장 전원본 텍스트 통일
계정과목가맹점명after 생성 시키워드 매칭으로 카테고리·키워드 부여
후처리가맹점명, 카드사after 저장 직전표시용 문구 치환

"카테고리 생성"은 card_before 기준으로 기본 규칙을 자동 생성하고, "카테고리 적용"은 위 순서대로 전처리(이미 before에 반영) → 계정과목 → 후처리를 적용해 card_after를 만듭니다. 규칙은 category_table.json에서 수동 수정 가능합니다.

6. 페이지별 사용법

신용카드 전처리 (/card/)

  • MyRisk/.source/Card에 국민/신한/현대/하나 카드사 거래 내역 .xls 또는 .xlsx를 넣습니다.
  • 전처리 페이지에서 "전처리 실행" 버튼을 클릭합니다. 서버가 .source/Card를 읽어 통합하고, MyRisk/.source에 card_before·card_after를 생성·갱신합니다. 카테고리 규칙은 MyRisk/category_table.json(신용카드 폐업)를 사용합니다.
  • 전처리 전/후 데이터를 테이블로 비교할 수 있습니다. 카드사, 년·월 등 필터로 범위를 줄여 확인하세요.
  • 데이터가 없으면 "MyRisk/.source/Card에 .xls/.xlsx가 있는지", "파일명에 카드사명이 포함되는지"를 확인합니다.

신용카드 카테고리 (/card/category)

  • 카테고리 테이블: 분류, 키워드, 카테고리 컬럼으로 구성됩니다. 행 추가·삭제·수정으로 규칙을 관리합니다.
  • "카테고리 생성": card_before.xlsx를 기준으로 계정과목 등 규칙을 자동 생성합니다. 기존 category_table.json의 신용카드 섹션이 있으면 갱신되므로, 필요 시 백업 후 실행하세요.
  • "카테고리 적용": 현재 카테고리 테이블 규칙을 card_before에 적용해 card_after를 생성·갱신합니다. 적용 후 "카테고리 적용후" 섹션에서 데이터 미리보기로 결과를 확인할 수 있습니다.
  • 카드사 필터·📄 출력: 페이지 헤더의 "카드사" 드롭다운으로 특정 카드사만 보거나, "📄 출력" 버튼으로 현재 화면을 인쇄·PDF 저장할 수 있습니다.
  • 카테고리별 통계, 월별 추이, 카테고리 그래프 등도 이 페이지에서 확인할 수 있습니다.

신용카드 기본분석 (/card/analysis/basic)

  • 전체 통계: 거래건수, 입금건수, 출금건수, 입금합계, 출금합계, 순잔액을 한눈에 확인합니다.
  • 가맹점별 분석: 가맹점(거래점)별 입금·출금·차액을 표시합니다. 행을 클릭하면 해당 가맹점의 상세 거래 내역이 나타납니다.
  • 입금/출금/차액 그래프: 가맹점별 입금·출금·차액을 막대 차트로 비교합니다.
  • 카드사별/카드별 집계: 카드사·카드번호 단위 집계와 파이 차트를 제공합니다.
  • 카드사 필터: 헤더의 "카드사" 드롭다운으로 특정 카드사만 분석할 수 있습니다.

8. 문제 해결

데이터가 표시되지 않음

  • MyRisk/.source/Card에 .xls 또는 .xlsx 파일이 있는지 확인하세요. 파일명에 국민카드/신한카드/현대카드/하나카드가 들어가야 합니다.
  • 전처리를 실행한 뒤 card_before.xlsx가 MyRisk/.source에 생성되었는지 확인하세요. 파일이 비어 있거나 Excel 등에서 열려 있으면 읽기 실패할 수 있습니다.
  • 브라우저 개발자 도구(F12) → Console 탭에서 오류 메시지를 확인하세요. API 오류 시 서버 터미널 로그도 함께 확인합니다.

카테고리·card_after가 생성/적용되지 않음

  • category_table.json가 MyRisk(프로젝트 루트)에 있는지 확인하세요. 없으면 카테고리 페이지에서 "카테고리 생성"을 먼저 실행합니다.
  • "카테고리 적용" 전에 카테고리 테이블에 필요한 규칙(분류, 키워드, 카테고리)이 들어 있는지 확인하세요.
  • card_after 파일을 다른 프로그램에서 열어둔 상태면 쓰기 오류가 날 수 있습니다. 파일을 닫은 뒤 다시 "카테고리 적용"을 실행하세요.

전처리·통합 오류

  • 원본 파일이 해당 카드사 표준 형식(이용일, 가맹점명, 입금액/출금액 등 필수 컬럼 포함)인지 확인하세요. 헤더 행이 정상 인식되지 않으면 데이터가 출력되지 않습니다.
  • 인코딩은 UTF-8 또는 CP949를 사용합니다. 한글이 깨지면 저장 시 인코딩을 바꿔 다시 시도해 보세요.
  • 카드사가 매칭되지 않으면 파일명이 카드사_기타.xlsx 형식인지 확인하세요. 첫 번째 _ 앞이 카드사명으로 사용됩니다.

서버 연결·실행 오류

  • 서버가 실행 중인지 확인하세요. python app.py 또는 start-server.bat으로 띄웁니다.
  • 포트 5000이 다른 프로그램에서 사용 중이면 연결이 안 될 수 있습니다.
  • 방화벽에서 localhost:5000 접속이 차단되지 않았는지 확인하세요.
