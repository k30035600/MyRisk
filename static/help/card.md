# 신용카드 도움말 (MyCard)

신용카드 통합정보는 **국민카드, 신한카드, 현대카드, 하나카드** 거래 내역을 전처리하고, 카테고리를 자동 분류한 뒤 분석하는 서비스입니다.

## 처음 사용하는 경우 (빠른 시작)

**1단계 - 원본 파일 넣기**

카드사에서 다운로드한 거래 내역(.xls 또는 .xlsx)을 **MyRisk/.source/Card** 폴더에 넣습니다. 이때 각 파일의 비밀번호를 해제합니다.

> 파일명 규칙: **카드사명_기타.xlsx**
> 예: `신한카드_김찬식_2024.xlsx`, `국민카드_거래내역.xlsx`
> 첫 번째 `_` 앞 부분이 카드사명으로 사용됩니다.

**2단계 - 전처리 실행**

상단 메뉴 **전처리**에서 **「전처리 실행」** 버튼을 누릅니다.

**3단계 - 카테고리 적용**

**카테고리** 메뉴에서 **「카테고리 생성」** → **「카테고리 적용」** 순서로 클릭합니다.

**4단계 - 분석 보기**

**기본분석** 메뉴에서 통계와 차트를 확인합니다.

## 지원 카드사 및 파일 형식

**.xls** 또는 **.xlsx** 파일만 사용할 수 있습니다. PDF 등은 지원하지 않습니다.

| 카드사 | 파일명 예시 |
|--------|------------|
| 국민카드 | `국민카드_김찬식_2021-2025.xlsx` |
| 신한카드 | `신한카드_김찬식_2024.xlsx` |
| 현대카드 | `현대카드_김찬식_2021-2025.xlsx` |
| 하나카드 | `하나카드_김찬식_2024-2025.xlsx` |

> 카드사명은 파일명의 **첫 번째 `_` 앞 부분**에서 자동으로 추출됩니다.

## 사용 흐름

1. **.source/Card**에 카드사별 거래 내역 파일을 넣습니다
2. **전처리** 페이지에서 **「전처리 실행」** 클릭 → **card_before** 생성
3. **카테고리** 페이지에서 **「카테고리 생성」** → 규칙 확인/수정 → **「카테고리 적용」** → **card_after** 생성
4. **기본분석** 페이지에서 card_after 기준 통계·차트 확인

## 메뉴·버튼별 안내

### 전처리 (/card/)

- **「전처리 실행」**: .source/Card의 모든 엑셀을 읽어 합치고 card_before를 저장합니다
- **필터**: 카드사·년·월로 범위를 줄일 수 있습니다
- 전처리 전/후 테이블로 결과를 비교합니다

### 카테고리 (/card/category)

- **「카테고리 생성」**: card_before 기준으로 규칙을 자동 생성합니다
- **「카테고리 적용」**: 규칙을 적용하여 card_after를 생성합니다
- 카드사 필터와 **「📄 출력」** 버튼을 사용할 수 있습니다
- 테이블에서 행 추가·삭제·수정으로 규칙을 직접 편집할 수 있습니다

### 기본분석 (/card/analysis/basic)

- 거래건수, 입금·출금 합계, 순잔액을 한눈에 확인
- 가맹점별·월별·카드사별·카드별 집계와 차트
- **분류 현황 바**: 테이블 상단에 대분류별(I_자금이동·II_필수생활비·III_재량소비 등) 분류 비율이 컬러 배지로 표시됩니다
- 카드사 필터와 **「📄 출력」** 으로 인쇄 가능. 인쇄 보고서에는 카테고리 분류 현황(대분류별 건수·입출금액)이 포함됩니다

## 카테고리 분류 상세

신용카드 거래의 카테고리는 **category_table.json** 파일로 관리되며, 은행·카드·금융정보가 동일한 테이블을 공유합니다.

### 적용 순서

```
전처리 → 계정과목 → 후처리
```

### 전처리 (분류 = 전처리)

- **언제**: 엑셀을 읽은 직후, card_before 저장 전
- **무엇을**: **가맹점명·카드사** 텍스트를 통일·정리
- **예**: 동일한 가맹점의 다른 표기를 하나로 통합

### 계정과목 (분류 = 계정과목)

- **언제**: card_after 생성 시
- **무엇을**: **가맹점명**에서 키워드를 찾아 카테고리 부여
- **키워드 규칙**:
  - `/`로 구분하면 **OR 조건** (예: `파리바게뜨/베이커리`)
  - `re:` 접두사로 **정규식** 사용 가능
  - "주식회사" → "(주)" 등 정규화 후 매칭
- **우선순위 (5단계 캐스케이드)**: 여러 키워드가 동시에 매칭되면 아래 순서로 결정합니다
  1. **키워드 길이** — 더 긴 키워드가 우선
  2. **위험도 우선순위** — 대분류 코드별 위험 순위가 높은 쪽 우선
  3. **텍스트 위치** — 가맹점명에서 먼저 나타나는 키워드 우선
  4. **소분류 코드** — 코드값이 큰 쪽 우선
  5. **기존 유지** — 동일하면 먼저 매칭된 것 유지
- **현금처리 보존**: 입금 행에 부여된 `현금처리` 라벨은 계정과목 적용 시 덮어쓰지 않습니다

### 후처리 (분류 = 후처리)

- **언제**: 계정과목 적용 후, card_after 저장 직전
- **무엇을**: **가맹점명·카드사** 문구를 보고서용으로 치환

### 계정과목 코드 체계

은행거래와 동일한 **대분류 7 → 중분류 13 → 소분류 34** 구조를 사용합니다.

| 대분류 | 소분류 코드 | 면책 심사 성격 |
|--------|------------|--------------|
| Ⅰ. 자금이동 | M01~M02 | 비소비성 |
| Ⅱ. 필수생활비 | L01~L08 | 필수 — 면책 인정 |
| Ⅲ. 재량소비 | D01~D08 | 소명 권고 |
| Ⅳ. 고위험항목 | H02~H04 | 면책불허가 |
| Ⅴ. 금융거래 | F01~F03 | 금융 — 채무 관련 |
| Ⅵ. 인적거래 | P01~P04 | 인적 — 소명 권고 |
| Ⅶ. 미분류 | X01~X06 | 재검토 필요 |

> 상세 규칙은 [계정과목 표준 검수 보고서](/category-audit-report-page)를 참고하세요.

### 분류별 요약표

| 분류 | 대상 컬럼 | 시점 | 용도 |
|------|-----------|------|------|
| 전처리 | 가맹점명, 카드사 | before 저장 전 | 텍스트 통일 |
| 계정과목 | 가맹점명 | after 생성 시 | 키워드 매칭 → 카테고리 부여 |
| 후처리 | 가맹점명, 카드사 | after 저장 직전 | 표시용 치환 |

## 파일 위치 정리

| 용도 | 경로 |
|------|------|
| 카드 원본 엑셀 | **MyRisk/.source/Card** — 파일명에 국민카드/신한카드/현대카드/하나카드 포함 |
| 카테고리 규칙 | **MyRisk/.source/category_table.json** |
| 전처리 결과 | **card_before** — .source 또는 MyCard 폴더 |
| 카테고리 적용 결과 | **card_after** — 기본분석·금융정보 병합에서 사용 |

## 문제 해결

**데이터가 표시되지 않아요**

- .source/Card에 .xls/.xlsx 파일이 있는지 확인하세요
- 파일명에 카드사명이 포함되어 있는지 확인하세요
- Excel에서 파일을 닫은 뒤 전처리를 다시 실행하세요

**카테고리가 적용되지 않아요**

- category_table.json이 있는지 확인하세요
- **「카테고리 생성」** → **「카테고리 적용」** 순서로 실행하세요
- card_after를 다른 프로그램에서 열어두지 마세요

**전처리·통합 오류**

- 원본 파일이 해당 카드사 표준 형식(이용일, 가맹점명, 금액 등 필수 컬럼)인지 확인하세요
- 카드사가 인식되지 않으면 파일명이 `카드사_기타.xlsx` 형식인지 확인하세요
