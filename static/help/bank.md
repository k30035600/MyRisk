# 은행거래 도움말 (MyBank)

은행거래 통합정보는 **국민은행, 신한은행, 하나은행** 거래 내역을 전처리하고, 카테고리를 자동 분류한 뒤 분석하는 서비스입니다.

## 처음 사용하는 경우 (빠른 시작)

아래 4단계만 따라 하면 됩니다.

**1단계 - 원본 파일 넣기**

은행에서 다운로드한 거래 내역(.xls 또는 .xlsx)을 **MyRisk/.source/Bank** 폴더에 넣습니다. 이때 각 파일의 비밀번호를 해제합니다.

> 파일명에 **국민은행**, **신한은행**, **하나은행** 중 하나가 반드시 들어가야 합니다.
> 예: `신한은행_거래내역_2024.xlsx`

**2단계 - 전처리 실행**

상단 메뉴에서 **전처리**를 클릭한 뒤, **「전처리 실행」** 버튼을 누릅니다.

**3단계 - 카테고리 적용**

**카테고리** 메뉴에서 **「카테고리 생성」** → **「카테고리 적용」** 순서로 클릭합니다.

**4단계 - 분석 보기**

**기본분석** 메뉴에서 통계와 차트를 확인합니다.

> 한 번만 위 순서대로 하면, 이후에는 원본만 바꾸고 전처리 → 카테고리 적용만 반복하면 됩니다.

---

## 사용 흐름

1. **.source/Bank**에 은행별 거래 내역 파일을 넣습니다
2. **전처리** 페이지에서 **「전처리 실행」** 클릭 → **bank_before** 생성
3. **카테고리** 페이지에서 **「카테고리 생성」** → 규칙 확인/수정 → **「카테고리 적용」** → **bank_after** 생성
4. **기본분석** 페이지에서 bank_after 기준 통계·차트 확인

---

## 메뉴·버튼별 안내

### 전처리 (/bank/)

.source/Bank의 엑셀 파일을 읽어 하나로 합치고, 전처리 규칙을 적용합니다.

- **「전처리 실행」**: 모든 엑셀을 읽어 합친 뒤 bank_before를 저장합니다
- **필터**: 은행·년·월 드롭다운으로 보이는 범위를 줄일 수 있습니다
- 데이터가 없으면 .source/Bank에 파일이 있는지, 파일명에 은행명이 들어가는지 확인하세요

### 카테고리 (/bank/category)

카테고리 규칙(분류, 키워드, 카테고리)을 편집하고 적용합니다.

- **「카테고리 생성」**: bank_before 기준으로 규칙을 자동 생성합니다
- **「카테고리 적용」**: 규칙을 적용하여 bank_after를 생성합니다
- 테이블에서 직접 행 추가·삭제·수정 가능합니다
- 수정 후 반드시 **「카테고리 적용」** 을 다시 눌러야 반영됩니다

### 기본분석 (/bank/analysis/basic)

bank_after 기준으로 통계와 차트를 보여줍니다.

- 거래건수, 입금·출금 합계, 순잔액을 한눈에 확인
- 적요별·월별·은행·계좌별 집계와 차트
- **「📄 출력」** 버튼으로 인쇄용 화면 생성

## 카테고리 분류 상세

은행 거래의 카테고리는 **category_table.json** 파일 하나로 관리되며, **분류** 값에 따라 적용 시점이 달라집니다.

### 적용 순서

```
전처리 → 계정과목 → 후처리
```

### 전처리 (분류 = 전처리)

- **언제**: 엑셀을 읽은 직후, bank_before 저장 전
- **무엇을**: 적요·내용·송금메모·거래점의 텍스트를 통일·정리
- **왜**: 오타나 다른 표현을 같은 단어로 바꿔서 이후 분류 정확도를 높입니다
- **예**: "초록마을" → "초록증권"

### 계정과목 (분류 = 계정과목)

- **언제**: bank_after 생성 시
- **무엇을**: before_text(취소+적요+내용+송금메모+거래점을 합친 것)에서 키워드를 찾아 카테고리 부여
- **왜**: 거래를 외식, 의료비, 교통비 등 항목별로 자동 분류합니다
- **규칙**: 키워드에 `/`를 쓰면 OR 조건 (예: `파리바게뜨/베이커리`)
- **우선순위**: 여러 키워드가 매칭되면 **글자수가 긴 것**이 우선입니다

### 후처리 (분류 = 후처리)

- **언제**: 계정과목 적용 후, bank_after 저장 직전
- **무엇을**: 적요·내용·송금메모의 문구를 한 번 더 치환
- **왜**: 보고서용으로 상호명 정리, 개인정보 대체 등

### 분류별 요약표

| 분류 | 대상 컬럼 | 시점 | 용도 |
|------|-----------|------|------|
| 전처리 | 적요, 내용, 송금메모, 거래점 | before 저장 전 | 텍스트 통일·정리 |
| 계정과목 | before_text | after 생성 시 | 키워드 매칭 → 카테고리 부여 |
| 후처리 | 적요, 내용, 송금메모 | after 저장 직전 | 표시용 문구 치환 |

## 파일 위치 정리

| 용도 | 경로 |
|------|------|
| 은행 원본 엑셀 | **MyRisk/.source/Bank** — 파일명에 국민은행/신한은행/하나은행 포함 |
| 카테고리 규칙 | **MyRisk/.source/category_table.json** |
| 전처리 결과 | **bank_before** — .source 또는 MyBank 폴더 |
| 카테고리 적용 결과 | **bank_after** — 기본분석에서 사용 |

## 문제 해결

**데이터가 없어요**

- .source/Bank에 .xls/.xlsx 파일이 있는지 확인하세요
- 파일명에 은행명(국민은행/신한은행/하나은행)이 포함되어 있는지 확인하세요
- Excel에서 파일을 닫은 뒤 전처리를 다시 실행하세요

**카테고리가 적용되지 않아요**

- category_table.json이 있는지 확인하세요
- **「카테고리 생성」** 후 **「카테고리 적용」** 순서로 실행했는지 확인하세요
- bank_after를 다른 프로그램에서 열어두지 마세요

## 자주 묻는 질문

**Q. 다른 은행도 사용할 수 있나요?**

현재는 **국민은행, 신한은행, 하나은행**만 파일명에서 자동 인식합니다. 다른 은행은 파일명에 위 세 은행명 중 하나를 넣으면 전처리가 가능하지만, 컬럼 구조가 다르면 코드 수정이 필요할 수 있습니다.

**Q. bank_before와 bank_after의 차이는?**

**bank_before**는 원본에 전처리 규칙만 적용한 결과입니다. **bank_after**는 여기에 계정과목·후처리까지 적용한 최종 결과로, 분석과 금융정보 병합에서 사용됩니다.

**Q. 카테고리 규칙을 직접 수정하려면?**

카테고리 페이지에서 테이블의 분류·키워드·카테고리 셀을 직접 편집한 뒤, **「카테고리 적용」** 을 다시 눌러주세요.
