# 금융정보 도움말 (MyCash)

금융정보는 **은행거래(bank_after)**와 **신용카드(card_after)**를 하나로 합쳐서 병합 관리하고, **위험도 분류**와 **종합 분석**을 제공하는 서비스입니다.

## 처음 사용하는 경우 (빠른 시작)

**1단계 - 은행·카드 데이터 준비**

먼저 **은행거래**와 **신용카드** 메뉴에서 각각 전처리·카테고리 적용을 끝내세요.

> bank_after, card_after가 만들어져 있어야 합니다. 둘 중 하나만 있어도 병합 가능합니다.

**2단계 - 금융정보 병합**

**병합작업** 메뉴에서 **「cash_after 생성(병합)」** 버튼을 누릅니다.

> 은행 + 카드 거래가 합쳐지고, 위험도 1~10호가 자동으로 적용됩니다.

**3단계 - 분석 확인**

- **업종분류**: 위험도별 필터와 출력
- **종합분석**: 통계·차트와 검토 종합의견

> **중요**: 금융정보 메뉴에 들어갈 때마다 기존 cash_after가 삭제됩니다. 데이터를 보려면 매번 **「cash_after 생성(병합)」** 을 실행해야 합니다.

## 사용 흐름

1. 은행·카드 전처리와 카테고리 적용을 먼저 완료합니다
2. **금융정보 병합작업**(/cash/)에서 **「cash_after 생성(병합)」** 클릭
3. **금융정보 업종분류**(/cash/category)에서 위험도별 필터·출력 확인
4. **금융정보 종합분석**(/cash/analysis/basic)에서 통계·차트·종합의견 확인

## 메뉴·버튼별 안내

### 병합작업 (/cash/)

은행(bank_after)과 카드(card_after)를 한 화면에서 보고 병합합니다.

- **「cash_after 생성(병합)」 / 「병합작업 다시 실행」**: 두 데이터를 합쳐 cash_after.json을 만들고 위험도를 적용합니다
- bank_after나 card_after가 없으면 오류 메시지가 나옵니다. 은행·카드 메뉴에서 먼저 전처리·카테고리 적용을 완료하세요

### 업종분류 (/cash/category)

- 카테고리 정의 테이블과 **위험도분류 적용후** 테이블을 제공합니다
- 위험도·위험도키워드·거래일·거래시간 내림차순으로 정렬됩니다
- 각 컬럼의 드롭다운으로 필터링하고 **「📄 출력」** 으로 인쇄할 수 있습니다
- 출력 시 1호(분류제외지표)는 제외되고, 위험도 0.1 이상만 인쇄됩니다

### 종합분석 (/cash/analysis/basic)

- cash_after 기준 전체 통계, 카테고리별·월별·금융사별 집계와 차트
- 우측 컨테이너에 **검토 종합의견**이 표시됩니다

## cash_after 생성(병합) 과정

**「cash_after 생성(병합)」** 버튼을 누르면 다음이 수행됩니다.

1. bank_after와 card_after를 하나로 합칩니다 (금융사·거래일·입출금액 등 컬럼 병합)
2. category_table 기반으로 업종분류·위험도 매칭을 적용합니다
3. **1~10호 위험도 지표**를 순서대로 적용합니다
4. 고위험 카테고리(가상자산·증권투자·금전대부) 보정으로 위험도 5.0 설정
5. 결과를 **MyCash/cash_after.json**에 저장합니다

## 위험도 분류 상세 (1~10호)

모든 거래의 기본 위험도는 **0.1**(1호, 분류제외지표)입니다. 2호부터 10호까지 순서대로 조건을 확인하고, 조건에 맞으면 **나중 호수가 이전 호수를 덮어씁니다**.

### 호별 조건 요약

| 호수 | 분류명 | 위험도 | 금액 조건 | 키워드·기타 조건 |
|------|--------|--------|-----------|-----------------|
| 1호 | 분류제외지표 | 0.1 | 없음 | 2~10호에 해당하지 않는 거래 |
| 2호 | 심야폐업지표 | 0.5 | 폐업: 없음 / 심야: 출금 10만원↑ | 폐업 업체 또는 심야 시간대 거래 |
| 3호 | 자료소명지표 | 1.0 | 출금 500만원↑ | 해당 행의 키워드(또는 기타거래) |
| 4호 | 비정형지표 | 1.5 | 출금 100만원↑ × 3회↑ | 같은 키워드(특정인) 반복 출금 |
| 5호 | 투기성지표 | 2.0 | 출금만 50만원↑ | 증권, 선물, 자산운용, 위탁 |
| 6호 | 사기파산지표 | 2.5 | 출금만 50만원↑ | 대부, P2P, 카드깡, 원리금 |
| 7호 | 가상자산지표 | 3.0 | 출금만 50만원↑ | 업비트, 빗썸, 코인원, 코빗, 가상자산, VASP 등 |
| 8호 | 자산은닉지표 | 3.5 | 출금만 50만원↑ | 해외송금, SWIFT, Wise, 외화송금 등 |
| 9호 | 과소비지표 | 4.0 | 출금만 30만원↑ | 백화점, 명품, 귀금속, 유흥, 유흥주점 등 |
| 10호 | 사행성지표 | 5.0 | 출금만 10만원↑ | 경마, 복권, 도박, 사설도박, 휴게텔, 안마 등 |

### 검색 텍스트 (5~10호)

5호부터 10호까지 키워드를 찾을 때 사용하는 텍스트는 **(카테고리 + 키워드) + 기타거래**를 합친 문자열입니다.

### 위험도키워드 저장 방식

- **1호**: 위험도키워드 없음
- **2호**: 심야 또는 폐업 조건 해당 시 위험도 0.5 설정
- **3·4호**: 해당 행의 키워드(또는 기타거래) 값을 저장
- **5~10호**: 검색 텍스트에서 처음 매칭된 키워드 하나를 저장

### 고위험 카테고리 보정

**가상자산·증권투자·금전대부** 등 고위험 카테고리에 해당하는 행은 **위험도 5.0**이 설정될 수 있습니다.

### 프린터 출력 시

**「📄 출력」** 클릭 시 **1호(분류제외지표) 행은 제외**하고, **위험도 0.1 이상**인 행만 인쇄됩니다.

## 문제 해결

**「cash_after 생성 중...」만 보이고 안 돼요**

생성에 실패하면 아래에 오류 사유가 표시됩니다. 흔한 원인:

- **bank_after와 card_after가 모두 없음**: 은행 또는 카드 메뉴에서 먼저 전처리·카테고리 적용을 완료하세요 (둘 중 하나만 있어도 OK)
- **읽기 오류**: after 파일이 정상 생성되었는지 해당 메뉴에서 확인하세요
- **쓰기 실패**: 한글 경로 문제일 수 있습니다. 한글이 없는 경로에서 작업하세요
- **시간 초과**: 데이터가 많으면 120초를 초과할 수 있습니다. 새로고침 후 다시 시도하세요

**데이터가 없습니다**

cash_after.json이 없거나 비어 있는 상태입니다. **「cash_after 생성(병합)」** 을 실행하세요.

**위험도가 모두 0.1이에요**

2~10호 조건에 맞는 거래가 없거나, cash_after를 다시 만들면 최신 규칙이 적용됩니다.

**한글 깨짐 / Add-Content 오류**

경로에 한글이 있으면 오류가 생길 수 있습니다. `readme/스크립트/start-cursor-no-hangul.ps1`을 실행한 뒤 한글이 없는 경로에서 작업하세요.

## 자주 묻는 질문

**Q. 금융정보에 들어가면 cash_after가 사라져요.**

의도된 동작입니다. 매번 최신 병합·위험도 결과를 만들기 위해, 금융정보 메뉴 진입 시 cash_after를 삭제합니다.

**Q. 위험도키워드에 이상한 값이 들어가요.**

3·4호는 해당 행의 키워드, 5~10호는 매칭된 키워드(증권, 대부 등)를 저장합니다. cash_after를 다시 생성하면 반영됩니다.

**Q. 종합 도움말은 어디서 보나요?**

홈에서 **금융정보 보고서** 또는 [/help](/help)로 접속하세요.
