# MyRisk 금융정보 분석 시스템 - 종합 도움말

MyRisk는 **은행 거래 내역**과 **신용카드 거래 내역**을 한곳에서 관리하고, 자동으로 분류·분석해 주는 웹 시스템입니다.

> 이 시스템은 **은행거래(MyBank)**, **신용카드(MyCard)**, **금융정보(MyCash)** 세 가지 서비스로 구성되어 있습니다.

## 전체 사용 순서

처음 사용하시는 분은 아래 순서를 따라 하세요.

### 1단계: 원본 파일 준비

- 은행에서 다운로드한 거래 내역 파일(.xls 또는 .xlsx)을 **MyRisk/.source/Bank** 폴더에 넣습니다. 이때 각 파일의 비밀번호를 해제합니다.
- 카드사에서 다운로드한 거래 내역 파일을 **MyRisk/.source/Card** 폴더에 넣습니다. 이때 각 파일의 비밀번호를 해제합니다.
- 파일명에 반드시 **은행명**(국민은행, 신한은행, 하나은행) 또는 **카드사명**(국민카드, 신한카드, 현대카드, 하나카드)이 포함되어야 합니다.

### 2단계: 전처리 실행

- 각 서비스(은행거래, 신용카드)의 **전처리** 페이지에서 **「전처리 실행」** 버튼을 클릭합니다.
- 시스템이 엑셀 파일을 읽어 표준 형식으로 변환합니다.

### 3단계: 카테고리 분류

- **카테고리** 페이지에서 **「카테고리 생성」** 버튼으로 분류 규칙을 만듭니다.
- **「카테고리 적용」** 버튼을 눌러 거래 내역에 카테고리를 자동으로 부여합니다.

### 4단계: 금융정보 병합

- **금융정보 병합작업** 페이지에서 **「cash_after 생성(병합)」** 버튼을 누릅니다.
- 은행 + 카드 거래가 하나로 합쳐지고, 위험도가 자동으로 매겨집니다.

### 5단계: 분석 확인

- 각 서비스의 **기본분석** 또는 **종합분석** 페이지에서 통계, 차트, 필터를 사용하여 결과를 확인합니다.

> 한 번만 위 순서대로 진행하면, 이후에는 원본 파일만 바꾸고 전처리 → 카테고리 적용만 반복하면 됩니다.

## 서버 실행 방법

프로젝트 폴더(MyRisk)에서 다음 중 하나를 실행하세요.

```
python app.py
```

또는 Windows에서:
- `start-server.bat` 더블 클릭
- PowerShell에서 `start-server.ps1` 실행

실행 후 브라우저에서 **http://localhost:8080** 으로 접속합니다.

> 서버를 끄려면 터미널에서 **Ctrl+C**를 누르세요.

## 서비스별 안내

### 은행거래 (MyBank)

- **지원 은행**: 국민은행, 신한은행, 하나은행
- 파일명에 은행명이 포함되어야 인식됩니다 (예: `신한은행_거래내역_2024.xlsx`)
- 카테고리 분류는 **전처리 → 계정과목 → 후처리** 순으로 적용됩니다
- 자세한 내용: [은행거래 도움말](/bank/help)

### 신용카드 (MyCard)

- **지원 카드사**: 국민카드, 신한카드, 현대카드, 하나카드
- 파일명 형식: `카드사_기타.xlsx` (예: `신한카드_김찬식_2024.xlsx`)
- 카테고리 분류는 **전처리 → 계정과목 → 후처리** 순으로 적용됩니다
- 자세한 내용: [신용카드 도움말](/card/help)

### 금융정보 (MyCash)

- 은행 + 카드 데이터를 하나로 합쳐 **병합 관리·종합 분석**합니다
- 병합 시 **1~10호 위험도 지표**가 자동으로 적용됩니다
- 자세한 내용: [금융정보 도움말](/cash/help)

## 페이지별 기능

### 전처리 페이지 (은행: /bank/, 카드: /card/)

- 원본 엑셀 파일을 표준 형식으로 변환합니다
- **「전처리 실행」** 버튼으로 실행
- 전처리 전/후 테이블로 결과를 비교할 수 있습니다
- **은행·카드사·년·월** 필터로 원하는 범위만 볼 수 있습니다

### 카테고리 페이지 (은행: /bank/category, 카드: /card/category)

- **「카테고리 생성」**: 분류 규칙을 자동으로 만듭니다
- **「카테고리 적용」**: 규칙을 거래 내역에 적용하여 최종 데이터를 생성합니다
- 테이블에서 분류·키워드·카테고리를 직접 편집할 수 있습니다
- 수정 후에는 반드시 **「카테고리 적용」** 을 다시 눌러야 반영됩니다

### 금융정보 병합작업 (/cash/)

- **「cash_after 생성(병합)」** 버튼으로 은행+카드 데이터를 합칩니다
- 병합 시 위험도 1~10호가 자동으로 적용됩니다

> **중요**: 금융정보 메뉴에 들어갈 때마다 기존 cash_after가 삭제됩니다. 데이터를 보려면 매번 병합을 실행해야 합니다.

### 기본분석·종합분석 페이지

- 전체 통계, 적요별·월별·은행별/카드사별 집계와 차트를 제공합니다
- **「📄 출력」** 버튼으로 인쇄용 화면을 띄울 수 있습니다

## 위험도 분류란?

금융정보(cash_after) 생성 시 각 거래에 **위험도 등급**이 매겨집니다.

| 호수 | 분류명 | 위험도 | 조건 요약 |
|------|--------|--------|-----------|
| 1호 | 분류제외지표 | 0.1 | 기본값 (해당 없는 거래) |
| 2호 | 심야폐업지표 | 0.5 | 폐업 업체 또는 심야 시간대 출금 |
| 3호 | 자료소명지표 | 1.0 | 출금 500만원 이상 |
| 4호 | 비정형지표 | 1.5 | 같은 곳에 출금 100만원 이상 × 3회 이상 |
| 5호 | 투기성지표 | 2.0 | 증권·선물·자산운용 관련 출금 50만원 이상 |
| 6호 | 사기파산지표 | 2.5 | 대부·P2P·카드깡 관련 출금 50만원 이상 |
| 7호 | 가상자산지표 | 3.0 | 업비트·빗썸 등 가상자산 관련 출금 50만원 이상 |
| 8호 | 자산은닉지표 | 3.5 | 해외송금·SWIFT 관련 출금 50만원 이상 |
| 9호 | 과소비지표 | 4.0 | 백화점·명품·유흥 관련 출금 30만원 이상 |
| 10호 | 사행성지표 | 5.0 | 경마·도박·복권 관련 출금 10만원 이상 |

> 숫자가 높을수록 위험도가 높습니다. 나중 호수가 이전 호수를 덮어씁니다.
>
> 상세한 키워드 목록과 저장 방식은 [금융정보 도움말](/cash/help)을 참고하세요.

## 문제 해결

### 페이지가 안 열려요 (404 오류)

- 서버가 실행 중인지 확인하세요 (`python app.py`)
- 서버를 막 시작했다면 잠시 기다린 후 새로고침(F5)해 보세요
- 주소가 정확한지 확인하세요 (아래 접속 주소 목록 참고)

### 데이터가 표시되지 않아요

- **.source/Bank** 또는 **.source/Card** 폴더에 엑셀 파일이 있는지 확인하세요
- 파일명에 **은행명** 또는 **카드사명**이 포함되어 있는지 확인하세요
- 해당 파일을 **Excel에서 닫은 상태**로 전처리를 실행하세요
- 금융정보에서 데이터가 없으면 **「cash_after 생성(병합)」** 을 실행하세요

### 카테고리가 적용되지 않아요

- **「카테고리 생성」** 을 먼저 실행했는지 확인하세요
- after 파일을 다른 프로그램(Excel 등)에서 열어두지 마세요
- **「카테고리 적용」** 을 다시 실행해 보세요

### 한글이 깨져 보여요

- 경로에 한글이 있으면 오류가 생길 수 있습니다
- `readme/스크립트/start-cursor-no-hangul.ps1`을 실행한 뒤 한글이 없는 경로(예: `C:\CursorWorkspace`)에서 작업하세요

## 자주 묻는 질문

**Q. MyBank와 MyCash의 차이는 뭔가요?**

MyBank는 **은행 거래만** 다룹니다. MyCash는 은행 + 카드 거래를 **합쳐서** 종합 분석합니다.

**Q. 전처리했는데 테이블이 비어 있어요.**

.source/Bank 또는 .source/Card 폴더의 파일명에 은행명/카드사명이 포함되어 있는지 확인하세요. Excel에서 파일을 닫고 다시 전처리하세요.

**Q. 포트 번호를 바꾸고 싶어요.**

환경 변수 `PORT`를 설정하면 됩니다. 예: `set PORT=9000` 후 `python app.py`

**Q. 금융정보에서 cash_after가 안 보여요.**

금융정보 메뉴에 들어갈 때마다 cash_after가 삭제됩니다. **「cash_after 생성(병합)」** 버튼을 눌러 다시 만드세요.

**Q. 위험도가 모두 0.1이에요.**

2~10호 조건에 맞는 거래가 없거나, cash_after를 다시 만들면 최신 규칙이 적용됩니다.
